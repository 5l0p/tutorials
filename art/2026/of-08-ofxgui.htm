openFrameworks - Part 8: ofxGui: Built-in UI Controls

Every creative coding sketch eventually needs a way to tweak parameters at runtime without recompiling. openFrameworks ships with ofxGui, a built-in addon that gives you sliders, toggles, buttons, and colour pickers in roughly ten lines of code. More importantly, ofxGui uses the ofParameter system, which means your application logic binds directly to the parameter value - no manual "read the slider, apply to variable" wiring required. This article shows the complete workflow from adding the addon to saving and loading presets.

What You Will Learn

  • How to add ofxGui to your project via the Project Generator's addon list
  • How to declare ofParameter variables and add them to an ofxPanel
  • The available controls: sliders (float/int), toggles, buttons, and colour pickers
  • How to group parameters with ofParameterGroup for organised panels
  • Saving and loading parameter presets to/from XML with panel.saveToFile()
  • Preventing the GUI from consuming mouse events your sketch also handles

Adding ofxGui to Your Project

ofxGui ships with every oF download. In the project generator, tick the ofxGui checkbox before generating. On the command line: 

./projectGenerator/projectGenerator --project apps/myApps/mySketch --addons ofxGui

Or open addons.make and add a line: 

ofxGui

Then include the header in ofApp.h: 

#include <ofMain.h>
#include <ofxGui.h>

The Core Types

The GUI system is built on two layers:

  • ofParameter<T> - a typed, named value that can notify listeners on change. Supports float, int, bool, ofColor, and more.
  • ofxPanel - a floating window that hosts controls. Each control is bound to an ofParameter.

A Basic GUI Setup

// ofApp.h
#pragma once
#include <ofMain.h>
#include <ofxGui.h>

class ofApp : public ofBaseApp {
public:
    void setup();
    void draw();
private:
    ofxPanel gui;

    ofParameter<float> speed;
    ofParameter<float> radius;
    ofParameter<int>   numCircles;
    ofParameter<bool>  showGrid;
    ofParameter<ofColor> fillColor;
};

// ofApp.cpp
#include "ofApp.h"

void ofApp::setup() {
    ofSetFrameRate(60);

    // Declare parameters: name, initial value, min, max
    speed.set("Speed", 1.0f, 0.0f, 5.0f);
    radius.set("Radius", 40.0f, 2.0f, 200.0f);
    numCircles.set("Count", 10, 1, 100);
    showGrid.set("Show Grid", false);
    fillColor.set("Colour", ofColor(80, 180, 255));

    gui.setup("Parameters");        // panel title
    gui.add(speed);
    gui.add(radius);
    gui.add(numCircles);
    gui.add(showGrid);
    gui.add(fillColor);
}

void ofApp::draw() {
    ofBackground(15, 15, 25);

    if (showGrid) {
        ofSetColor(40, 40, 60);
        for (int x = 0; x < ofGetWidth(); x += 40)
            ofDrawLine(x, 0, x, ofGetHeight());
        for (int y = 0; y < ofGetHeight(); y += 40)
            ofDrawLine(0, y, ofGetWidth(), y);
    }

    float t = ofGetElapsedTimef() * speed;
    for (int i = 0; i < numCircles; i++) {
        float angle = (float)i / numCircles * TWO_PI + t;
        float x = ofGetWidth()  / 2.0f + cos(angle) * 200.0f;
        float y = ofGetHeight() / 2.0f + sin(angle) * 200.0f;
        ofSetColor(fillColor);
        ofDrawCircle(x, y, radius);
    }

    gui.draw();    // always draw GUI last so it renders on top
}

Notice that speed, numCircles, showGrid, and fillColor are used directly in arithmetic and comparisons without any getter calls. ofParameter<T> has an implicit conversion to T, so it behaves like a plain variable in most contexts.

ofParameterGroup: Organising Controls

For larger sketches, group related parameters. ofxPanel::add() accepts a group, which renders as a collapsible sub-panel: 

ofParameterGroup motion;
ofParameterGroup appearance;

void ofApp::setup() {
    motion.setName("Motion");
    motion.add(speed.set("Speed", 1.0f, 0.0f, 5.0f));
    motion.add(numCircles.set("Count", 10, 1, 100));

    appearance.setName("Appearance");
    appearance.add(radius.set("Radius", 40.0f, 2.0f, 200.0f));
    appearance.add(fillColor.set("Colour", ofColor(80, 180, 255)));
    appearance.add(showGrid.set("Show Grid", false));

    gui.setup("My Sketch");
    gui.add(motion);
    gui.add(appearance);
}

ofxButton

Buttons are momentary: the bound bool is true for exactly one frame after the button is clicked. Use ofxButton for one-shot actions: 

ofxButton clearBtn;
ofxButton saveBtn;

gui.add(clearBtn.setup("Clear Canvas"));
gui.add(saveBtn.setup("Save PNG"));

void ofApp::update() {
    if (clearBtn) {
        strokes.clear();
    }
    if (saveBtn) {
        ofSaveScreen("output.png");
    }
}

Listening to Parameter Changes

You can register a callback that fires whenever a parameter changes, using ofAddListener: 

void ofApp::setup() {
    radius.set("Radius", 40.0f, 2.0f, 200.0f);
    ofAddListener(radius.changeEvent, this, &ofApp::onRadiusChanged);
}

void ofApp::onRadiusChanged(float &val) {
    ofLogNotice("ofApp") << "Radius changed to " << val;
    rebuildGeometry();   // recompute anything derived from radius
}

Saving and Loading Presets

ofxPanel can serialise all its parameters to an XML file: 

// Save current state
gui.saveToFile("preset_warm.xml");

// Load a preset
gui.loadFromFile("preset_warm.xml");

// Auto-save on exit / auto-load on startup (common pattern):
void ofApp::setup() {
    // ... build gui ...
    gui.loadFromFile("settings.xml");   // restore last session
}

void ofApp::exit() {
    gui.saveToFile("settings.xml");
}

The XML is human-readable and version-control friendly. An entry looks like: <speed>2.3</speed>. You can ship multiple preset files with your project and let users switch between them with a keyboard shortcut.

Positioning the Panel

gui.setPosition(ofGetWidth() - 220, 10);    // top-right corner
gui.setPosition(10, 10);                    // top-left (default)

// Hide / show the panel
void ofApp::keyPressed(int key) {
    if (key == 'h' || key == 'H') gui.setPosition(gui.getPosition().x == -1000
        ? ofVec2f(10, 10) : ofVec2f(-1000, 10));
    // simpler:
    if (key == TAB) guiVisible = !guiVisible;
}

void ofApp::draw() {
    // ...
    if (guiVisible) gui.draw();
}

Multiple Panels

ofxPanel mainPanel;
ofxPanel debugPanel;

void ofApp::setup() {
    mainPanel.setup("Main");
    mainPanel.add(/* ... */);
    mainPanel.setPosition(10, 10);

    debugPanel.setup("Debug");
    debugPanel.add(/* ... */);
    debugPanel.setPosition(240, 10);
}

void ofApp::draw() {
    // ...
    mainPanel.draw();
    debugPanel.draw();
}

ofxColorSlider

The colour parameter uses a combined HSB picker by default. You can also add HSB sliders individually for finer control - search the ofxGui examples for ofxGuiExtended, a community fork with more widget types.

Keyboard Focus and Mouse Capture

When the user is interacting with the GUI, you usually want to suppress whatever your sketch does with those mouse events. Check whether the mouse is over the GUI panel: 

void ofApp::mousePressed(int x, int y, int button) {
    if (gui.getShape().inside(x, y)) return;   // let gui handle it
    // ... your sketch logic
}

Summary

Declare ofParameter<float> members in your ofApp header, then register them with gui.add(param.set("Label", default, min, max)) in setup(). The parameter value is always current — read it directly in update() or draw(). Group related parameters with ofParameterGroup for collapsible panels. Call gui.saveToFile("settings.xml") and gui.loadFromFile() to persist presets. Check gui.getShape().inside(x, y) in mouse callbacks to avoid passing GUI clicks to your sketch logic.
