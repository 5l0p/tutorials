DSP - Part 15: Wavetable Synthesis

Every time you call sinf() in an audio loop you are burning a transcendental function evaluation — typically 20–40 clock cycles — to produce one sample. At 48 kHz with eight oscillators, that is hundreds of thousands of sine evaluations per second before your DSP has done anything interesting. A wavetable oscillator replaces that computation with a table lookup and an interpolation: the waveform is computed once, stored in an array, and replayed at any frequency by advancing a fractional index. Properly constructed, it is faster, sounds identical to a computed sine, and naturally extends to arbitrary waveforms that no closed-form formula can generate.

What You Will Learn

  • Why sinf() is expensive and what a single-cycle wavetable replaces it with.
  • How to fill a wavetable from a mathematical waveform and compute an oscillator using linear interpolation.
  • How cubic Hermite interpolation improves quality for modulated frequencies.
  • Why naive single-table oscillators alias at high frequencies and how band-limited tables fix it.
  • How to build a multi-octave band-limited wavetable set in C.
  • How to morph smoothly between two wavetables.

The Single-Cycle Wavetable

A single-cycle wavetable stores exactly one period of a waveform in an array of N floats. A phase accumulator advances through this array at a rate proportional to the desired output frequency. When the phase wraps past N it folds back to zero, producing a continuous periodic signal. 

#include <math.h>
#include <string.h>

#define WT_SIZE 2048   /* table size; power of two for efficient wrapping */

typedef struct {
    float table[WT_SIZE];
} Wavetable;

/* Fill a wavetable with a sine wave */
void wt_fill_sine(Wavetable *wt) {
    for (int i = 0; i < WT_SIZE; i++)
        wt->table[i] = sinf(2.0f * (float)M_PI * i / WT_SIZE);
}

/* Fill with a naive sawtooth (will alias - see band-limited section below) */
void wt_fill_saw_naive(Wavetable *wt) {
    for (int i = 0; i < WT_SIZE; i++)
        wt->table[i] = 2.0f * (float)i / WT_SIZE - 1.0f;
}

/* Fill with a naive square wave */
void wt_fill_square_naive(Wavetable *wt) {
    for (int i = 0; i < WT_SIZE; i++)
        wt->table[i] = (i < WT_SIZE / 2) ? 1.0f : -1.0f;
}

The Oscillator

The oscillator maintains a floating-point phase in the range [0, WT_SIZE). Each sample, it reads the table at the current phase using interpolation, then advances by increment = WT_SIZE * frequency / sample_rate. 

#define SAMPLE_RATE 48000

typedef struct {
    const Wavetable *wt;
    double phase;        /* current read position [0, WT_SIZE) */
    double increment;    /* phase advance per sample */
} WTOscillator;

void wt_osc_init(WTOscillator *osc, const Wavetable *wt, float freq) {
    osc->wt        = wt;
    osc->phase     = 0.0;
    osc->increment = (double)WT_SIZE * freq / SAMPLE_RATE;
}

void wt_osc_set_freq(WTOscillator *osc, float freq) {
    osc->increment = (double)WT_SIZE * freq / SAMPLE_RATE;
}

/* Linear interpolation read */
static float wt_read_linear(const float *table, double phase) {
    int i0   = (int)phase;
    int i1   = (i0 + 1) & (WT_SIZE - 1);   /* wrap with power-of-two mask */
    float t  = (float)(phase - (double)i0);
    return table[i0] + t * (table[i1] - table[i0]);
}

float wt_osc_next(WTOscillator *osc) {
    float out   = wt_read_linear(osc->wt->table, osc->phase);
    osc->phase += osc->increment;
    if (osc->phase >= WT_SIZE)
        osc->phase -= WT_SIZE;
    return out;
}

The power-of-two mask trick in (i0 + 1) & (WT_SIZE - 1) replaces a conditional branch with a bitwise AND for wrapping. This only works when WT_SIZE is a power of two — another reason to keep it that way.

Cubic Hermite Interpolation

Linear interpolation introduces a gentle low-pass roll-off that becomes audible when the oscillator is frequency-modulated quickly (vibrato, FM synthesis) because the error changes with each step. Cubic Hermite uses four points and fits a smooth cubic, giving much lower interpolation distortion for only a few extra multiply-adds. 

static float hermite4(float p0, float p1, float p2, float p3, float t) {
    float c0 =  p1;
    float c1 = (p2 - p0) * 0.5f;
    float c2 =  p0 - 2.5f*p1 + 2.0f*p2 - 0.5f*p3;
    float c3 = -0.5f*p0 + 1.5f*p1 - 1.5f*p2 + 0.5f*p3;
    return ((c3*t + c2)*t + c1)*t + c0;
}

static float wt_read_cubic(const float *table, double phase) {
    int i1 = (int)phase;
    int i0 = (i1 - 1) & (WT_SIZE - 1);
    int i2 = (i1 + 1) & (WT_SIZE - 1);
    int i3 = (i1 + 2) & (WT_SIZE - 1);
    float t = (float)(phase - (double)i1);
    return hermite4(table[i0], table[i1], table[i2], table[i3], t);
}

The Aliasing Problem

A naive sawtooth wave contains harmonics at every multiple of the fundamental: f, 2f, 3f, 4f, ... When the fundamental is 1 kHz and the sample rate is 48 kHz, the harmonics above 24 kHz (the Nyquist limit) cannot be represented and fold back into the audible spectrum as aliasing — a harsh, inharmonic buzzing.

For a single table at 48 kHz the highest representable harmonic is the one whose frequency is below 24 kHz. A 1 kHz saw has room for harmonics 1 through 23. A 10 kHz saw can only have harmonics 1 and 2. If you use the same table filled with infinite harmonics for all frequencies, the high-frequency notes sound dirty.

The fix is band-limited wavetables: pre-compute a separate version of the waveform for each octave, limiting the harmonics to those that fit below Nyquist at the frequencies that octave covers. At runtime, select the table appropriate for the current playback frequency.

Building Band-Limited Wavetables

The cleanest method is additive synthesis: for each table, sum only the harmonics that lie below Nyquist at the highest frequency in that octave's range. 

/* Number of octave tables covering the MIDI range (roughly 8 octaves) */
#define NUM_WT_TABLES 8

typedef struct {
    Wavetable tables[NUM_WT_TABLES];
    float top_freq[NUM_WT_TABLES];   /* highest fundamental this table is valid for (Hz) */
} BandlimitedWT;

/* Build a band-limited sawtooth wavetable set.
   tables[0] is for the lowest octave (most harmonics),
   tables[NUM_WT_TABLES-1] is for the highest (fewest harmonics). */
void bwt_build_saw(BandlimitedWT *bwt) {
    float nyquist = (float)SAMPLE_RATE / 2.0f;

    /* Each table covers one octave; top frequencies: 55, 110, 220, ..., 7040 Hz */
    float top = 55.0f;
    for (int t = 0; t < NUM_WT_TABLES; t++) {
        bwt->top_freq[t] = top * 2.0f;   /* inclusive upper limit for this band */

        /* Determine max harmonic number that fits below Nyquist at top frequency */
        int max_harm = (int)(nyquist / top);
        if (max_harm < 1) max_harm = 1;

        /* Additive synthesis: sum sine waves for each harmonic of a sawtooth */
        memset(bwt->tables[t].table, 0, sizeof(bwt->tables[t].table));
        for (int h = 1; h <= max_harm; h++) {
            float amp = 1.0f / h;    /* sawtooth harmonic amplitude */
            for (int i = 0; i < WT_SIZE; i++) {
                float angle = 2.0f * (float)M_PI * h * i / WT_SIZE;
                bwt->tables[t].table[i] += amp * sinf(angle);
            }
        }

        /* Normalise to peak 1.0 */
        float peak = 0.0f;
        for (int i = 0; i < WT_SIZE; i++) {
            float v = bwt->tables[t].table[i];
            if (v < 0) v = -v;
            if (v > peak) peak = v;
        }
        if (peak > 0.0f) {
            float inv = 1.0f / peak;
            for (int i = 0; i < WT_SIZE; i++)
                bwt->tables[t].table[i] *= inv;
        }

        top *= 2.0f;   /* next octave */
    }
}

/* Select the correct table for a given fundamental frequency */
const Wavetable *bwt_select(const BandlimitedWT *bwt, float freq) {
    for (int t = 0; t < NUM_WT_TABLES - 1; t++) {
        if (freq <= bwt->top_freq[t])
            return &bwt->tables[t];
    }
    return &bwt->tables[NUM_WT_TABLES - 1];
}

Gotcha: switching tables when the frequency crosses an octave boundary can cause a discontinuity if the waveform phase happens to be in a different position in the new table. In practice this is inaudible during normal note progression because table switches coincide with frequency changes the listener is already hearing. If you need absolutely seamless table transitions (for smooth pitch sweeps), crossfade between the old and new table over a brief window.

Wavetable Morphing

Morphing blends two wavetables together at a given mix ratio. The simplest approach is linear interpolation between the two tables, sample by sample. The result sounds like a smooth timbre transition — for example, fading from a sine to a saw, or between two user-defined waveforms. 

typedef struct {
    const Wavetable *wt_a;
    const Wavetable *wt_b;
    double phase;
    double increment;
    float  morph;   /* 0.0 = pure wt_a, 1.0 = pure wt_b */
} MorphOscillator;

void morph_osc_init(MorphOscillator *osc,
                    const Wavetable *a, const Wavetable *b,
                    float freq, float morph) {
    osc->wt_a      = a;
    osc->wt_b      = b;
    osc->phase     = 0.0;
    osc->increment = (double)WT_SIZE * freq / SAMPLE_RATE;
    osc->morph     = morph;
}

float morph_osc_next(MorphOscillator *osc) {
    float a = wt_read_linear(osc->wt_a->table, osc->phase);
    float b = wt_read_linear(osc->wt_b->table, osc->phase);
    float out = a + osc->morph * (b - a);

    osc->phase += osc->increment;
    if (osc->phase >= WT_SIZE)
        osc->phase -= WT_SIZE;
    return out;
}

You can drive morph from an LFO or envelope to create animated timbres. Modern synthesisers (Serum, Vital, WaveTable) extend this to tens or hundreds of wavetables arranged in a morph sequence, but the underlying mechanism is identical.

Complete Multi-Wavetable Oscillator

/* Full oscillator: band-limited table selection + cubic interpolation + morph */

typedef struct {
    const BandlimitedWT *bwt_a;
    const BandlimitedWT *bwt_b;
    double phase;
    double increment;
    float  morph;
    float  freq;
} FullWTOscillator;

void full_osc_init(FullWTOscillator *osc,
                   const BandlimitedWT *a, const BandlimitedWT *b,
                   float freq, float morph) {
    osc->bwt_a     = a;
    osc->bwt_b     = b;
    osc->phase     = 0.0;
    osc->increment = (double)WT_SIZE * freq / SAMPLE_RATE;
    osc->morph     = morph;
    osc->freq      = freq;
}

void full_osc_set_freq(FullWTOscillator *osc, float freq) {
    osc->freq      = freq;
    osc->increment = (double)WT_SIZE * freq / SAMPLE_RATE;
}

float full_osc_next(FullWTOscillator *osc) {
    const Wavetable *ta = bwt_select(osc->bwt_a, osc->freq);
    const Wavetable *tb = bwt_select(osc->bwt_b, osc->freq);

    float a   = wt_read_cubic(ta->table, osc->phase);
    float b   = wt_read_cubic(tb->table, osc->phase);
    float out = a + osc->morph * (b - a);

    osc->phase += osc->increment;
    if (osc->phase >= WT_SIZE)
        osc->phase -= WT_SIZE;
    return out;
}

Summary

A wavetable oscillator pre-computes one cycle of a waveform into an array and replays it by advancing a fractional phase. Cubic Hermite interpolation gives clean results for modulated frequencies. Naive waveforms alias at high frequencies; band-limited tables fix this by computing a separate Fourier-limited version per octave. Morphing between two tables is linear interpolation of the table outputs, enabling continuously variable timbres.

Part 16 goes deeper: FM synthesis, where one oscillator modulates the frequency of another. The same wavetable oscillator built here is the carrier and modulator in a two-operator FM pair.
