DSP - Part 9: Dynamics: Compressor
2026-09-02


A compressor reduces the dynamic range of an audio signal: loud passages are turned down
so that the average level can be raised without clipping. Understanding how to implement
one from scratch gives you insight into every dynamic-range processor - limiters, gates,
and multi-band compressors all use the same fundamental building blocks. This article
builds a complete mono compressor in C, then extends it to stereo with linked detection.


What You Will Learn


The feed-forward compressor architecture: level detection → gain computer → smoothing → gain application
How to compute the gain reduction from threshold, ratio, and knee parameters
Attack and release smoothing of the gain control signal
Make-up gain: restoring average level after compression
Stereo linking: taking the maximum of left and right levels for coupled gain reduction
How to convert the compressor into a brickwall limiter by setting ratio to infinity


Compressor Architecture


A feed-forward compressor has four stages in series:



Level detector: measures the input signal's level (peak or RMS, in dB).
Gain computer: applies the threshold/ratio/knee curve to compute how much
    gain reduction (in dB) is needed.
Gain smoother: applies attack and release to the gain reduction signal so it
    does not change abruptly.
Gain application: multiplies the audio signal by the computed gain,
    plus make-up gain.



The key insight is that the gain is computed and smoothed in the gain domain
(dB), not the audio domain. This gives independent control of the shape of gain
reduction and the timing.


Level Detection


We use the peak envelope follower from the previous article and convert its output to dB:



#include <math.h>

#define SAMPLE_RATE 48000

static float time_to_coeff(float ms) {
    return expf(-1.0f / (ms * 0.001f * SAMPLE_RATE));
}

static float lin_to_db(float x) {
    if (x <= 1e-10f) return -200.0f;
    return 20.0f * log10f(x);
}

static float db_to_lin(float db) {
    return powf(10.0f, db / 20.0f);
}


The Gain Computer


The gain computer maps input level in dB to gain reduction in dB according to the
threshold, ratio, and knee parameters.



Threshold: the level (dBFS) above which compression begins.
Ratio: the compression ratio. A ratio of 4:1 means that for every 4 dB above
    threshold, the output rises only 1 dB.
Knee width: a soft knee rounds the threshold transition over a range of
    ±knee_dB/2 around the threshold.



typedef struct {
    float threshold_db;
    float ratio;
    float knee_db;       /* 0 = hard knee */
} GainComputer;

/* Returns gain reduction in dB (always <= 0) */
float gain_computer_process(const GainComputer *gc, float input_db) {
    float x = input_db - gc->threshold_db;

    if (gc->knee_db > 0.0f) {
        /* Soft knee */
        float half = gc->knee_db / 2.0f;
        if (x < -half) {
            return 0.0f;
        } else if (x < half) {
            float t = (x + half) / gc->knee_db;
            x = x + (1.0f / gc->ratio - 1.0f) * t * t * gc->knee_db / 2.0f;
            return x - (input_db - gc->threshold_db);
        }
    } else {
        /* Hard knee */
        if (x <= 0.0f) return 0.0f;
    }

    /* Above threshold (and outside knee): apply ratio */
    float gr = x - x / gc->ratio;
    return -gr;
}


Attack and Release on the Gain Signal


The gain reduction signal is smoothed with separate attack and release time constants.
This is applied in the dB domain. Positive gain reduction values (more
compression) get the attack coefficient; falling gain reduction gets the release
coefficient.



typedef struct {
    float attack_coeff;
    float release_coeff;
    float gain_db;  /* current smoothed gain reduction (dB, <= 0) */
} GainSmoother;

void gain_smoother_init(GainSmoother *gs,
                        float attack_ms, float release_ms) {
    gs->attack_coeff  = time_to_coeff(attack_ms);
    gs->release_coeff = time_to_coeff(release_ms);
    gs->gain_db       = 0.0f;
}

/* target_db: gain reduction from the gain computer (<= 0)
   Returns smoothed gain reduction in dB */
float gain_smoother_process(GainSmoother *gs, float target_db) {
    /* More negative = more compression = attack direction */
    float c = (target_db < gs->gain_db)
              ? gs->attack_coeff
              : gs->release_coeff;
    gs->gain_db = c * gs->gain_db + (1.0f - c) * target_db;
    return gs->gain_db;
}


The Full Mono Compressor


typedef struct {
    GainComputer comp;
    GainSmoother smoother;
    float        env;           /* peak envelope (linear) */
    float        att_coeff;
    float        rel_coeff;
    float        makeup_db;
} Compressor;

void compressor_init(Compressor *c,
                     float threshold_db, float ratio, float knee_db,
                     float attack_ms, float release_ms,
                     float makeup_db) {
    c->comp.threshold_db = threshold_db;
    c->comp.ratio        = ratio;
    c->comp.knee_db      = knee_db;
    gain_smoother_init(&c->smoother, attack_ms, release_ms);
    c->att_coeff  = time_to_coeff(attack_ms);
    c->rel_coeff  = time_to_coeff(release_ms);
    c->env        = 0.0f;
    c->makeup_db  = makeup_db;
}

float compressor_process(Compressor *c, float x) {
    /* 1. Level detection: peak envelope */
    float abs_x = fabsf(x);
    float coeff = (abs_x > c->env) ? c->att_coeff : c->rel_coeff;
    c->env = coeff * c->env + (1.0f - coeff) * abs_x;

    /* 2. Convert to dB */
    float input_db = lin_to_db(c->env);

    /* 3. Gain computer */
    float gr_db = gain_computer_process(&c->comp, input_db);

    /* 4. Smooth the gain reduction */
    float smooth_gr = gain_smoother_process(&c->smoother, gr_db);

    /* 5. Apply gain + make-up gain */
    float total_gain = db_to_lin(smooth_gr + c->makeup_db);
    return x * total_gain;
}

void compressor_render(Compressor *c, const float *in,
                       float *out, int n) {
    for (int i = 0; i < n; i++)
        out[i] = compressor_process(c, in[i]);
}


Make-Up Gain


Compression reduces the overall level, so it sounds quieter even though peaks are
controlled. Make-up gain compensates for this. A common automatic approximation is:



/* Estimate make-up gain for a given threshold and ratio.
   Not exact - use as a starting point and adjust by ear. */
float auto_makeup(float threshold_db, float ratio) {
    return -(threshold_db * (1.0f - 1.0f / ratio)) / 2.0f;
}


Stereo Linking


For stereo material the left and right channels must receive the same gain reduction,
or the stereo image will shift as compression engages. The standard approach is to
detect on the maximum of the two channels (sum or max):



typedef struct {
    Compressor ch[2];
} StereoCompressor;

void stereo_compressor_process(StereoCompressor *sc,
                               float in_l, float in_r,
                               float *out_l, float *out_r) {
    /* Detect on max of L and R */
    float abs_l = fabsf(in_l);
    float abs_r = fabsf(in_r);
    float det   = (abs_l > abs_r) ? abs_l : abs_r;

    /* Drive both channels with linked detection level */
    float env   = sc->ch[0].env;
    float coeff = (det > env) ? sc->ch[0].att_coeff
                               : sc->ch[0].rel_coeff;
    env = coeff * env + (1.0f - coeff) * det;
    sc->ch[0].env = sc->ch[1].env = env;

    float input_db  = lin_to_db(env);
    float gr_db     = gain_computer_process(&sc->ch[0].comp, input_db);
    float smooth_gr = gain_smoother_process(&sc->ch[0].smoother, gr_db);

    float gain = db_to_lin(smooth_gr + sc->ch[0].makeup_db);
    *out_l = in_l * gain;
    *out_r = in_r * gain;
}


Limiter Variant


A limiter is simply a compressor with an extremely high ratio (10:1 or infinity:1) and
a very fast attack (0.1–1 ms). Setting ratio = 1000.0f and
attack_ms = 0.1f turns the compressor above into a brick-wall limiter.


Summary


A feed-forward compressor detects the input level, computes how much gain
reduction to apply (based on threshold, ratio, and knee), smooths that gain
signal with attack and release filters, and multiplies the signal by the
result. Make-up gain compensates for the average level reduction. Stereo
linking couples left and right channels by using the maximum of their envelope
values so the stereo image does not shift during compression. Setting ratio to
∞ and attack to zero gives a brickwall limiter.
