Containers - Part 9: Virtual Filesystems: proc, sysfs & devtmpfs

A freshly created container namespace has no /proc, no /sys, no /dev. Any program that reads process information, inspects kernel parameters, or opens device files will fail. This article explains what the three main virtual filesystems are, why each container needs its own fresh mount, and the exact mount calls required to set them up correctly.

What You Will Learn

  • What virtual filesystems are and how the kernel synthesises their content on demand
  • Why /proc must be remounted in a new PID namespace, and the correct mount flags to use
  • Why /sys is mounted read-only in containers and what it exposes
  • How to set up /dev with a tmpfs, essential device nodes, devpts with newinstance, and /dev/shm
  • A complete Go setupMounts() function that handles all of the above
  • Which /proc and /sys paths runc masks with bind mounts to prevent information leaks

What Is a Virtual Filesystem?

A virtual filesystem (VFS-backed, sometimes called a pseudo-filesystem) has no backing storage on disk. Its files and directories are synthesised by the kernel on demand. When you read /proc/self/status, the kernel generates the content dynamically from internal process data structures. Nothing is ever written to a block device. The filesystem type string passed to mount() tells the kernel which driver to use.

procfs: /proc

procfs (type proc) exports kernel and process information. Key directories and files:

  • /proc/<pid>/ - per-process information: status, maps, fd, ns, cmdline, environ
  • /proc/self/ - symlink to the calling process's directory
  • /proc/meminfo - memory statistics (reads from the host, unless overridden)
  • /proc/cpuinfo - CPU information
  • /proc/sys/ - kernel tuning parameters (writable with CAP_SYS_ADMIN)
  • /proc/mounts - the process's mount table (symlink to self/mounts)

Inside a container with a new PID namespace, /proc must be remounted. If the container inherits the host's /proc mount, it shows all host PIDs - and tools like ps, top, and kill will operate on host processes. A fresh proc mount inside the new PID namespace shows only the container's processes. 

/* Mount /proc inside a container */
#include <sys/mount.h>
#include <sys/stat.h>

void mount_proc(void) {
    mkdir("/proc", 0555);
    if (mount("proc", "/proc", "proc",
              MS_NOSUID | MS_NODEV | MS_NOEXEC, NULL) == -1) {
        perror("mount /proc");
    }
}

The MS_NOSUID | MS_NODEV | MS_NOEXEC flags prevent setuid binaries, device files, and executable files from being used on this mount - good practice even though /proc has none of these.

sysfs: /sys

sysfs (type sysfs) exports kernel object information. It represents the device tree, bus topology, kernel modules, and subsystem parameters. Important paths:

  • /sys/class/net/ - network interfaces
  • /sys/fs/cgroup/ - cgroup hierarchy (when cgroupfs is mounted here)
  • /sys/kernel/ - kernel debugging and security parameters
  • /sys/devices/ - device hierarchy

Containers typically mount /sys read-only (MS_RDONLY) to prevent modification of host kernel parameters: 

void mount_sysfs(void) {
    mkdir("/sys", 0555);
    if (mount("sysfs", "/sys", "sysfs",
              MS_RDONLY | MS_NOSUID | MS_NODEV | MS_NOEXEC, NULL) == -1) {
        perror("mount /sys");
    }
}

Note that /sys/fs/cgroup/ is a sub-mount within sysfs when cgroupfs is mounted there. Containers with a cgroup namespace get their own view of the cgroup tree; without one, they see the full host cgroup hierarchy (which leaks information about other containers).

devtmpfs: /dev

devtmpfs (type devtmpfs or tmpfs) provides device nodes. The kernel populates it automatically with entries for all registered devices when mounted as devtmpfs. Containers usually mount a plain tmpfs on /dev and then create only the device nodes they need: 

void mount_dev(void) {
    mkdir("/dev", 0755);

    /* Plain tmpfs - kernel won't auto-populate device nodes */
    if (mount("tmpfs", "/dev", "tmpfs",
              MS_NOSUID | MS_STRICTATIME,
              "mode=755,size=65536k") == -1) {
        perror("mount /dev");
        return;
    }

    /* Create standard devices */
    mknod("/dev/null",    S_IFCHR | 0666, makedev(1, 3));
    mknod("/dev/zero",    S_IFCHR | 0666, makedev(1, 5));
    mknod("/dev/full",    S_IFCHR | 0666, makedev(1, 7));
    mknod("/dev/random",  S_IFCHR | 0444, makedev(1, 8));
    mknod("/dev/urandom", S_IFCHR | 0444, makedev(1, 9));
    mknod("/dev/tty",     S_IFCHR | 0666, makedev(5, 0));
    mknod("/dev/pts",     S_IFDIR | 0755, 0);

    /* Mount devpts for terminal support */
    mkdir("/dev/pts", 0755);
    mount("devpts", "/dev/pts", "devpts",
          MS_NOSUID | MS_NOEXEC,
          "newinstance,ptmxmode=0666");

    /* /dev/ptmx should be a symlink to /dev/pts/ptmx */
    symlink("/dev/pts/ptmx", "/dev/ptmx");
}

The minimum set of devices for a working container is null, zero, random, urandom, and tty. Without /dev/null, processes that redirect to it will fail. Without /dev/urandom, Go's TLS stack and many other programs will fail to initialise.

devpts: /dev/pts

devpts provides pseudo-terminal (PTY) slave devices. When you SSH into a container or run docker exec -it, the terminal window is a PTY. The newinstance option is critical: without it all containers share a single PTY namespace and one container could open another's terminal: 

mount("devpts", "/dev/pts", "devpts",
      MS_NOSUID | MS_NOEXEC,
      "newinstance,ptmxmode=0666,mode=620,gid=5");

tmpfs: /dev/shm and /run

POSIX shared memory lives in /dev/shm (a tmpfs). Many IPC libraries, databases, and browsers use it. Mount it inside the container: 

mkdir("/dev/shm", 01777);
mount("shm", "/dev/shm", "tmpfs",
      MS_NOSUID | MS_NODEV, "mode=1777,size=65536k");

mkdir("/run", 0755);
mount("tmpfs", "/run", "tmpfs",
      MS_NOSUID | MS_NODEV | MS_STRICTATIME,
      "mode=755,size=65536k");

Complete Setup in Go

package main

import (
    "fmt"
    "os"

    "golang.org/x/sys/unix"
)

func setupMounts() error {
    mounts := []struct {
        src, dst, fstype string
        flags            uintptr
        opts             string
    }{
        {"proc",   "/proc", "proc",
         unix.MS_NOSUID | unix.MS_NODEV | unix.MS_NOEXEC, ""},
        {"sysfs",  "/sys",  "sysfs",
         unix.MS_RDONLY | unix.MS_NOSUID | unix.MS_NODEV | unix.MS_NOEXEC, ""},
        {"tmpfs",  "/dev",  "tmpfs",
         unix.MS_NOSUID | unix.MS_STRICTATIME, "mode=755,size=65536k"},
        {"devpts", "/dev/pts", "devpts",
         unix.MS_NOSUID | unix.MS_NOEXEC,
         "newinstance,ptmxmode=0666"},
        {"shm", "/dev/shm", "tmpfs",
         unix.MS_NOSUID | unix.MS_NODEV,
         "mode=1777,size=65536k"},
    }

    for _, m := range mounts {
        if err := os.MkdirAll(m.dst, 0755); err != nil {
            return fmt.Errorf("mkdir %s: %w", m.dst, err)
        }
        if err := unix.Mount(m.src, m.dst, m.fstype, m.flags, m.opts); err != nil {
            return fmt.Errorf("mount %s: %w", m.dst, err)
        }
    }

    // Create essential device nodes
    devs := []struct {
        path  string
        mode  uint32
        major uint32
        minor uint32
    }{
        {"/dev/null",    unix.S_IFCHR | 0666, 1, 3},
        {"/dev/zero",    unix.S_IFCHR | 0666, 1, 5},
        {"/dev/random",  unix.S_IFCHR | 0444, 1, 8},
        {"/dev/urandom", unix.S_IFCHR | 0444, 1, 9},
        {"/dev/tty",     unix.S_IFCHR | 0666, 5, 0},
    }

    for _, d := range devs {
        err := unix.Mknod(d.path, d.mode,
            int(unix.Mkdev(d.major, d.minor)))
        if err != nil && !os.IsExist(err) {
            return fmt.Errorf("mknod %s: %w", d.path, err)
        }
    }

    return nil
}

Masking Dangerous Paths

Even with a correct /proc mount, some paths expose sensitive information or allow escapes. runc binds a tmpfs (masking) over: 

/proc/kcore
/proc/keys
/proc/latency_stats
/proc/timer_list
/proc/timer_stats
/proc/sched_debug
/sys/firmware

These are bind-mounted with /dev/null or a small tmpfs to prevent access: 

mount("/dev/null", "/proc/kcore", NULL, MS_BIND, NULL);

Summary

A container namespace starts with no virtual filesystems. Mount a fresh proc on /proc to get a PID-namespace-correct process table. Mount sysfs read-only on /sys. Create /dev as a plain tmpfs, add device nodes explicitly, mount devpts with newinstance for isolated PTYs, and add /dev/shm. Mask sensitive /proc paths like /proc/kcore by bind-mounting /dev/null over them. These are the exact steps runc performs before executing the container workload.
