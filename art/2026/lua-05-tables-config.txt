Lua - Part 5: Tables as Configuration
2026-11-26


Every application has configuration - sample rates, buffer sizes, file paths, plugin
lists. The usual options are INI files (flat, no nesting, no types), JSON (typed but
verbose, no comments, requires a parser library), and YAML (complex parser, ambiguous
syntax, notorious edge cases). Lua tables give you something better: a configuration
format that is also a full programming language. Defaults can be computed.
Values can be validated. Lists and nested structures are first-class. And you already
have a Lua interpreter in your process, so the "parser" is zero extra code.
This article shows the complete C workflow for reading a Lua config file:
top-level values, nested tables, sequences, hash-table iteration, required fields, and
error messages that tell the user exactly what is missing and where.


What You Will Learn


How to read top-level Lua globals from C with lua_getglobal and lua_getfield.
How to read nested tables by chaining lua_getfield calls.
How to read sequence tables (arrays) with lua_rawgeti and luaL_len.
How to iterate over hash tables with lua_next.
How to validate required fields and supply defaults for optional ones.
A complete C config loader for an audio application.


Why lua_getfield Instead of lua_getglobal


lua_getglobal(L, "sample_rate") retrieves the global variable named
sample_rate and pushes its value onto the stack. That works for flat
configs but breaks down immediately when your config has structure:



-- config.lua
audio = {
    sample_rate = 48000,
    buffer_size = 512,
    plugins = { "reverb", "compressor", "limiter" },
    output = {
        device = "hw:0,0",
        channels = 2
    }
}



To read audio.output.channels, you navigate the table hierarchy in C using
lua_getfield. The key insight: lua_getfield(L, index, key)
reads key from the table at stack position index and pushes
the result. After each call, the new value is on top of the stack.



/* Step 1: get the top-level 'audio' table */
lua_getglobal(L, "audio");          /* stack: [audio_table]          */
if (!lua_istable(L, -1)) {
    fprintf(stderr, "config: 'audio' must be a table\n");
    lua_pop(L, 1);
    return -1;
}

/* Step 2: get audio.sample_rate */
lua_getfield(L, -1, "sample_rate"); /* stack: [audio_table, sr]      */
int sr = (int)luaL_optinteger(L, -1, 44100);
lua_pop(L, 1);                      /* stack: [audio_table]          */

/* Step 3: navigate into audio.output */
lua_getfield(L, -1, "output");      /* stack: [audio_table, output]  */
if (lua_istable(L, -1)) {
    lua_getfield(L, -1, "channels");/* stack: [audio_table, output, channels] */
    int ch = (int)luaL_optinteger(L, -1, 2);
    lua_pop(L, 1);                  /* stack: [audio_table, output]  */
}
lua_pop(L, 1);                      /* stack: [audio_table]          */

lua_pop(L, 1);                      /* stack: []  - done with audio  */



The pattern is always: push the value you want to inspect, inspect it, pop it before
moving on. Keep careful count of what is on the stack. A useful discipline: note the
stack depth at the entry of a function and assert it is the same (plus any intended
return values) at exit.


Reading Sequences with lua_rawgeti


A Lua sequence (a table used as an array, with consecutive integer keys starting at 1)
is read with lua_rawgeti(L, tableindex, i). This pushes the value at
integer key i without invoking metamethods - safe and fast for plain tables.



/* Read audio.plugins as a C string array */
lua_getglobal(L, "audio");           /* stack: [audio]          */
lua_getfield(L, -1, "plugins");      /* stack: [audio, plugins] */

if (lua_istable(L, -1)) {
    /* lua_rawlen returns the # operator result: length of the sequence */
    lua_Integer n = lua_rawlen(L, -1);

    for (lua_Integer i = 1; i <= n; i++) {
        lua_rawgeti(L, -1, i);       /* stack: [..., plugins, plugin_i] */
        const char *name = lua_tostring(L, -1);
        if (name) {
            printf("  plugin[%lld] = %s\n", (long long)i, name);
        }
        lua_pop(L, 1);               /* stack: [..., plugins]   */
    }
}

lua_pop(L, 1);  /* pop plugins */
lua_pop(L, 1);  /* pop audio   */



Use lua_rawgeti rather than lua_geti for sequence reading.
The raw version skips metamethod lookup, which makes it faster and avoids surprising
behaviour if someone attaches an __index metamethod to the table.
lua_rawlen similarly gives the raw length without metamethod invocation.


Iterating Hash Tables with lua_next


When you need to iterate over a table's key-value pairs without knowing the keys in
advance, use lua_next. It implements the Lua next() function:
given a table and a key, it finds the next key-value pair in iteration order and pushes
them both.



The loop structure is always the same: push nil as the initial key, then call
lua_next in a loop until it returns 0 (no more pairs). At the start of
each iteration, the key is at index -2 and the value at index -1. You must pop the
value before the next call to lua_next, but the key stays - it is consumed
by the next call.



/* Iterate over all key/value pairs in a table */
/* Assumes the table is on top of the stack    */
static void dump_table(lua_State *L, int table_idx) {
    lua_pushnil(L);   /* first key - nil means "start from the beginning" */
                      /* stack: [..., table, nil]                          */

    while (lua_next(L, table_idx) != 0) {
        /* stack: [..., table, key, value]                                 */

        /* lua_tostring on the key would modify it in place for numbers    */
        /* Use lua_type + specific getters to avoid that                   */
        const char *key = "(non-string)";
        if (lua_type(L, -2) == LUA_TSTRING)
            key = lua_tostring(L, -2);

        const char *val_type = lua_typename(L, lua_type(L, -1));
        printf("  %s = (%s)\n", key, val_type);

        lua_pop(L, 1);  /* pop value; key stays for next lua_next call    */
                        /* stack: [..., table, key]                        */
    }
    /* stack: [..., table] - lua_next popped the key on the final iteration */
}



Critical gotcha: do not call lua_tostring on the key while iterating.
lua_tostring may convert a number key to a string in place, and
lua_next uses the key's original type for its lookup. Converting the key
corrupts the iteration. If you need the key as a string, use lua_tostring
on a copy, or check the type first and only call it on confirmed string keys.


Validating Required Fields


A config loader that silently uses defaults for missing required fields creates subtle
bugs. When a required field is missing, the error message should name the field and its
location in the config structure:



/* require_integer: read an integer field, error if absent or wrong type */
static lua_Integer require_integer(lua_State *L, int table_idx,
                                   const char *field, const char *context) {
    lua_getfield(L, table_idx, field);      /* push the value */
    if (lua_isnil(L, -1)) {
        lua_pop(L, 1);
        luaL_error(L, "config: required field '%s.%s' is missing", context, field);
    }
    if (!lua_isinteger(L, -1) && !lua_isnumber(L, -1)) {
        lua_pop(L, 1);
        luaL_error(L, "config: '%s.%s' must be a number (got %s)",
                   context, field, lua_typename(L, lua_type(L, -1)));
    }
    lua_Integer val = lua_tointeger(L, -1);
    lua_pop(L, 1);
    return val;
}

/* optional_string: read a string field, return default if absent */
static const char *optional_string(lua_State *L, int table_idx,
                                   const char *field, const char *def) {
    lua_getfield(L, table_idx, field);
    if (lua_isnil(L, -1) || !lua_isstring(L, -1)) {
        lua_pop(L, 1);
        return def;
    }
    /* Copy - the pointer into Lua memory is only valid while the string is on the stack */
    const char *s = lua_tostring(L, -1);
    /* Caller must not store the raw pointer past the stack lifetime */
    /* For persistent storage: strdup(s) and manage ownership in C  */
    lua_pop(L, 1);
    return s;  /* valid until next Lua API call that may GC */
}


Complete Audio Config Loader


A complete C program that loads and validates an audio configuration from Lua:



/* config_loader.c */
#include <lua.h>
#include <lualib.h>
#include <lauxlib.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX_PLUGINS 32

typedef struct {
    int    sample_rate;
    int    buffer_size;
    char   device[64];
    int    channels;
    char   plugins[MAX_PLUGINS][64];
    int    plugin_count;
} AudioConfig;

static int load_audio_config(lua_State *L, AudioConfig *cfg) {
    /* Get top-level 'audio' table */
    lua_getglobal(L, "audio");               /* stack: [audio?]            */
    if (!lua_istable(L, -1)) {
        fprintf(stderr, "config: 'audio' table not found\n");
        lua_pop(L, 1);
        return -1;
    }
    int audio_idx = lua_gettop(L);           /* remember the table's index */

    /* Required: sample_rate */
    lua_getfield(L, audio_idx, "sample_rate");
    if (lua_isnil(L, -1)) {
        fprintf(stderr, "config: audio.sample_rate is required\n");
        lua_pop(L, 2);
        return -1;
    }
    cfg->sample_rate = (int)lua_tointeger(L, -1);
    lua_pop(L, 1);

    /* Required: buffer_size */
    lua_getfield(L, audio_idx, "buffer_size");
    if (lua_isnil(L, -1)) {
        fprintf(stderr, "config: audio.buffer_size is required\n");
        lua_pop(L, 2);
        return -1;
    }
    cfg->buffer_size = (int)lua_tointeger(L, -1);
    lua_pop(L, 1);

    /* Validate sensible values */
    if (cfg->sample_rate != 44100 && cfg->sample_rate != 48000 &&
        cfg->sample_rate != 96000) {
        fprintf(stderr, "config: audio.sample_rate %d is not supported\n",
                cfg->sample_rate);
        lua_pop(L, 1);
        return -1;
    }

    /* Nested: audio.output */
    lua_getfield(L, audio_idx, "output");    /* stack: [audio, output?]    */
    if (lua_istable(L, -1)) {
        int out_idx = lua_gettop(L);

        lua_getfield(L, out_idx, "device");
        const char *dev = lua_isstring(L, -1) ? lua_tostring(L, -1) : "default";
        strncpy(cfg->device, dev, sizeof(cfg->device) - 1);
        lua_pop(L, 1);

        lua_getfield(L, out_idx, "channels");
        cfg->channels = lua_isinteger(L, -1) ? (int)lua_tointeger(L, -1) : 2;
        lua_pop(L, 1);
    } else {
        /* Defaults if output table absent */
        strncpy(cfg->device, "default", sizeof(cfg->device) - 1);
        cfg->channels = 2;
    }
    lua_pop(L, 1);  /* pop output */

    /* Sequence: audio.plugins */
    cfg->plugin_count = 0;
    lua_getfield(L, audio_idx, "plugins");   /* stack: [audio, plugins?]   */
    if (lua_istable(L, -1)) {
        lua_Integer n = lua_rawlen(L, -1);
        int plugins_idx = lua_gettop(L);
        for (lua_Integer i = 1; i <= n && cfg->plugin_count < MAX_PLUGINS; i++) {
            lua_rawgeti(L, plugins_idx, i);  /* stack: [..., plugins, name] */
            if (lua_isstring(L, -1)) {
                const char *name = lua_tostring(L, -1);
                strncpy(cfg->plugins[cfg->plugin_count], name, 63);
                cfg->plugin_count++;
            }
            lua_pop(L, 1);                   /* stack: [..., plugins]      */
        }
    }
    lua_pop(L, 1);  /* pop plugins */

    lua_pop(L, 1);  /* pop audio table */
    return 0;
}

int main(int argc, char *argv[]) {
    const char *cfgfile = argc > 1 ? argv[1] : "audio.lua";

    lua_State *L = luaL_newstate();
    luaL_openlibs(L);

    if (luaL_loadfile(L, cfgfile) != LUA_OK) {
        fprintf(stderr, "load error: %s\n", lua_tostring(L, -1));
        lua_close(L);
        return 1;
    }
    if (lua_pcall(L, 0, 0, 0) != LUA_OK) {
        fprintf(stderr, "runtime error: %s\n", lua_tostring(L, -1));
        lua_close(L);
        return 1;
    }

    AudioConfig cfg = {0};
    if (load_audio_config(L, &cfg) != 0) {
        lua_close(L);
        return 1;
    }

    printf("sample_rate  = %d\n",   cfg.sample_rate);
    printf("buffer_size  = %d\n",   cfg.buffer_size);
    printf("device       = %s\n",   cfg.device);
    printf("channels     = %d\n",   cfg.channels);
    printf("plugins      = %d\n",   cfg.plugin_count);
    for (int i = 0; i < cfg.plugin_count; i++)
        printf("  [%d] %s\n", i, cfg.plugins[i]);

    lua_close(L);
    return 0;
}

Comparison with JSON and INI


INI files cannot represent nested structure without inventing a convention
([audio.output] as a section name is not standard). They have no types -
everything is a string. Adding a list requires another convention. Comments are not
part of the format in most implementations.



JSON is typed, supports nesting and arrays, and is language-neutral. It cannot have
comments (technically). It cannot compute values. A JSON parser must be linked. For a
configuration file in a Lua-embedding application, using JSON means shipping two
parsers - the Lua interpreter you already have, plus a JSON library.



Lua config tables have a genuine limitation: they are code. A malicious config file
can execute arbitrary code when loaded. For controlled environments (your own
deployments, development tools, game modding with a trusted modder base) this is
acceptable. For configs loaded from untrusted sources, see Part 9 (sandboxing):
you can load config files inside a restricted Lua state that has no access to
io, os, or require.


Summary


Lua tables are an excellent configuration format when you already embed Lua. Load the
file with luaL_loadfile + lua_pcall, navigate to the top-level
table with lua_getglobal, and descend into nested tables with chained
lua_getfield calls. Read sequences with lua_rawgeti inside a
lua_rawlen loop. Iterate unknown-key tables with lua_next,
remembering not to modify the key during iteration.



Write small helper functions for required and optional fields - the error messages they
produce ("config: required field 'audio.sample_rate' is missing") are what your users
will read when something goes wrong.



The next article scales this idea up: instead of reading passive values, you read a
modules table that describes a graph of application components, and the C
code dispatches based on a type field in each entry.
