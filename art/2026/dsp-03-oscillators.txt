DSP - Part 3: Oscillators: Basic Waveforms
2026-08-27


An oscillator is the heartbeat of any synthesiser. Before envelopes, filters, or effects
can do anything useful, you need a signal to process. This article implements the four
classic waveforms - sine, square, sawtooth, and triangle - in C using the
phase accumulator pattern, the simplest and most portable approach to digital
oscillator design.

The Phase Accumulator Pattern


Every periodic waveform can be defined as a function of a phase variable that sweeps
from 0 to 1 (or 0 to 2*PI, depending on your convention) and then wraps back to zero.
The phase accumulator pattern maintains a single floating-point variable phase
and increments it by a fixed amount each sample:



phase += freq / sample_rate;
if (phase >= 1.0f) phase -= 1.0f;



The increment freq / sample_rate is the number of full cycles per sample.
At 440 Hz and 48000 Hz sample rate the increment is 440/48000 ≈ 0.009167 cycles per
sample, so it takes about 109 samples to complete one full cycle - as expected for
440 Hz at 48 kHz.



This normalised-phase approach (phase in [0, 1)) is convenient because the waveform
formulas become simple expressions. We can always convert to radians with
2*PI*phase when we need to.


The Oscillator Struct


#include <math.h>
#include <stdint.h>

#define SAMPLE_RATE 48000
#define M_PI 3.14159265358979323846f

typedef enum {
    OSC_SINE,
    OSC_SQUARE,
    OSC_SAW,
    OSC_TRIANGLE
} OscType;

typedef struct {
    float    phase;       /* current phase, normalised [0, 1) */
    float    freq;        /* frequency in Hz */
    float    amplitude;   /* linear amplitude */
    OscType  type;
} Oscillator;

void osc_init(Oscillator *osc, float freq, float amplitude, OscType type) {
    osc->phase     = 0.0f;
    osc->freq      = freq;
    osc->amplitude = amplitude;
    osc->type      = type;
}


Sine Wave


The sine wave is the purest tone - a single frequency with no harmonics. It is the
reference waveform for testing and the building block for additive synthesis and FM.



float osc_sine(float phase) {
    return sinf(2.0f * M_PI * phase);
}



sinf is expensive compared to the other waveforms. If you need many
sine oscillators, consider a pre-computed lookup table (the wavetable article covers
this). For a modest number of voices, sinf is fine.


Square Wave


A square wave alternates between +1 and -1 at the oscillator frequency. It contains
all odd harmonics at amplitudes 1/n (fundamental, 3rd, 5th, ...). The naive
implementation is a simple comparison:



float osc_square(float phase) {
    return (phase < 0.5f) ? 1.0f : -1.0f;
}



The pulse width (the duty cycle of the +1 phase) defaults to 0.5, giving a symmetric
square wave. A pulse-width-modulated (PWM) square replaces the 0.5 threshold with a
variable pw in [0, 1):



float osc_pulse(float phase, float pw) {
    return (phase < pw) ? 1.0f : -1.0f;
}



Warning: this naive square wave aliases heavily. The PolyBLEP article shows how to
fix it. Use the naive version only for testing or at very low frequencies.


Sawtooth Wave


A sawtooth wave ramps linearly from -1 to +1 over each cycle, then jumps back to -1.
It contains all harmonics at amplitudes 1/n (fundamental, 2nd, 3rd, ...) - a rich,
bright timbre used in synthesis for strings, brass, and pads.



float osc_saw(float phase) {
    return 2.0f * phase - 1.0f;
}



The formula maps phase [0, 1) linearly to [-1, +1). Like the square wave it has a
hard discontinuity (the jump from +1 to -1 at phase wrap), which aliases.


Triangle Wave


A triangle wave ramps up in the first half-cycle and ramps down in the second. It
contains only odd harmonics at amplitudes 1/n2, which roll off much faster
than the square or saw - giving a softer, more muted sound closer to a flute or
clarinet.



float osc_triangle(float phase) {
    if (phase < 0.5f)
        return 4.0f * phase - 1.0f;
    else
        return 3.0f - 4.0f * phase;
}



You can also compute the triangle from the absolute value of a ramp:



float osc_triangle_v2(float phase) {
    return 1.0f - 4.0f * fabsf(phase - 0.5f);
}



The triangle wave has no discontinuities in the waveform itself (only in its
derivative), so aliasing is much less severe than with the saw or square.


The Render Loop


Here is a complete per-sample render function and a block render that fills a mono buffer:



float osc_tick(Oscillator *osc) {
    float sample;
    switch (osc->type) {
        case OSC_SINE:     sample = osc_sine(osc->phase);     break;
        case OSC_SQUARE:   sample = osc_square(osc->phase);   break;
        case OSC_SAW:      sample = osc_saw(osc->phase);      break;
        case OSC_TRIANGLE: sample = osc_triangle(osc->phase); break;
        default:           sample = 0.0f;                     break;
    }

    /* Advance phase */
    osc->phase += osc->freq / (float)SAMPLE_RATE;
    if (osc->phase >= 1.0f) osc->phase -= 1.0f;

    return sample * osc->amplitude;
}

void osc_render(Oscillator *osc, float *buf, int n_samples) {
    for (int i = 0; i < n_samples; i++)
        buf[i] = osc_tick(osc);
}


Mixing Multiple Oscillators


Summing oscillators is trivial - add the samples and scale down to avoid clipping:



#define NUM_OSCS 4

void mix_oscillators(Oscillator oscs[], int n_oscs,
                     float *out, int n_samples) {
    float scale = 1.0f / (float)n_oscs;
    for (int i = 0; i < n_samples; i++) {
        float sum = 0.0f;
        for (int j = 0; j < n_oscs; j++)
            sum += osc_tick(&oscs[j]);
        out[i] = sum * scale;
    }
}



The division by n_oscs prevents summed oscillators from exceeding ±1.
In a real synthesiser you would use individual amplitude envelopes and a master level
control, but normalising by count is fine for prototyping.


Frequency Modulation of Phase


You can modulate one oscillator's frequency (or more precisely, its phase increment)
with another oscillator to get vibrato or more complex FM effects:



void osc_render_with_vibrato(Oscillator *carrier,
                             Oscillator *lfo,
                             float vibrato_depth,
                             float *out, int n_samples) {
    for (int i = 0; i < n_samples; i++) {
        float lfo_val  = osc_tick(lfo);
        float mod_freq = carrier->freq + lfo_val * vibrato_depth;
        carrier->phase += mod_freq / (float)SAMPLE_RATE;
        if (carrier->phase >= 1.0f) carrier->phase -= 1.0f;
        if (carrier->phase <  0.0f) carrier->phase += 1.0f;
        float sample = osc_sine(carrier->phase);
        out[i] = sample * carrier->amplitude;
    }
}


Putting It All Together


#include <stdio.h>
#include <math.h>

int main(void) {
    Oscillator osc;
    osc_init(&osc, 440.0f, 0.5f, OSC_SAW);

    float buf[SAMPLE_RATE];  /* 1 second of audio */
    osc_render(&osc, buf, SAMPLE_RATE);

    /* Find peak amplitude */
    float peak = 0.0f;
    for (int i = 0; i < SAMPLE_RATE; i++) {
        float a = fabsf(buf[i]);
        if (a > peak) peak = a;
    }
    printf("Peak: %.4f (%.2f dBFS)\n",
           peak, 20.0f * log10f(peak));
    return 0;
}


Notes on Numerical Stability


After many millions of samples the phase accumulator can accumulate floating-point
error. With float this becomes noticeable only after many hours of
continuous operation. The subtraction-based wrap (if (phase >= 1.0f) phase -= 1.0f)
is more stable than taking fmodf(phase, 1.0f) because fmodf is slightly
more expensive and introduces rounding differently. Both are correct.



If you need double-precision phase (e.g., very low-frequency LFOs running for hours),
use double for phase and phase_inc but convert to
float at the output.


Summary


The phase accumulator pattern adds a per-sample phase increment to a running
phase variable, wrapping at 2π (or 1.0 for normalised phase). Sine uses
sinf(phase); square uses a sign comparison to midpoint; sawtooth is the
raw normalised phase rescaled to [−1, 1]; triangle uses the absolute value of
the sawtooth. All four waveforms alias at high frequencies — the next article
covers PolyBLEP correction for band-limited output.
