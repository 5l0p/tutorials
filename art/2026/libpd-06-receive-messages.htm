libpd - Part 6: Receiving Messages from Pd

The inbound side of the libpd communication channel lets your C code react to events generated by the Pd patch - a note-on event, a detected beat, a DSP overload warning, or any other control signal the patch emits. libpd uses a hook-based callback system: you register a C function pointer, and libpd calls it whenever the corresponding message type arrives. This article explains how each hook works and covers the threading implications that catch most developers off guard.

What You Will Learn

  • How [send] objects in the patch emit named messages that C hooks can receive
  • How to subscribe to a named channel with libpd_bind and unsubscribe with libpd_unbind
  • Registering the five hook types: bang, float, symbol, list, and message
  • The print hook: capturing Pd's console output from C
  • Threading implications: hooks fire inside libpd_process_float on the audio thread

How Message Receiving Works

Inside a Pd patch, [send receiverName] (or [s receiverName]) objects emit messages onto a named channel. On the libpd side, you first subscribe to a channel name, then libpd calls your hook when a message arrives on that channel.

Subscription:

#include <z_libpd.h>

/* Subscribe to messages sent to "note_out" in the patch */
libpd_bind("note_out");

Without calling libpd_bind, messages sent to "note_out" are silently ignored. You can subscribe to multiple channels; call libpd_bind once per channel name before the audio engine starts.

The Bang Hook

static void on_bang(const char *recv) {
    printf("Bang received on: %s\n", recv);
    /* e.g., "beat_tick" fires every beat from the patch */
}

libpd_set_banghook(on_bang);
libpd_bind("beat_tick");

The Float Hook

static void on_float(const char *recv, float x) {
    if (strcmp(recv, "rms_level") == 0) {
        /* Update a level meter in the UI */
        ui_set_level_meter(x);
    }
}

libpd_set_floathook(on_float);
libpd_bind("rms_level");

The Symbol Hook

static void on_symbol(const char *recv, const char *sym) {
    printf("Symbol '%s' on channel '%s'\n", sym, recv);
}

libpd_set_symbolhook(on_symbol);
libpd_bind("patch_event");

The List Hook

Pd lists carry a mix of floats and symbols. libpd passes them through the t_atom array:

#include <z_libpd.h>
#include <m_pd.h>    /* for t_atom, GETFLOAT, GETSYMBOL */

static void on_list(const char *recv,
                    int argc,
                    t_atom *argv)
{
    for (int i = 0; i < argc; i++) {
        if (argv[i].a_type == A_FLOAT) {
            printf("  [%d] float: %f\n", i, argv[i].a_w.w_float);
        } else if (argv[i].a_type == A_SYMBOL) {
            printf("  [%d] sym: %s\n",  i, argv[i].a_w.w_symbol->s_name);
        }
    }
}

libpd_set_listhook(on_list);
libpd_bind("chord_data");

The Message Hook

A Pd message differs from a list in that it has a selector string. The message hook receives the selector as a separate argument:

static void on_message(const char *recv,
                       const char *msg,
                       int argc,
                       t_atom *argv)
{
    printf("Message '%s' on '%s' with %d args\n", msg, recv, argc);
    /* e.g., recv="synth_out", msg="note", argv=[60, 100] */
}

libpd_set_messagehook(on_message);
libpd_bind("synth_out");

The Print Hook

Pd's [print] object and built-in error messages write to stdout by default. Redirect them to your own logging system with the print hook:

static void on_print(const char *msg) {
    /* msg already includes a trailing newline for errors,
       but regular print output may not */
    fprintf(stderr, "[pd] %s", msg);
}

/* Set BEFORE libpd_init - catches init-time messages too */
libpd_set_printhook(on_print);
libpd_init();

Critical Threading Warning

All hooks are called on the audio thread - specifically, they fire during libpd_process_float, which runs inside your audio callback. This means:

  • Do NOT call any blocking function from a hook (no mutex lock that might block, no printf in production code, no heap allocation).
  • Do NOT touch UI or OpenGL from a hook directly.
  • DO use a lock-free ring buffer to pass data from the hook to the main thread.

A safe pattern using a simple ring buffer:

typedef struct {
    char  recv[32];
    float value;
} FloatMsg;

#define RING_SIZE 256
static FloatMsg g_ring[RING_SIZE];
static volatile int g_ring_w = 0;
static volatile int g_ring_r = 0;

/* Called on the audio thread */
static void on_float(const char *recv, float x) {
    int next = (g_ring_w + 1) & (RING_SIZE - 1);
    if (next != g_ring_r) { /* ring not full */
        strncpy(g_ring[g_ring_w].recv, recv, 31);
        g_ring[g_ring_w].value = x;
        g_ring_w = next;   /* atomic on most platforms */
    }
}

/* Called on the main/UI thread, e.g. each frame */
void process_pd_events(void) {
    while (g_ring_r != g_ring_w) {
        FloatMsg *m = &g_ring[g_ring_r];
        ui_update_param(m->recv, m->value);
        g_ring_r = (g_ring_r + 1) & (RING_SIZE - 1);
    }
}

Unbinding

To stop receiving messages on a channel:

void *binding = libpd_bind("rms_level");
/* ... later ... */
libpd_unbind(binding);

Note that libpd_bind returns a pointer you need to keep if you plan to unbind later. Losing this pointer causes a memory leak - you can no longer release the binding.

Complete Example

#include <z_libpd.h>
#include <m_pd.h>
#include <stdio.h>
#include <string.h>

static void on_bang(const char *r) {
    /* Audio thread: enqueue for main thread */
    printf("Bang: %s\n", r); /* OK in demos, not production */
}

static void on_float(const char *r, float x) {
    printf("Float on %s: %f\n", r, x);
}

static void on_print(const char *msg) {
    fprintf(stderr, "[pd] %s", msg);
}

int main(void) {
    libpd_set_printhook(on_print);
    libpd_init();
    libpd_init_audio(0, 2, 44100);

    libpd_set_banghook(on_bang);
    libpd_set_floathook(on_float);

    libpd_bind("beat");
    libpd_bind("rms");

    libpd_start_message(1);
    libpd_add_float(1.0f);
    libpd_finish_message("pd", "dsp");

    void *patch = libpd_openfile("feedback.pd", ".");

    float out[512 * 2];
    for (int i = 0; i < 440; i++) { /* ~5 seconds at 512/44100 */
        libpd_process_float(8, NULL, out);
        /* hooks may fire inside libpd_process_float */
    }

    libpd_closefile(patch);
    return 0;
}

Summary

Subscribe to a named channel with libpd_bind("name"), then register hook functions with libpd_set_floathook, libpd_set_banghook, and so on. libpd calls the hook synchronously from inside libpd_process_float on the audio thread, so hooks must be lock-free and non-blocking. To pass values to the main thread, write into a lock-free ring buffer inside the hook and drain it from the main thread between audio callbacks. Unsubscribe with libpd_unbind before closing the patch.
