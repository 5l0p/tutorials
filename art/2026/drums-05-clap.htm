Drums - Part 5: Clap Synthesis in Pure Data

A hand clap is the sound of several hands coming together slightly out of phase — not a single impulse but a cluster of transients separated by a few milliseconds, each with a slightly different spectrum. Synthesising this convincingly requires a roll mechanism (the pre-attack burst of clicks), multiple noise sources for spectral variety, and separate envelope paths for the initial impact and the reverb tail.

What You Will Learn

  • Three parallel noise sources: white, SID-style, and pink noise approximation
  • The roll: a [metro]-driven burst of clicks before the main body
  • MAIN and VERB envelope paths: separate shapes for impact body and reverb feed
  • Low and high shelf EQ on the combined signal
  • Reverb using [rev3~] or a Freeverb external
  • The extend-decay 3-segment envelope for all decay stages
  • X/Y mapping: brightness vs tail length

Three Noise Sources

Using three spectrally different noise sources and blending them gives a more convincing clap than white noise alone:

White noise: The dominant source in most patches. Provides a broad-spectrum, crisp attack character.

[noise~]
|
[bp~ $wFilterCut $wFilterQ]   ; bandpass shapes the spectral peak
|
[*~ $wNoiseMix]

SID noise: Hard-clipped white noise for a gritty, lo-fi digital texture. Adds presence in the 1–3 kHz range.

[noise~]
|
[clip~ -0.1 0.1]
|
[*~ 10]
|
[bp~ $sFilterCut $sFilterQ]
|
[*~ $sNoiseMix]

Pink noise approximation: Cascade several lowpass filters at descending cutoffs to roll off high-frequency energy and emphasise the low-mid, giving a softer, more natural character:

[noise~]
|
[lop~ 4000]
|
[lop~ 1500]
|
[lop~ 500]
|
(sum the three intermediate outputs with appropriate scaling)
|
[bp~ $pFilterCut $pFilterQ]
|
[*~ $pNoiseMix]

Mix all three noise sources into a single noise bus: 

noise bus = [+~ white, SID, pink noise paths]

Sample Rate Reduction and Distortion

Apply sample rate reduction and distortion to the combined noise bus before the envelope stages: 

noise bus
    |
[lop~ $noiseRate]    ; lowpass acts as rate approximation (or use samphold~)
    |
[clip~ -$distort $distort]   ; soft saturation
    |
processed noise bus

The $distort parameter (0–100) provides the drive level. Internally, the clip threshold is 1 / (1 + distort * 0.05) — so distort=0 is no clipping, distort=100 is hard clipping at 0.17.

The Roll

A real clap has 2–16 transient bursts (the individual hands) spread over 10–60 ms before the main body. Model this as a metro-gated noise burst: 

; Roll mechanism:
[r $0-trigger]
|
[bang]
|
[t b b]
|     |
|   [metro $rollInterval]          ; interval = RollTime / Rolls (ms)
|   |
|   [counter 0 $rolls]             ; count up to Rolls, then stop
|   |
|   [bang when counter == Rolls]
|   → [stop metro]
|
[bang]                             ; main hit always fires immediately too
→ fire MAIN + VERB envelopes

In Pd, implement the roll as: 

[r $0-trigger]
|
[bang]
|
[t b b]
|         |
|     [s $0-roll-start]       ; start the roll metro
|
[delay $rollTime]             ; main body fires after roll completes
|
[bang]
→ fire main body envelopes

; Roll subpatch:
[r $0-roll-start]
|
[bang]
|
[f $rolls]     ; load roll count
|
[metro $rollInterval]
|
[f] [- 1]      ; decrement counter
|
[sel 0]        ; stop when count reaches 0
|
[metro stop]
; Each metro tick: bang → gates noise bus briefly through a very fast vline~
[bang]
→ [vline~]:  1 0, 0 $rollAtk     ; fast attack/decay per click

Each roll click is a fast noise burst (2–5 ms). The amplitude of successive clicks can grow from a floor level to the main hit level — simulating individual hands approaching with increasing force.

MAIN and VERB Envelope Paths

The processed noise bus is split into two parallel paths, each with its own extend-decay envelope: 

processed noise bus
    │
    ├── MAIN path ──────────────────────────────────
    │   [vline~]   ; MainAtk → 1, decay to MainExtLvl, tail to 0
    │   [*~]
    │   [*~ $mainBodyOutLevel]
    │
    └── VERB path ──────────────────────────────────
        [vline~]   ; VerbAtk → 1, decay to VerbExtLvl, tail to 0
        [*~]
        [*~ $verbBodyOutLevel]

The MAIN path typically has a shorter, punchier decay (10–100 ms) that provides the initial impact character. The VERB path has a slightly longer, more diffuse decay (20–400 ms) that feeds the reverb unit.

The split is not a strict separation — both paths go into the same final chain and are summed before the EQ and reverb. The VERB path's longer tail gives the reverb input a richer signal to work with.

Shelf EQ

After summing MAIN + VERB, apply low and high shelf EQ for tonal shaping: 

; Low shelf — boost or cut below $loShelfCut:
; In vanilla Pd, approximate with a lop~ and gain correction:
[lop~ $loShelfCut]        ; extract lows
|
[*~ $loShelfGain]         ; boost (>1) or cut (<1)
; Add back to signal with appropriate mixing

; High shelf — boost or cut above $hiShelfCut:
[hip~ $hiShelfCut]
|
[*~ $hiShelfGain]

True shelf EQ requires a biquad filter. The lop/hip approximation works for broad shaping. For a more accurate shelf, use the biquad coefficients from the filter cookbook (Part 6 of the DSP series): 

; Biquad low shelf (from Audio EQ Cookbook, Robert Bristow-Johnson):
; Set A = sqrt(10^(dBgain/20)), S = slope, w0 = 2*pi*fc/Fs
; Compute b0,b1,b2,a0,a1,a2 and plug into [biquad~]
[biquad~ b1 b2 a1 a2]   ; b0/a0 normalised, format: [biquad~ b1 b2 a1 a2 b0]

Reverb

Use [rev3~] (built into Pd) for a quick approximation, or install the freeverb~ external for higher quality: 

; With rev3~:
combined EQ output
    |
[rev3~ 10 60 0.85]     ; roomsize, revtime, damping
    |                 |
 [dry out]         [wet out]
    |                 |
[*~ $dryLevel]    [*~ $wetLevel]
    │                 │
    [+~]
    |
[*~ outEnv]

[rev3~] takes four arguments: room size (ms), reverb time (ms), damping (0–1), and is stereo out. The first outlet is wet-only output.

If freeverb~ is available: 

[freeverb~]
; Left inlet: audio
; Right inlet: room size (0–1), damping (0–1) as messages
; Left outlet: left channel
; Right outlet: right channel

Output Envelope and Final Chain

[+~ dry, wet reverb]
    |
[*~ $outEnv vline~]    ; final output envelope shapes the whole sound
    |
[*~ $velocity]
    |
[outlet~]

The output envelope's OutDecayTime (50–600 ms) shapes the overall perceived length of the clap. Combined with the longer decay of the reverb, this produces the characteristic "splaaat" of a long reverbed clap.

local function loadPreset(name)
    local p = presets[name]
    for k, v in pairs(p) do pd.send("clap-" .. k, v) end
end

Build Order

  1. Single white [noise~] through a bandpass, with a [vline~] 150 ms decay → verify the basic clap hit.
  2. Add the roll: 4 clicks over 15 ms using [metro].
  3. Add the VERB envelope path in parallel.
  4. Add [rev3~] with dry/wet mix.
  5. Add SID noise source.
  6. Add pink noise approximation.
  7. Add shelf EQ.
  8. Add the output envelope.
  9. Wire all $0- receivers.
  10. Load presets and verify character matches the descriptions.

Summary

The clap synthesises the multi-hand cluster effect through a metro-driven roll (N quick noise bursts before the main hit), three spectrally different noise sources (white, SID, pink), MAIN and VERB envelope paths that layer body and tail, shelf EQ, and reverb. X maps to filter cutoff (dark→bright); Y maps to decay time across all envelope stages. The roll count and time sit on a secondary mapping layer — 0 rolls for a single snappy hit, 16 rolls for a full hand-clap fan. The preset system loads all 20+ parameters in one Lua table iteration.
