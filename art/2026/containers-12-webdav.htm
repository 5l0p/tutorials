Containers - Part 12: WebDAV: HTTP as a Filesystem

WebDAV (Web Distributed Authoring and Versioning, RFC 4918) extends HTTP with methods for creating, moving, copying, and locking files. From a client's perspective, a WebDAV share looks like a network filesystem. From a server's perspective, it is an HTTP server that handles a few extra methods. This article implements a WebDAV file server in Go using golang.org/x/net/webdav, and shows how to mount it on Linux with davfs2 and on macOS directly from Finder.

What You Will Learn

  • The WebDAV HTTP methods: PROPFIND, MKCOL, COPY, MOVE, LOCK, UNLOCK, and how they extend standard HTTP
  • How to implement a WebDAV server in Go using golang.org/x/net/webdav
  • How to test it with curl for PROPFIND, PUT, MKCOL, MOVE, and DELETE
  • How to mount the server on Linux with davfs2 and on macOS via Finder
  • How to add HTTP Basic Authentication with constant-time comparison
  • How to implement the webdav.FileSystem interface for a custom storage backend

WebDAV HTTP Methods

WebDAV adds the following methods to standard HTTP:

  • PROPFIND - retrieve properties of a resource (name, size, content type, last modified, whether it is a collection). The response is XML (207 Multi-Status).
  • PROPPATCH - set or remove properties on a resource.
  • MKCOL - create a collection (directory). Returns 201 Created.
  • COPY - copy a resource to a new URL. Uses Destination: header.
  • MOVE - move a resource. Uses Destination: header.
  • LOCK - acquire an exclusive or shared lock on a resource. Returns a lock token.
  • UNLOCK - release a lock.
  • GET - unchanged from HTTP, returns file content.
  • PUT - upload a file to the given URL.
  • DELETE - delete a file or collection.

The Depth: header controls whether PROPFIND recurses into sub-collections (Depth: infinity) or only looks at the immediate resource (Depth: 0).

Go WebDAV Server

golang.org/x/net/webdav provides a complete WebDAV handler that serves an http.FileSystem: 

package main

import (
    "flag"
    "log"
    "net/http"
    "os"

    "golang.org/x/net/webdav"
)

func main() {
    root := flag.String("root", ".", "directory to serve")
    addr := flag.String("addr", ":8080", "listen address")
    flag.Parse()

    if _, err := os.Stat(*root); err != nil {
        log.Fatalf("root directory: %v", err)
    }

    handler := &webdav.Handler{
        Prefix:     "/",
        FileSystem: webdav.Dir(*root),
        LockSystem: webdav.NewMemLS(),
        Logger: func(r *http.Request, err error) {
            if err != nil {
                log.Printf("%s %s: %v", r.Method, r.URL.Path, err)
            }
        },
    }

    log.Printf("WebDAV server on %s, serving %s", *addr, *root)
    log.Fatal(http.ListenAndServe(*addr, handler))
}

Run with go run server.go -root /tmp/shared -addr :8080. The server is immediately usable from any WebDAV client.

Testing with curl

# List root (PROPFIND with Depth: 1)
curl -s -X PROPFIND http://localhost:8080/ \
     -H "Depth: 1" \
     -H "Content-Type: application/xml" \
     --data '<?xml version="1.0"?><propfind xmlns="DAV:"><allprop/></propfind>' \
     | xmllint --format -

# Upload a file (PUT)
curl -T /etc/hostname http://localhost:8080/hostname.txt

# Download a file (GET)
curl http://localhost:8080/hostname.txt

# Create a directory (MKCOL)
curl -X MKCOL http://localhost:8080/newdir/

# Move a file (MOVE)
curl -X MOVE http://localhost:8080/hostname.txt \
     -H "Destination: http://localhost:8080/newdir/hostname.txt"

# Delete
curl -X DELETE http://localhost:8080/newdir/hostname.txt

PROPFIND Response Format

A PROPFIND response is a 207 Multi-Status XML document. Each resource is a <response> element with a <href> and a <propstat> containing the requested properties: 

<?xml version="1.0" encoding="UTF-8"?>
<multistatus xmlns="DAV:">
  <response>
    <href>/</href>
    <propstat>
      <prop>
        <resourcetype><collection/></resourcetype>
        <getlastmodified>Tue, 17 Mar 2026 12:00:00 GMT</getlastmodified>
        <getcontentlength>4096</getcontentlength>
      </prop>
      <status>HTTP/1.1 200 OK</status>
    </propstat>
  </response>
</multistatus>

Mounting on Linux with davfs2

# Install davfs2
sudo apt-get install davfs2

# Mount (run as root, or add user to davfs2 group)
sudo mount -t davfs http://localhost:8080/ /mnt/webdav

# Or via /etc/fstab
# http://localhost:8080/  /mnt/webdav  davfs  _netdev,uid=1000,gid=1000  0 0

# Use normally
ls /mnt/webdav/
cp /etc/hosts /mnt/webdav/

# Unmount
sudo umount /mnt/webdav

davfs2 has a credentials file at ~/.davfs2/secrets or /etc/davfs2/secrets: 

http://localhost:8080/  username  password

Adding Authentication

Wrap the WebDAV handler with HTTP Basic Authentication: 

package main

import (
    "crypto/subtle"
    "flag"
    "log"
    "net/http"
    "os"

    "golang.org/x/net/webdav"
)

func basicAuth(next http.Handler, user, pass string) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        u, p, ok := r.BasicAuth()
        if !ok || subtle.ConstantTimeCompare([]byte(u), []byte(user)) != 1 ||
            subtle.ConstantTimeCompare([]byte(p), []byte(pass)) != 1 {
            w.Header().Set("WWW-Authenticate", `Basic realm="webdav"`)
            http.Error(w, "Unauthorized", http.StatusUnauthorized)
            return
        }
        next.ServeHTTP(w, r)
    })
}

func main() {
    root := flag.String("root", ".", "directory to serve")
    addr := flag.String("addr", ":8080", "listen address")
    user := flag.String("user", "admin", "username")
    pass := flag.String("pass", "secret", "password")
    flag.Parse()

    if err := os.MkdirAll(*root, 0755); err != nil {
        log.Fatal(err)
    }

    dav := &webdav.Handler{
        Prefix:     "/",
        FileSystem: webdav.Dir(*root),
        LockSystem: webdav.NewMemLS(),
    }

    log.Printf("serving %s on %s (user=%s)", *root, *addr, *user)
    log.Fatal(http.ListenAndServe(*addr,
        basicAuth(dav, *user, *pass)))
}

Mounting on macOS

macOS has a built-in WebDAV client:

  • Finder: Go → Connect to Server (Cmd-K), enter http://your-server:8080/, click Connect.
  • Command line: open dav://your-server:8080/
  • CLI mount: mount -t webdav http://your-server:8080/ /Volumes/dav (requires the WebDAV kernel extension loaded)

Custom FileSystem Backend

The webdav.FileSystem interface lets you implement any storage backend: 

type FileSystem interface {
    Mkdir(ctx context.Context, name string, perm os.FileMode) error
    OpenFile(ctx context.Context, name string,
             flag int, perm os.FileMode) (File, error)
    RemoveAll(ctx context.Context, name string) error
    Rename(ctx context.Context, oldName, newName string) error
    Stat(ctx context.Context, name string) (os.FileInfo, error)
}

Implement this interface to back a WebDAV share with an S3 bucket, a database, or an in-memory store. The File interface mirrors os.File.

WebDAV in Container Context

WebDAV is less common for direct container volume sharing than 9P or NFS, but it is useful when you need to expose data to clients that only have HTTP access - CI runners, web browsers, or legacy systems. Some container orchestration platforms expose artifact stores and config maps over WebDAV internally.

Summary

WebDAV adds PROPFIND, MKCOL, COPY, MOVE, LOCK, and UNLOCK to HTTP, giving it filesystem semantics accessible from any HTTP client or OS WebDAV mount. golang.org/x/net/webdav provides a drop-in http.Handler backed by any webdav.FileSystem implementation — the default serves a real directory. On Linux, davfs2 mounts it as a kernel filesystem. On macOS, Finder connects natively. For containers, WebDAV is most useful when HTTP is the only available transport — CI runners, artifact stores, or legacy clients.
