DSP - Part 12: Spectral Processing with FFT

The FFT from Part 11 converts a block of samples into a frequency-domain representation. That representation is far more than an analysis tool - it is an editing surface. You can zero out frequency bins to suppress noise, freeze a spectral frame to sustain a chord indefinitely, or convolve a signal with a long impulse response in O(N log N) time instead of O(N2). This article covers the machinery that makes per-block spectral processing work correctly in a continuous audio stream, then builds four practical effects on top of it.

The Block Processing Problem

An FFT operates on a finite block of N samples. If you slice the incoming audio into non-overlapping blocks, process each in the frequency domain, and IFFT back, you will hear clicks at every block boundary. Two things cause this:

  1. Windowing energy loss. Applying a Hann window to each block tapers the edges to zero. Any audio near the edges of a block is attenuated. Without overlap, that energy is simply lost.
  2. Discontinuity artefacts. The IFFT implicitly assumes the block repeats periodically. Hard edges between blocks introduce energy at all frequencies.

The solution is overlap-add (OLA): advance the input pointer by only H = N/2 (or N/4) samples between FFTs, apply a Hann window, process, IFFT, and add the output into an accumulation buffer that is offset by H from the previous block. The overlap between successive windowed blocks means the tapered edges of one block fill in the gaps left by the previous block. With a Hann window and 50% overlap (H = N/2), the sum of overlapping windows is constant - no amplitude modulation, no clicks. 

/* Proof that Hann with 50% overlap sums to a constant.
   w[n] = 0.5*(1 - cos(2*pi*n/(N-1)))
   w[n] + w[n + N/2]  (for the case N is large, periodic approximation)
         ≈ 0.5*(1-cos(2*pi*n/N)) + 0.5*(1-cos(2*pi*(n+N/2)/N))
         = 0.5*(1-cos(θ)) + 0.5*(1+cos(θ))   [since cos(θ+π)=-cos(θ)]
         = 1.0
*/

Overlap-Add Engine in C

The engine keeps two buffers: an input overlap buffer for the analysis half (accumulating incoming samples) and an output overlap buffer for the synthesis half (accumulating IFFT results). Each call to ola_process_block consumes H new input samples and produces H output samples. 

#include <math.h>
#include <string.h>
#include <stdlib.h>

#define FFT_SIZE  1024
#define HOP       (FFT_SIZE / 2)   /* 50% overlap */

typedef struct { float re, im; } Complex;

/* Forward declaration - use the FFT from Part 11 */
void fft(Complex *buf, int n, int direction);
void ifft(Complex *buf, int n);

typedef struct {
    float   in_buf[FFT_SIZE];   /* circular input accumulator */
    float   out_buf[FFT_SIZE];  /* overlap-add output accumulator */
    int     in_pos;             /* write position in in_buf */
    float   window[FFT_SIZE];   /* pre-computed Hann window */
    Complex fft_buf[FFT_SIZE];  /* working FFT buffer */
} OLAEngine;

/* Callback type: process one FFT frame in the frequency domain.
   bins[] has FFT_SIZE complex bins, modify in place. */
typedef void (*SpectralFn)(Complex *bins, int n, void *user);

void ola_init(OLAEngine *e) {
    memset(e->in_buf,  0, sizeof(e->in_buf));
    memset(e->out_buf, 0, sizeof(e->out_buf));
    e->in_pos = 0;
    /* Pre-compute Hann window */
    for (int i = 0; i < FFT_SIZE; i++)
        e->window[i] = 0.5f * (1.0f - cosf(2.0f * (float)M_PI * i / (FFT_SIZE - 1)));
}

/* Process 'hop' new samples, produce 'hop' output samples.
   Returns 1 when a full FFT frame has been processed this call. */
int ola_push(OLAEngine *e, const float *in, float *out,
             SpectralFn fn, void *user) {
    /* Write HOP new samples into circular input buffer */
    for (int i = 0; i < HOP; i++) {
        e->in_buf[(e->in_pos + i) % FFT_SIZE] = in[i];
    }

    /* Build windowed analysis frame: newest FFT_SIZE samples */
    for (int i = 0; i < FFT_SIZE; i++) {
        int idx = (e->in_pos - FFT_SIZE + i + FFT_SIZE) % FFT_SIZE;
        e->fft_buf[i].re = e->in_buf[idx] * e->window[i];
        e->fft_buf[i].im = 0.0f;
    }

    /* Forward FFT */
    fft(e->fft_buf, FFT_SIZE, -1);

    /* User-supplied spectral processing */
    if (fn) fn(e->fft_buf, FFT_SIZE, user);

    /* Inverse FFT */
    ifft(e->fft_buf, FFT_SIZE);

    /* Overlap-add: accumulate windowed IFFT output into out_buf */
    for (int i = 0; i < FFT_SIZE; i++) {
        int idx = (e->in_pos + i) % FFT_SIZE;
        e->out_buf[idx] += e->fft_buf[i].re * e->window[i];
    }

    /* Read HOP samples from out_buf, then zero them */
    for (int i = 0; i < HOP; i++) {
        int idx = (e->in_pos + i) % FFT_SIZE;
        out[i] = e->out_buf[idx];
        e->out_buf[idx] = 0.0f;
    }

    e->in_pos = (e->in_pos + HOP) % FFT_SIZE;
    return 1;
}

The OLA engine introduces a latency of FFT_SIZE samples (one full window). At 48 kHz with FFT_SIZE=1024 that is about 21 ms - tolerable for effects but too high for real-time instrument monitoring. Smaller FFT sizes reduce latency at the cost of frequency resolution.

Spectral Gate

A spectral gate zeroes any FFT bin whose magnitude falls below a threshold. This is the simplest spectral effect and a surprisingly effective noise suppressor: background hiss lives in many small bins, while music concentrates energy in far fewer strong bins. 

typedef struct {
    float threshold;  /* linear magnitude threshold */
} SpectralGateParams;

void spectral_gate(Complex *bins, int n, void *user) {
    SpectralGateParams *p = (SpectralGateParams *)user;
    for (int k = 0; k < n; k++) {
        float mag = sqrtf(bins[k].re * bins[k].re + bins[k].im * bins[k].im);
        if (mag < p->threshold) {
            bins[k].re = 0.0f;
            bins[k].im = 0.0f;
        }
    }
}

Common mistake: setting the threshold in dBFS and forgetting to convert to linear before comparing against the FFT bin magnitude. The FFT magnitudes are not normalised to 1.0 - they scale with N and the window gain. For a Hann window and FFT_SIZE=1024, a full-scale sine wave produces a peak bin magnitude of roughly N/2 = 512. Calibrate your threshold by running a known tone through and printing the bin magnitudes.

Gotcha: zeroing bins abruptly produces musical noise - a metallic buzzing caused by phase discontinuities between frames. A soft gate (fade the gain down over several frames rather than hard-clipping to zero) sounds much cleaner.

Spectral Freeze

Spectral freeze captures one FFT frame and replaces every subsequent frame with it. The IFFT of a frozen spectrum produces the same waveform repeatedly, which sounds like an infinite sustain of the captured moment - a harmonic drone or frozen vowel. 

typedef struct {
    Complex frozen[FFT_SIZE];   /* stored spectrum */
    int     active;             /* 0 = pass-through, 1 = frozen */
} FreezeState;

void spectral_freeze(Complex *bins, int n, void *user) {
    FreezeState *fs = (FreezeState *)user;
    if (!fs->active) {
        /* Copy current spectrum into frozen storage */
        memcpy(fs->frozen, bins, n * sizeof(Complex));
        fs->active = 1;
    }
    /* Replace current bins with stored spectrum */
    memcpy(bins, fs->frozen, n * sizeof(Complex));
}

void freeze_release(FreezeState *fs) {
    fs->active = 0;
}

In practice you want the freeze to crossfade in over several frames rather than switch instantly. Store a blend factor (0.0 = live, 1.0 = fully frozen) and interpolate between the live bins and the frozen bins per-sample before the IFFT. This gives a smooth activation like a reverb tail fading in.

Fast Convolution

Convolving an N-sample signal with an M-sample impulse response directly costs O(N*M) multiplications. When M is large - a reverb impulse response can be hundreds of thousands of samples - this is prohibitively expensive. Fast convolution replaces the time-domain convolution with pointwise multiplication in the frequency domain, exploiting the convolution theorem: convolution in time equals multiplication in frequency. 

/* Fast convolution: convolve 'signal' (len samples) with 'ir' (ir_len samples).
   Output 'out' must have at least (len + ir_len - 1) samples.
   Both inputs are zero-padded to the next power of two >= len + ir_len - 1.
   This is a single-shot (non-streaming) implementation. */

#include <stdlib.h>
#include <string.h>
#include <math.h>

static int next_pow2(int n) {
    int p = 1;
    while (p < n) p <<= 1;
    return p;
}

void fast_convolve(const float *signal, int sig_len,
                   const float *ir,     int ir_len,
                   float       *out) {
    int fft_size = next_pow2(sig_len + ir_len - 1);

    Complex *a = calloc(fft_size, sizeof(Complex));
    Complex *b = calloc(fft_size, sizeof(Complex));

    /* Load inputs (zero-padded by calloc) */
    for (int i = 0; i < sig_len; i++) a[i].re = signal[i];
    for (int i = 0; i < ir_len;  i++) b[i].re = ir[i];

    /* Forward FFT of both */
    fft(a, fft_size, -1);
    fft(b, fft_size, -1);

    /* Pointwise complex multiply: a[k] *= b[k] */
    for (int k = 0; k < fft_size; k++) {
        float re = a[k].re * b[k].re - a[k].im * b[k].im;
        float im = a[k].re * b[k].im + a[k].im * b[k].re;
        a[k].re = re;
        a[k].im = im;
    }

    /* Inverse FFT */
    ifft(a, fft_size);

    /* Copy real part to output (length = sig_len + ir_len - 1) */
    int out_len = sig_len + ir_len - 1;
    for (int i = 0; i < out_len; i++)
        out[i] = a[i].re;

    free(a);
    free(b);
}

For real-time convolution with a long IR you need the overlap-save or partitioned convolution technique, which splits the IR into equal-length partitions and sums the results. This keeps latency proportional to the partition size, not the full IR length. That is a more involved topic; the single-shot version above is sufficient for understanding the principle.

Phase Vocoder Loop Structure

A phase vocoder is the full overlap-add engine extended to track and manipulate the phase of each bin independently. Phase tracking enables time-stretching (slowing down or speeding up audio without changing pitch) and pitch-shifting. The key insight is that when you change the hop size H between analysis and synthesis, the bins advance at a different rate - you must compute how much phase each bin should have advanced and set it explicitly, otherwise you get phase discontinuities that sound like flanging or smearing. 

#include <math.h>

#define BINS (FFT_SIZE / 2 + 1)   /* positive frequencies only */

typedef struct {
    float last_phase[BINS];     /* analysis phase from previous frame */
    float synth_phase[BINS];    /* synthesis phase accumulator */
} PhaseVocoderState;

/* Compute the instantaneous frequency of bin k given the measured phase
   and the expected phase advance for the nominal frequency of that bin.
   Returns true frequency in radians/sample. */
float pv_true_freq(float measured_phase, float last_phase,
                   int k, int hop) {
    float expected = 2.0f * (float)M_PI * k * hop / FFT_SIZE;
    float delta    = measured_phase - last_phase - expected;

    /* Wrap delta to [-pi, pi] */
    while (delta >  (float)M_PI) delta -= 2.0f * (float)M_PI;
    while (delta < -(float)M_PI) delta += 2.0f * (float)M_PI;

    return expected + delta;   /* radians per hop */
}

/* Process one spectrum frame for time-stretch by factor 'stretch'.
   Called instead of the trivial pass-through inside the OLA loop.
   synthesis_hop = analysis_hop * stretch. */
void pv_process_frame(PhaseVocoderState *pv,
                      Complex *bins, float stretch,
                      int analysis_hop) {
    int synthesis_hop = (int)(analysis_hop * stretch + 0.5f);

    for (int k = 0; k < BINS; k++) {
        float mag   = sqrtf(bins[k].re * bins[k].re + bins[k].im * bins[k].im);
        float phase = atan2f(bins[k].im, bins[k].re);

        /* True instantaneous frequency for this bin */
        float true_freq = pv_true_freq(phase, pv->last_phase[k], k, analysis_hop);
        pv->last_phase[k] = phase;

        /* Advance synthesis phase by the true frequency scaled by synthesis hop */
        pv->synth_phase[k] += true_freq * stretch;

        /* Re-synthesise bin with original magnitude, corrected phase */
        bins[k].re = mag * cosf(pv->synth_phase[k]);
        bins[k].im = mag * sinf(pv->synth_phase[k]);
    }

    /* Mirror the positive-frequency bins to negative frequencies */
    for (int k = BINS; k < FFT_SIZE; k++) {
        bins[k].re =  bins[FFT_SIZE - k].re;
        bins[k].im = -bins[FFT_SIZE - k].im;   /* conjugate */
    }
}

The phase vocoder is the conceptual core of most professional time-stretching algorithms (iZotope Radius, Elastique, etc.). The version above has a well-known artefact: it produces phasiness on transients because transients span all bins simultaneously and the per-bin phase accumulation loses their coherence. Real implementations add transient detection to lock phases together around attack events.

Putting It Together: A Working Spectral Gate Demo

/* spectral_gate_demo.c
   Reads 48 kHz float32 mono from stdin, applies spectral gate, writes to stdout.
   Usage: ./spectral_gate_demo < input.raw > output.raw
   Compile: cc -o spectral_gate_demo spectral_gate_demo.c -lm
*/
#include <stdio.h>
#include <string.h>
#include <math.h>
#include <stdlib.h>

/* ... include the fft(), ifft(), OLAEngine, ola_init(), ola_push(),
       SpectralGateParams, spectral_gate() code from above ... */

int main(void) {
    OLAEngine engine;
    ola_init(&engine);

    SpectralGateParams params;
    params.threshold = 30.0f;   /* tune to taste: ~30 for mild noise gate */

    float in_block[HOP];
    float out_block[HOP];

    while (fread(in_block, sizeof(float), HOP, stdin) == (size_t)HOP) {
        ola_push(&engine, in_block, out_block, spectral_gate, &params);
        fwrite(out_block, sizeof(float), HOP, stdout);
    }
    return 0;
}

Summary

Overlap-add is the scaffolding that makes all frequency-domain audio effects work correctly in a continuous stream. With it in place, spectral editing reduces to modifying FFT bins between the forward and inverse transforms. The spectral gate and spectral freeze are the simplest possible edits - zero or replace bins. Fast convolution replaces an expensive FIR filter with two FFTs and a pointwise multiply. The phase vocoder extends the framework with phase tracking, enabling time-stretch and pitch-shift.
