Containers - Part 22: Testing Container Code
2026-06-01


Container code is kernel code in user space. A test suite for namespace and cgroup
manipulation needs real kernel calls — there is nothing useful to mock. This article
shows how to structure integration tests for the code you built throughout this series:
namespace creation, cgroup attachment, pivot_root, and the mini runtime from Parts 19–20.


What You Will Learn


Why container tests must be integration tests, and how to mark them with build tags
Skipping tests that require CAP_SYS_ADMIN when not running as root
Using t.Cleanup and deferred teardown to avoid leaking namespaces and cgroups
Golden-state assertions against /proc and /sys pseudo-filesystems
Subprocess testing: re-executing the test binary inside a new namespace
A reusable harness for spinning up and tearing down the mini runtime


The Integration Testing Constraint


Functions like syscall.Unshare, syscall.Clone, and the cgroup
file interface in /sys/fs/cgroup go straight to the kernel. They cannot be
mocked at the Go level without rewriting the syscall layer. The only honest test
strategy is: run on a real Linux kernel, as root (or with the right capabilities), and
assert on observable kernel state.



Separate these tests from unit tests with a build tag so they are never accidentally
run in a restricted CI environment:



//go:build integration

package container_test



Run them explicitly:



sudo go test -tags=integration ./...


Skipping Without Root


Many namespace operations require CAP_SYS_ADMIN. A test that calls
syscall.Unshare(syscall.CLONE_NEWNS) without it will fail with
EPERM. Skip cleanly rather than fail noisily:



func requireRoot(t *testing.T) {
    t.Helper()
    if os.Getuid() != 0 {
        t.Skip("test requires root (CAP_SYS_ADMIN)")
    }
}



Call it at the top of every test that touches namespaces or cgroups:



func TestCreatePIDNamespace(t *testing.T) {
    requireRoot(t)
    // ... rest of test
}



If you run tests in a user namespace (rootless), you can relax the check:
os.Getuid() != 0 && !hasCapability(CAP_SYS_ADMIN). For simplicity,
the root check is usually sufficient in a container or VM CI environment.


Teardown with t.Cleanup


Namespace and cgroup resources outlive the process that created them unless explicitly
removed. Register teardown in t.Cleanup immediately after creation so the
test always cleans up, even on failure:



func TestCgroupCreation(t *testing.T) {
    requireRoot(t)

    cgroupPath := filepath.Join("/sys/fs/cgroup/memory", "testns_"+t.Name())

    if err := os.Mkdir(cgroupPath, 0755); err != nil {
        t.Fatalf("create cgroup: %v", err)
    }
    t.Cleanup(func() {
        if err := os.Remove(cgroupPath); err != nil {
            t.Logf("cleanup cgroup %s: %v", cgroupPath, err)
        }
    })

    // Assert the cgroup directory and its standard files exist.
    for _, f := range []string{"memory.limit_in_bytes", "memory.usage_in_bytes", "tasks"} {
        path := filepath.Join(cgroupPath, f)
        if _, err := os.Stat(path); err != nil {
            t.Errorf("expected cgroup file %s: %v", f, err)
        }
    }
}



The same pattern applies to mount points, temporary directories, and veth pairs:
always register the inverse operation in t.Cleanup before the next
operation that could fail.


Testing Namespace Unshare


syscall.Unshare affects the calling goroutine's thread. Because the Go
scheduler can move goroutines between OS threads, namespace tests must be locked to
a single thread with runtime.LockOSThread:



func TestUTSNamespace(t *testing.T) {
    requireRoot(t)

    runtime.LockOSThread()
    defer runtime.UnlockOSThread()

    // Record the original hostname so we can check isolation.
    origHostname, err := os.Hostname()
    if err != nil {
        t.Fatal(err)
    }

    if err := syscall.Unshare(syscall.CLONE_NEWUTS); err != nil {
        t.Fatalf("unshare UTS: %v", err)
    }

    const testHostname = "container-test"
    if err := syscall.Sethostname([]byte(testHostname)); err != nil {
        t.Fatalf("sethostname: %v", err)
    }

    got, err := os.Hostname()
    if err != nil {
        t.Fatal(err)
    }
    if got != testHostname {
        t.Errorf("hostname in namespace = %q, want %q", got, testHostname)
    }

    // The parent namespace hostname is unaffected — verified via /proc after the
    // test exits this goroutine's namespace by killing the OS thread.
    _ = origHostname
}


Subprocess Re-Execution Pattern


Namespace code that must run in a new namespace context cannot be tested in the same
process: once a process is in a namespace, you cannot easily get back out. The
standard pattern is to re-execute the test binary as a child process inside the new
namespace, much like runc init does.



// TestMain detects a special env var and runs the "namespace init" role.
func TestMain(m *testing.M) {
    if role := os.Getenv("TEST_NS_ROLE"); role != "" {
        runNamespaceRole(role)
        os.Exit(0)
    }
    os.Exit(m.Run())
}

func runNamespaceRole(role string) {
    switch role {
    case "pid_init":
        // This runs inside a new PID namespace.
        // PID 1 in the new namespace.
        fmt.Println(os.Getpid()) // Should print 1.
    default:
        fmt.Fprintf(os.Stderr, "unknown role: %s\n", role)
        os.Exit(1)
    }
}

func TestPIDNamespaceChildIsOne(t *testing.T) {
    requireRoot(t)

    cmd := exec.Command("/proc/self/exe", "-test.run=^$")
    cmd.Env = append(os.Environ(), "TEST_NS_ROLE=pid_init")
    cmd.SysProcAttr = &syscall.SysProcAttr{
        Cloneflags: syscall.CLONE_NEWPID | syscall.CLONE_NEWNS,
    }

    out, err := cmd.Output()
    if err != nil {
        t.Fatalf("child: %v", err)
    }

    pid := strings.TrimSpace(string(out))
    if pid != "1" {
        t.Errorf("child PID in new namespace = %s, want 1", pid)
    }
}


Golden-State Assertions on /proc


The /proc filesystem exposes kernel state as readable files. Assert on
/proc/self/ns/* symlinks to verify namespace isolation, and on
/proc/self/cgroup to verify cgroup membership:



func readNSInode(t *testing.T, ns string) uint64 {
    t.Helper()
    path := fmt.Sprintf("/proc/self/ns/%s", ns)
    info, err := os.Lstat(path)
    if err != nil {
        t.Fatalf("stat %s: %v", path, err)
    }
    stat, ok := info.Sys().(*syscall.Stat_t)
    if !ok {
        t.Fatal("unexpected stat type")
    }
    return stat.Ino
}

func TestMountNamespaceIsIsolated(t *testing.T) {
    requireRoot(t)
    runtime.LockOSThread()
    defer runtime.UnlockOSThread()

    parentIno := readNSInode(t, "mnt")

    if err := syscall.Unshare(syscall.CLONE_NEWNS); err != nil {
        t.Fatalf("unshare: %v", err)
    }

    childIno := readNSInode(t, "mnt")

    if parentIno == childIno {
        t.Error("mount namespace inode unchanged after CLONE_NEWNS — isolation failed")
    }
}



For cgroup v2, read /proc/self/cgroup and assert the path segment:



func TestProcessAttachedToCgroup(t *testing.T) {
    requireRoot(t)

    cgroupName := "testrun_" + t.Name()
    cgroupPath := filepath.Join("/sys/fs/cgroup", cgroupName)

    if err := os.Mkdir(cgroupPath, 0755); err != nil {
        t.Fatalf("mkdir: %v", err)
    }
    t.Cleanup(func() { os.Remove(cgroupPath) })

    // Write our PID into the cgroup.
    procsFile := filepath.Join(cgroupPath, "cgroup.procs")
    pid := fmt.Sprintf("%d\n", os.Getpid())
    if err := os.WriteFile(procsFile, []byte(pid), 0644); err != nil {
        t.Fatalf("write procs: %v", err)
    }

    // Read /proc/self/cgroup and check we appear in the right cgroup.
    data, err := os.ReadFile("/proc/self/cgroup")
    if err != nil {
        t.Fatal(err)
    }
    if !strings.Contains(string(data), cgroupName) {
        t.Errorf("/proc/self/cgroup does not contain %q:\n%s", cgroupName, data)
    }
}


Asserting on /sys/fs/cgroup Memory Limits


After setting a memory limit, read it back from /sys/fs/cgroup and compare:



func TestCgroupMemoryLimit(t *testing.T) {
    requireRoot(t)

    cgroupPath := filepath.Join("/sys/fs/cgroup/memory", "memlimit_"+t.Name())
    if err := os.Mkdir(cgroupPath, 0755); err != nil {
        t.Fatalf("mkdir: %v", err)
    }
    t.Cleanup(func() { os.Remove(cgroupPath) })

    const wantLimit = 64 * 1024 * 1024 // 64 MiB

    limitFile := filepath.Join(cgroupPath, "memory.limit_in_bytes")
    if err := os.WriteFile(limitFile, []byte(fmt.Sprintf("%d\n", wantLimit)), 0644); err != nil {
        t.Fatalf("write limit: %v", err)
    }

    data, err := os.ReadFile(limitFile)
    if err != nil {
        t.Fatal(err)
    }
    got, err := strconv.ParseInt(strings.TrimSpace(string(data)), 10, 64)
    if err != nil {
        t.Fatalf("parse limit: %v", err)
    }
    if got != wantLimit {
        t.Errorf("memory limit = %d, want %d", got, wantLimit)
    }
}


A Reusable Runtime Harness


Tests for the mini runtime (Parts 19–20) need to repeatedly spin up a container,
run a command inside it, and tear it down. Extract this into a harness so each test
is three lines:



// RuntimeHarness manages the lifecycle of a mini container for testing.
type RuntimeHarness struct {
    t         *testing.T
    RootFS    string // temp dir used as container rootfs
    CgroupDir string // cgroup created for this container
}

func NewRuntimeHarness(t *testing.T) *RuntimeHarness {
    t.Helper()
    requireRoot(t)

    rootfs := t.TempDir() // t.TempDir() registers its own t.Cleanup

    cgroupDir := filepath.Join("/sys/fs/cgroup/memory", "test_"+sanitize(t.Name()))
    if err := os.Mkdir(cgroupDir, 0755); err != nil {
        t.Fatalf("harness: mkdir cgroup: %v", err)
    }
    t.Cleanup(func() { os.Remove(cgroupDir) })

    // Populate a minimal rootfs: /bin, /proc, /sys, /tmp.
    for _, dir := range []string{"bin", "proc", "sys", "tmp"} {
        if err := os.Mkdir(filepath.Join(rootfs, dir), 0755); err != nil {
            t.Fatalf("harness: mkdir %s: %v", dir, err)
        }
    }

    // Copy a static busybox binary for a useful shell in tests.
    busybox, err := exec.LookPath("busybox")
    if err == nil {
        copyFile(t, busybox, filepath.Join(rootfs, "bin", "sh"))
    }

    return &RuntimeHarness{t: t, RootFS: rootfs, CgroupDir: cgroupDir}
}

func (h *RuntimeHarness) Run(args ...string) (string, error) {
    cmd := exec.Command("/proc/self/exe", append([]string{"--container-init"}, args...)...)
    cmd.Env = []string{
        "CONTAINER_ROOTFS=" + h.RootFS,
        "CONTAINER_CGROUP=" + h.CgroupDir,
    }
    cmd.SysProcAttr = &syscall.SysProcAttr{
        Cloneflags: syscall.CLONE_NEWPID |
            syscall.CLONE_NEWNS |
            syscall.CLONE_NEWUTS |
            syscall.CLONE_NEWIPC,
    }
    out, err := cmd.CombinedOutput()
    return string(out), err
}

func sanitize(s string) string {
    return strings.Map(func(r rune) rune {
        if r = 'a' && r <= 'z' || r = 'A' && r <= 'Z' || r = '0' && r <= '9' {
            return r
        }
        return '_'
    }, s)
}



Using the harness in a test:



func TestContainerHostname(t *testing.T) {
    h := NewRuntimeHarness(t)

    out, err := h.Run("hostname")
    if err != nil {
        t.Fatalf("run: %v\n%s", err, out)
    }

    // The container's hostname should be isolated from the host.
    hostHostname, _ := os.Hostname()
    containerHostname := strings.TrimSpace(out)
    if containerHostname == hostHostname {
        t.Errorf("container hostname %q matches host — UTS namespace not isolated", containerHostname)
    }
}

func TestContainerPIDOne(t *testing.T) {
    h := NewRuntimeHarness(t)

    out, err := h.Run("sh", "-c", "echo $$")
    if err != nil {
        t.Fatalf("run: %v\n%s", err, out)
    }

    pid := strings.TrimSpace(out)
    if pid != "1" {
        t.Errorf("init PID = %s, want 1", pid)
    }
}


Testing Seccomp Filter Application


Verify that a seccomp filter denies the right syscalls. Use the subprocess pattern:
run a child that attempts the blocked syscall and assert the exit status is SIGSYS (31):



func TestSeccompBlocksMknod(t *testing.T) {
    requireRoot(t)

    cmd := exec.Command("/proc/self/exe", "-test.run=^$")
    cmd.Env = append(os.Environ(), "TEST_NS_ROLE=try_mknod")
    cmd.SysProcAttr = &syscall.SysProcAttr{
        Cloneflags: syscall.CLONE_NEWUSER | syscall.CLONE_NEWNS,
    }

    err := cmd.Run()
    if err == nil {
        t.Fatal("expected child to be killed by seccomp, but it succeeded")
    }

    exitErr, ok := err.(*exec.ExitError)
    if !ok {
        t.Fatalf("unexpected error type: %T %v", err, err)
    }

    status := exitErr.Sys().(syscall.WaitStatus)
    if sig := status.Signal(); sig != syscall.SIGSYS {
        t.Errorf("killed by signal %v, want SIGSYS (seccomp)", sig)
    }
}


Table-Driven Namespace Tests


Use table-driven tests to cover all six namespace types without repeating boilerplate:



func TestNamespaceIsolation(t *testing.T) {
    requireRoot(t)

    tests := []struct {
        name      string
        cloneFlag uintptr
        nsFile    string
    }{
        {"UTS", syscall.CLONE_NEWUTS, "uts"},
        {"IPC", syscall.CLONE_NEWIPC, "ipc"},
        {"MNT", syscall.CLONE_NEWNS, "mnt"},
        {"PID", syscall.CLONE_NEWPID, "pid"},
        {"NET", syscall.CLONE_NEWNET, "net"},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            runtime.LockOSThread()
            defer runtime.UnlockOSThread()

            before := readNSInode(t, tt.nsFile)

            if err := syscall.Unshare(tt.cloneFlag); err != nil {
                t.Fatalf("unshare %s: %v", tt.name, err)
            }

            after := readNSInode(t, tt.nsFile)
            if before == after {
                t.Errorf("%s namespace inode unchanged after CLONE_%s", tt.name, tt.name)
            }
        })
    }
}


Running in CI


Integration tests for container code require a Linux kernel with namespace and cgroup
support. In GitHub Actions, use ubuntu-latest with sudo:



# .github/workflows/integration.yml
name: integration-tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version: '1.23' }
      - name: Run integration tests
        run: sudo go test -tags=integration -v -count=1 ./...



The -count=1 flag disables the test cache, which is important for tests
that depend on kernel state — a cached pass from a previous run could hide a regression
introduced by a kernel upgrade or configuration change.


Summary


Container code cannot be meaningfully unit-tested: it calls the kernel, and the kernel
cannot be mocked. Use a build tag to isolate integration tests; skip with
t.Skip when not running as root. Register all namespace and cgroup
teardown in t.Cleanup immediately after creation. Lock goroutines to OS
threads with runtime.LockOSThread before Unshare. Use the
subprocess re-execution pattern to run code that must live inside a new namespace.
Assert on /proc/self/ns/* inodes to verify isolation, and on
/proc/self/cgroup and /sys/fs/cgroup files to verify cgroup
membership and resource limits. Wrap the mini runtime in a RuntimeHarness
so each test is a one-liner.
