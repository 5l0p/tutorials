RPIPicoDSP - Part 2: Audio Output on the Pico

Getting audio out of a microcontroller is where theory meets hardware. The RP2040 has no dedicated audio peripheral - no I2S hardware block, no built-in DAC - but it has something more flexible: PIO state machines that can implement any clocked serial protocol in firmware, driven by DMA so the CPU never touches individual bits. This article covers the three realistic output options, explains why the DMA double-buffer pattern is non-negotiable for glitch-free audio, and ends with a complete working example that drives a PCM5102 DAC via I2S and outputs a 440 Hz sine wave.

Option 1: PWM Audio

The Pico's hardware PWM can produce audio by driving a pin with a rapidly varying duty cycle. A low-pass filter on the output (a simple RC network) smooths the square wave into an analogue voltage proportional to the duty cycle. At 125 MHz system clock and a PWM period of 256 counts, the PWM carrier frequency is 488 kHz - well above audio range. A first-order RC with a 10 kHz corner frequency suppresses the carrier enough for headphone-level listening.

The fundamental problem with PWM audio is aliasing. The output is quantised in time to the PWM period (about 2 µs at 488 kHz carrier). If your audio sample rate is 44.1 kHz, consecutive PWM duty-cycle updates are roughly 22 µs apart. The discrete jumps between duty cycles introduce harmonic distortion and noise that no amount of filtering fully removes. The effective dynamic range is also limited: with an 8-bit equivalent resolution (256 levels) you get at best 48 dB SNR.

PWM audio is useful for: simple tones, game sound effects, debug audio ("can I hear a 440 Hz tone at all?"). It is not suitable for music production or hi-fi listening. For those, use I2S. 

/* Minimal PWM audio setup - 8-bit resolution at ~488 kHz carrier */
#include "hardware/pwm.h"
#include "hardware/irq.h"

#define AUDIO_PIN   0    /* GP0 - add RC filter to this pin */
#define WRAP_VALUE  255  /* 8-bit resolution: 0..255 */

void pwm_audio_init(void) {
    gpio_set_function(AUDIO_PIN, GPIO_FUNC_PWM);
    uint slice = pwm_gpio_to_slice_num(AUDIO_PIN);
    pwm_config cfg = pwm_get_default_config();
    pwm_config_set_wrap(&cfg, WRAP_VALUE);
    /* No clock divider: 125 MHz / 256 = ~488 kHz carrier */
    pwm_config_set_clkdiv(&cfg, 1.0f);
    pwm_init(slice, &cfg, true);
}

/* Call this from a timer or DMA IRQ at your sample rate */
void pwm_audio_put_sample(uint8_t sample) {
    uint slice = pwm_gpio_to_slice_num(AUDIO_PIN);
    pwm_set_gpio_level(AUDIO_PIN, sample);
}

Option 2: I2S via PIO - The Right Approach

I2S (Inter-IC Sound) is the standard serial protocol for connecting a microcontroller to a dedicated external DAC chip. Three signals carry the audio:

  • BCLK (Bit Clock) - one pulse per bit. For 44.1 kHz stereo 16-bit audio: 44100 × 2 channels × 16 bits = 1.4112 MHz.
  • LRCLK (Left-Right Clock, also called WS - Word Select) - one pulse per frame, toggling at the sample rate (44.1 kHz). Low = left channel, high = right channel in standard I2S.
  • DATA - the serial bit stream, MSB first, with the MSB one BCLK cycle after the LRCLK edge (the I2S standard introduces this one-bit offset; the Philips/I2S spec calls this "left-justified one bit late").

The RP2040 has no hardware I2S peripheral. Instead, a PIO state machine clocks out the bits. A PIO program is a tight loop of 9 instructions or fewer that shifts bits out of its output shift register (OSR), pulses BCLK, and manages LRCLK. The DMA feeds 32-bit words (two 16-bit samples packed, left in the upper half) into the PIO's TX FIFO.

The I2S PIO Program

The Pico SDK ships a ready-made I2S PIO program in pico-extras (the audio_i2s library). For learning purposes, here is a minimal hand-written version that illustrates the mechanics. Understanding this is valuable even if you use the SDK library in practice. 

; i2s_out.pio - minimal I2S master output
; Assumes: DATA on pin base, BCLK on pin base+1, LRCLK on pin base+2
; The autopull threshold is set to 32 bits in the C setup code.
; DMA feeds packed stereo: bits[31:16] = left, bits[15:0] = right.

.program i2s_out
.side_set 2                ; BCLK and LRCLK are side-set pins

                            ; side bits: [LRCLK, BCLK]
.wrap_target
    set x, 14   side 0b00  ; 15 more bits for left channel (we do 16 total)
left_loop:
    out pins, 1 side 0b01  ; output 1 bit, BCLK high
    jmp x-- left_loop side 0b00  ; BCLK low, count down

    out pins, 1 side 0b11  ; last left bit + LRCLK goes high (right channel starts)
    set x, 14   side 0b10

right_loop:
    out pins, 1 side 0b11  ; output 1 bit, BCLK high, LRCLK high
    jmp x-- right_loop side 0b10 ; BCLK low

    out pins, 1 side 0b10  ; last right bit
.wrap

This PIO program is compiled by pioasm (part of the SDK build) from a .pio file into a C header containing the instruction words and a pio_program_t struct. pio_add_program loads it into one of the two PIO blocks at runtime.

DMA Double-Buffering

The audio DMA works like this: you have two sample buffers, A and B. DMA is told to transfer buffer A to the PIO TX FIFO. While DMA is playing A, your CPU fills B with new DSP output. When DMA finishes A it fires an interrupt. The interrupt handler immediately restarts DMA from B and sets a flag. Your main audio loop sees the flag, fills A, and clears the flag. Then the cycle repeats.

Why two buffers? If you had only one, you would have to stop DMA, refill the buffer, and restart - creating a gap (a click or dropout) in the audio stream. With two buffers the DMA never stops; it always has a fresh buffer to play.

The DMA interrupt handler must do three things in order:

  1. Clear the DMA interrupt flag. If you do not clear it before returning, the handler fires again immediately in an infinite loop.
  2. Restart DMA from the other buffer (the one the CPU just finished filling).
  3. Set a flag or semaphore telling the CPU to fill the now-playing buffer.

The PCM5102 DAC

The PCM5102 is an excellent, cheap stereo I2S DAC with a 3.5 mm headphone output driver built in. It supports 16–32 bit samples at sample rates from 8 kHz to 384 kHz. Wiring to the Pico: 

PCM5102   Pico GPIO   Notes
-------   ---------   -----
BCK       GP10        BCLK
LRCK      GP11        LRCLK  (= pin base + 2 in the PIO side-set)
DIN       GP9         DATA   (= pin base)
SCK       GND         The PCM5102 has an internal PLL; tie SCK low
FLT       GND         Normal latency filter
DEMP      GND         De-emphasis off
XSMT      3V3         Soft mute off (3V3 = unmuted)
VCC       3V3
GND       GND

No additional components are needed; the PCM5102 includes its own output filter and headphone driver.

Complete I2S Output Example

This example uses the Pico SDK's pico-extras audio I2S library, which provides a well-tested PIO program and DMA plumbing. The pico-extras repository lives alongside pico-sdk. 

/* CMakeLists.txt */

cmake_minimum_required(VERSION 3.13)
include(pico_sdk_import.cmake)

# pico-extras provides the audio_i2s library
# Set PICO_EXTRAS_PATH or clone alongside pico-sdk
include($ENV{PICO_EXTRAS_PATH}/external/pico_extras_import.cmake)

project(i2s_sine C CXX ASM)
set(CMAKE_C_STANDARD 11)
pico_sdk_init()

add_executable(i2s_sine main.c)

target_link_libraries(i2s_sine
    pico_stdlib
    pico_audio_i2s      # from pico-extras
)

# Route stdio to USB serial for debug output
pico_enable_stdio_usb(i2s_sine 1)
pico_enable_stdio_uart(i2s_sine 0)
pico_add_extra_outputs(i2s_sine)

/* main.c - 440 Hz sine wave via I2S DMA double-buffer */
#include <stdio.h>
#include <math.h>
#include <string.h>
#include "pico/stdlib.h"
#include "pico/audio_i2s.h"

#define SAMPLE_RATE     44100
#define BUFFER_FRAMES   256     /* samples per channel per buffer */
#define I2S_DATA_PIN    9       /* DATA  = GP9  */
#define I2S_CLOCK_PIN   10      /* BCLK  = GP10, LRCLK = GP11 automatically */

/* Precomputed 512-point sine table, scaled to 16-bit signed range.
   Storing in flash (const) saves precious SRAM. */
#define SINE_TABLE_LEN 512
static const int16_t sine_table[SINE_TABLE_LEN] = {
    /* Values filled at compile time - in practice generate with a script
       or compute at startup once into a static array.
       For brevity we compute at startup in this example. */
    0 /* placeholder - see init code below */
};

/* Runtime sine table (computed at startup to avoid a huge literal) */
static int16_t sine_lut[SINE_TABLE_LEN];

/* Phase accumulator for the 440 Hz tone */
static uint32_t phase_acc  = 0;
static uint32_t phase_inc  = 0;   /* set in main() */

/* --- Audio callback ---
   Called by the audio_i2s library when it needs a new buffer.
   This runs in interrupt context - keep it short and deterministic. */
static void audio_callback(audio_buffer_pool_t *pool) {
    audio_buffer_t *buf = take_audio_buffer(pool, false);
    if (!buf) return;

    int16_t *samples = (int16_t *)buf->buffer->bytes;
    uint32_t n = buf->max_sample_count;

    for (uint32_t i = 0; i < n; i++) {
        /* Look up sine value using the upper bits of the phase accumulator
           as an index into the table. This is wavetable oscillator basics. */
        uint32_t idx = (phase_acc >> 23) & (SINE_TABLE_LEN - 1); /* 9 bits */
        int16_t s = sine_lut[idx];
        phase_acc += phase_inc;

        /* Interleaved stereo: left then right */
        samples[i * 2]     = s;   /* left  */
        samples[i * 2 + 1] = s;   /* right */
    }
    buf->sample_count = n;
    give_audio_buffer(pool, buf);
}

int main(void) {
    stdio_init_all();

    /* Build the sine lookup table at startup */
    for (int i = 0; i < SINE_TABLE_LEN; i++) {
        sine_lut[i] = (int16_t)(32767.0f * sinf(2.0f * (float)M_PI * i / SINE_TABLE_LEN));
    }

    /* Phase increment for 440 Hz at SAMPLE_RATE.
       The accumulator is 32 bits; one full rotation = 2^32 steps.
       phase_inc = 440 / 44100 * 2^32 */
    phase_inc = (uint32_t)((440.0f / SAMPLE_RATE) * (float)(1u << 31) * 2.0f);

    /* Configure the I2S audio output */
    audio_i2s_config_t config = {
        .data_pin       = I2S_DATA_PIN,
        .clock_pin_base = I2S_CLOCK_PIN, /* BCLK=GP10, LRCLK=GP11 */
        .dma_channel    = 0,
        .pio_sm         = 0,
    };

    const audio_format_t audio_format = {
        .format         = AUDIO_PCM_FORMAT_S16,
        .sample_freq    = SAMPLE_RATE,
        .channel_count  = 2,
    };

    const audio_buffer_format_t producer_format = {
        .format          = &audio_format,
        .sample_stride   = 4,  /* 2 bytes left + 2 bytes right */
    };

    audio_buffer_pool_t *producer_pool = audio_new_producer_pool(
        &producer_format, 3, BUFFER_FRAMES);

    bool ok = audio_i2s_setup(&audio_format, &audio_format);
    if (!ok) {
        printf("audio_i2s_setup failed\n");
        while (1) tight_loop_contents();
    }

    ok = audio_i2s_connect(producer_pool);
    audio_i2s_set_enabled(true);

    printf("I2S audio running at %d Hz\n", SAMPLE_RATE);

    /* Core 0 main loop: call audio_callback whenever a buffer is needed.
       In Part 4 we move this to an interrupt on core 1. */
    while (true) {
        audio_callback(producer_pool);
    }
}

A few implementation notes:

  • Phase accumulator size: A 32-bit accumulator gives 232 / 44100 ≈ 97 kHz of phase resolution. The ratio 440/44100 fits with no audible pitch error.
  • Table indexing: We shift the accumulator right by 23 bits to get a 9-bit index (values 0–511). Any power-of-two table length works; mask with SINE_TABLE_LEN - 1 to wrap.
  • Interleaved stereo: The I2S library expects interleaved L/R pairs: samples[0] = L0, samples[1] = R0, samples[2] = L1, etc. This matches the I2S wire format where left and right alternate on each LRCLK period.
  • Buffer count: We allocate three buffers (not two) so the library always has a spare. With two buffers, a brief overrun on the CPU side would cause a dropout. Three gives a one-buffer margin.

Summary

PWM audio is quick to set up but suffers from quantisation noise and aliasing that make it unsuitable for music-quality output. I2S via a PIO state machine feeding a PCM5102 DAC gives CD-quality output with no analogue filtering required on your side. The DMA double-buffer pattern - two buffers, DMA plays one while CPU fills the other, interrupt swaps them - is the backbone of all glitch-free embedded audio. You now have a working I2S sine-wave output that you can build on for the rest of the series.
