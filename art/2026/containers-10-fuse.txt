Containers - Part 10: FUSE: Writing a Userspace Filesystem
2026-08-08


FUSE (Filesystem in Userspace) lets you write a filesystem driver as an
ordinary process rather than a kernel module. The kernel's FUSE module bridges
VFS calls to your process via a device node, your process answers them, and the
kernel presents the result to the caller. This is how SSHFS, S3FS, and many
container volume drivers work. This article explains the architecture, shows a
complete passthrough filesystem in C with libfuse, and then builds the
same thing in Go with bazil.org/fuse.


What You Will Learn


How the FUSE kernel module bridges VFS calls to a userspace process via /dev/fuse
A complete passthrough filesystem in C using libfuse3, covering getattr, readdir, open, read, write, and create

The same passthrough filesystem in Go using bazil.org/fuse interfaces
Why FUSE adds latency and how to mitigate it for high-throughput workloads
How FUSE is used in container contexts (sshfs, s3fs, fuse-overlayfs) and what capabilities a container needs


How FUSE Works


When a process accesses a path on a FUSE-mounted filesystem:



The kernel's VFS layer calls the FUSE module with the operation (open, read, write, readdir, etc.).
The FUSE module writes the request as a message to /dev/fuse.
The userspace filesystem process reads from /dev/fuse, processes the request, and writes a reply.
The FUSE module receives the reply and returns it to the original caller.



Each VFS operation corresponds to a message type defined in
<linux/fuse.h>. The libfuse library hides the message protocol
behind a struct of function pointers (struct fuse_operations).

C: A Passthrough Filesystem with libfuse3


A passthrough filesystem maps every operation on the mount point to the
corresponding operation on a real directory (the "source"). This is the
simplest non-trivial FUSE filesystem:



#define FUSE_USE_VERSION 31
#include <errno.h>
#include <fcntl.h>
#include <fuse3/fuse.h>
#include <limits.h>
#include <stdio.h>
#include <string.h>
#include <sys/stat.h>
#include <unistd.h>

static const char *source_path;

static const char *real_path(const char *path) {
    static char buf[PATH_MAX];
    snprintf(buf, sizeof(buf), "%s%s", source_path, path);
    return buf;
}

static int pt_getattr(const char *path, struct stat *st,
                      struct fuse_file_info *fi) {
    (void)fi;
    if (lstat(real_path(path), st) == -1) return -errno;
    return 0;
}

static int pt_readdir(const char *path, void *buf,
                      fuse_fill_dir_t filler,
                      off_t off, struct fuse_file_info *fi,
                      enum fuse_readdir_flags flags) {
    (void)off; (void)fi; (void)flags;
    DIR *dp = opendir(real_path(path));
    if (!dp) return -errno;

    struct dirent *de;
    while ((de = readdir(dp)) != NULL) {
        struct stat st = { .st_ino = de->d_ino, .st_mode = de->d_type << 12 };
        if (filler(buf, de->d_name, &st, 0, 0)) break;
    }
    closedir(dp);
    return 0;
}

static int pt_open(const char *path, struct fuse_file_info *fi) {
    int fd = open(real_path(path), fi->flags);
    if (fd == -1) return -errno;
    fi->fh = fd;
    return 0;
}

static int pt_read(const char *path, char *buf, size_t size,
                   off_t offset, struct fuse_file_info *fi) {
    (void)path;
    ssize_t n = pread(fi->fh, buf, size, offset);
    if (n == -1) return -errno;
    return (int)n;
}

static int pt_write(const char *path, const char *buf, size_t size,
                    off_t offset, struct fuse_file_info *fi) {
    (void)path;
    ssize_t n = pwrite(fi->fh, buf, size, offset);
    if (n == -1) return -errno;
    return (int)n;
}

static int pt_release(const char *path, struct fuse_file_info *fi) {
    (void)path;
    close(fi->fh);
    return 0;
}

static int pt_create(const char *path, mode_t mode,
                     struct fuse_file_info *fi) {
    int fd = open(real_path(path), fi->flags, mode);
    if (fd == -1) return -errno;
    fi->fh = fd;
    return 0;
}

static int pt_unlink(const char *path) {
    if (unlink(real_path(path)) == -1) return -errno;
    return 0;
}

static int pt_mkdir(const char *path, mode_t mode) {
    if (mkdir(real_path(path), mode) == -1) return -errno;
    return 0;
}

static const struct fuse_operations pt_ops = {
    .getattr = pt_getattr,
    .readdir = pt_readdir,
    .open    = pt_open,
    .read    = pt_read,
    .write   = pt_write,
    .release = pt_release,
    .create  = pt_create,
    .unlink  = pt_unlink,
    .mkdir   = pt_mkdir,
};

int main(int argc, char *argv[]) {
    if (argc < 3) {
        fprintf(stderr, "usage: passthrough <source> <mountpoint>\n");
        return 1;
    }
    source_path = argv[1];

    /* Shift argv so fuse_main sees [mountpoint] */
    argv[1] = argv[2];
    argc = 2;

    return fuse_main(argc, argv, &pt_ops, NULL);
}



Compile and run:



gcc -o passthrough passthrough.c $(pkg-config --cflags --libs fuse3)
mkdir /tmp/source /tmp/mnt
echo hello > /tmp/source/test.txt
./passthrough /tmp/source /tmp/mnt
cat /tmp/mnt/test.txt      # hello
fusermount3 -u /tmp/mnt


The fuse_operations Struct


struct fuse_operations has over 40 function pointers. The most commonly
implemented ones are:



getattr - stat(). Called before almost every other operation.
readdir - enumerate directory contents.
open / release - open and close a file handle.
read / write - read and write file data.
create / unlink / mkdir / rmdir - filesystem mutation.
rename / link / symlink / readlink - path operations.
truncate / chmod / chown / utimens - metadata.



Any pointer left NULL causes the kernel to return ENOSYS to callers
of the corresponding syscall.


Go: bazil.org/fuse


bazil.org/fuse is the idiomatic Go FUSE library. It uses Go interfaces
rather than function pointers:



package main

import (
    "context"
    "log"
    "os"
    "syscall"

    "bazil.org/fuse"
    "bazil.org/fuse/fs"
)

type FS struct{ root string }

func (f FS) Root() (fs.Node, error) {
    return &Dir{path: f.root}, nil
}

type Dir struct{ path string }

func (d *Dir) Attr(ctx context.Context, a *fuse.Attr) error {
    info, err := os.Stat(d.path)
    if err != nil {
        return syscall.ENOENT
    }
    a.Inode = 1
    a.Mode  = info.Mode()
    return nil
}

func (d *Dir) Lookup(ctx context.Context, name string,
                     node *fs.LookupRequest, resp *fs.LookupResponse) (fs.Node, error) {
    p := d.path + "/" + name
    info, err := os.Stat(p)
    if err != nil {
        return nil, syscall.ENOENT
    }
    if info.IsDir() {
        return &Dir{path: p}, nil
    }
    return &File{path: p}, nil
}

type File struct{ path string }

func (f *File) Attr(ctx context.Context, a *fuse.Attr) error {
    info, err := os.Stat(f.path)
    if err != nil {
        return syscall.ENOENT
    }
    a.Mode = info.Mode()
    a.Size = uint64(info.Size())
    return nil
}

func (f *File) ReadAll(ctx context.Context) ([]byte, error) {
    return os.ReadFile(f.path)
}

func main() {
    if len(os.Args) != 3 {
        log.Fatal("usage: gofuse <source> <mountpoint>")
    }
    source := os.Args[1]
    mnt    := os.Args[2]

    c, err := fuse.Mount(mnt,
        fuse.FSName("passthrough"),
        fuse.Subtype("gopass"),
    )
    if err != nil {
        log.Fatal(err)
    }
    defer c.Close()

    log.Printf("mounted %s on %s", source, mnt)
    if err := fs.Serve(c, FS{root: source}); err != nil {
        log.Fatal(err)
    }
}


Performance Considerations


FUSE adds at least two context switches per operation (kernel to userspace for
the request, userspace back to kernel for the reply). For metadata-heavy
workloads (many small files, frequent stat calls) this overhead is significant.
libfuse3 introduced -o clone_fd and -o max_idle_threads options to
reduce serialisation. For high-throughput use, consider kernel filesystem
drivers (ext4, XFS, btrfs) or passthrough NFS.


FUSE in Container Contexts


FUSE filesystems are used in container environments for:



SSHFS - mount a remote host's directory locally.
S3FS, goofys - mount an S3 bucket as a filesystem.
rclone mount - generic cloud storage backend.
fuse-overlayfs - OverlayFS in userspace for rootless containers (where kernel OverlayFS requires privileges).



To use FUSE inside a container the container needs /dev/fuse and the
CAP_SYS_ADMIN capability (or a user namespace that grants it). Docker
exposes /dev/fuse with --device /dev/fuse.


Summary


FUSE routes VFS calls from the kernel through /dev/fuse to a userspace
process that answers them. libfuse hides the protocol behind
struct fuse_operations; bazil.org/fuse does the same in Go with
interfaces. The main cost is two context switches per operation, which matters
for metadata-heavy workloads. In container environments, FUSE enables volume
drivers like sshfs and s3fs; rootless containers use fuse-overlayfs when kernel
OverlayFS requires privileges.
