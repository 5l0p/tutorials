Pico — Part 6: 4-Operator FM Synthesis
2026-11-12


Frequency modulation synthesis produces a vast palette of timbres from nothing but
sine oscillators and addition. When one oscillator's output is used as the frequency
input to another, the resulting waveform develops sidebands - new frequency components
appear above and below the carrier, controlled by the modulation index. The Yamaha DX7
did this in 1983 with six operators and 32 algorithm graphs. We will do it with four
operators and one algorithm on an RP2040 running at 125 MHz, fit it inside 20% of the
audio CPU budget, and explain every decision along the way.


What You Will Learn


The FM operator struct: phase accumulator, frequency ratio, envelope, output
ADSR envelope implementation in Q15 fixed-point with linear segments
Algorithm 5 from the DX7: two independent carrier-modulator pairs in parallel
Modulation index scaling: how to go from a 0-127 parameter to a phase offset in radians
Storing the sine wavetable in flash XIP and reading it from the per-sample render loop
SysTick benchmark: CPU% of the 4-operator engine at 44.1 kHz


FM Synthesis in Brief


In FM synthesis, an operator is a sine oscillator with an amplitude envelope. A
modulator operator drives the phase of a carrier operator. The carrier's
instantaneous phase becomes:



carrier_phase(t) = 2*pi*fc*t + I * modulator_output(t)



where I is the modulation index. When I = 0 the carrier is a pure sine.
As I increases, sidebands appear at fc ± n*fm for integer n, with amplitudes
proportional to Bessel functions. The perceived timbre shifts from pure to brassy to
bell-like to metallic as I grows.



The key insight for implementation: "modulating the frequency" is the same as
"adding an offset to the phase accumulator". You never need a continuously varying
frequency - you just add the modulator's current sample (scaled by I) to the carrier's
phase each tick.


The Operator Struct


#include <stdint.h>

#define SAMPLE_RATE      44100
#define TABLE_SIZE       2048
#define PHASE_FRAC_BITS  22

/* ADSR states */
typedef enum {
    ENV_IDLE = 0,
    ENV_ATTACK,
    ENV_DECAY,
    ENV_SUSTAIN,
    ENV_RELEASE
} EnvState;

/* ADSR envelope - all levels in Q15 [0, 32767],
   times as samples-per-stage (precomputed from ms). */
typedef struct {
    EnvState state;
    int32_t  level;          /* current output level, Q15 */
    int32_t  attack_inc;     /* level increment per sample during attack  */
    int32_t  decay_inc;      /* level decrement per sample during decay   */
    int32_t  sustain_level;  /* sustain level, Q15                        */
    int32_t  release_inc;    /* level decrement per sample during release */
} ADSR;

/* One FM operator */
typedef struct {
    uint32_t phase;       /* current phase, fractional fixed-point */
    uint32_t phase_inc;   /* base phase increment = ratio * note_inc */
    float    ratio;       /* frequency ratio relative to base note  */
    ADSR     env;         /* amplitude envelope                     */
    int32_t  output;      /* most recent output sample, Q15         */
} Operator;



The output field is written each tick and read by any operators that use
this one as a modulator. Because the render loop processes operators in a fixed order
(modulators before carriers), reading output from the previous tick is
correct - there is a one-sample delay in the modulation path, which is inaudible.


ADSR Envelope in Fixed-Point


The envelope level is a Q15 value in the range [0, 32767]. Linear ADSR increments are
the simplest approach that works musically. Each stage adds or subtracts a fixed
per-sample delta until the target level is reached, then transitions to the next stage.



/* Set attack/decay/release times in milliseconds. Sustain is a 0-127 level. */
static void adsr_set(ADSR *a,
                     float attack_ms, float decay_ms,
                     uint8_t sustain_127, float release_ms) {
    /* Convert ms to per-sample increments.
       attack goes from 0 to 32767 over attack_ms milliseconds. */
    float attack_samples  = attack_ms  * 0.001f * SAMPLE_RATE;
    float decay_samples   = decay_ms   * 0.001f * SAMPLE_RATE;
    float release_samples = release_ms * 0.001f * SAMPLE_RATE;

    a->sustain_level = (int32_t)((sustain_127 / 127.0f) * 32767.0f);

    /* Increment = total_change / samples.
       Use 1 as minimum to avoid zero-increment lockup. */
    a->attack_inc  = (int32_t)(32767.0f / attack_samples);
    if (a->attack_inc  < 1) a->attack_inc  = 1;

    a->decay_inc   = (int32_t)((32767.0f - a->sustain_level) / decay_samples);
    if (a->decay_inc   < 1) a->decay_inc   = 1;

    a->release_inc = (int32_t)(a->sustain_level / release_samples);
    if (a->release_inc < 1) a->release_inc = 1;

    a->state = ENV_IDLE;
    a->level = 0;
}

static void adsr_note_on(ADSR *a)  { a->state = ENV_ATTACK; }
static void adsr_note_off(ADSR *a) { a->state = ENV_RELEASE; }

/* Advance envelope one sample, return current level Q15 [0, 32767]. */
static inline int32_t adsr_tick(ADSR *a) {
    switch (a->state) {
    case ENV_ATTACK:
        a->level += a->attack_inc;
        if (a->level >= 32767) {
            a->level = 32767;
            a->state = ENV_DECAY;
        }
        break;
    case ENV_DECAY:
        a->level -= a->decay_inc;
        if (a->level <= a->sustain_level) {
            a->level = a->sustain_level;
            a->state = ENV_SUSTAIN;
        }
        break;
    case ENV_SUSTAIN:
        /* level stays constant until note_off */
        break;
    case ENV_RELEASE:
        a->level -= a->release_inc;
        if (a->level <= 0) {
            a->level = 0;
            a->state = ENV_IDLE;
        }
        break;
    case ENV_IDLE:
        break;
    }
    return a->level;
}


Algorithm Graph: DX7 Algorithm 5


The DX7 has 32 algorithm graphs describing how six operators are connected. Algorithm 5
connects operators in two independent carrier-modulator pairs with self-feedback on one
modulator. With four operators we implement the same structure without feedback:



/* Algorithm: two parallel carrier/modulator pairs
   Op0 -modulates-> Op1 (carrier 1)
   Op2 -modulates-> Op3 (carrier 2)
   Output = Op1 + Op3

   Operator ratios and envelope settings give the two pairs
   different timbres. Mix them for richness. */

typedef struct {
    Operator op[4];
    int32_t  mod_index[2];   /* modulation index Q15, one per pair */
} FMVoice;



The modulation index controls how much of the modulator's output shifts the carrier's
phase. A value of 0 means no modulation (pure sine). A value of 32767 (Q15 = 1.0)
is a full-scale modulation - very metallic. In practice values from 0.1 to 0.6 cover
most useful timbres.


The Per-Sample Render


/* Sine table in flash (see Part 5 for generation). */
extern const int16_t osc_sine_table[TABLE_SIZE];

/* Read one sample from the table with linear interpolation.
   phase is the full 32-bit accumulator. Returns Q15. */
static inline int32_t sine_from_phase(uint32_t phase) {
    uint32_t idx0 = (phase >> PHASE_FRAC_BITS) & (TABLE_SIZE - 1);
    uint32_t idx1 = (idx0 + 1) & (TABLE_SIZE - 1);
    uint32_t frac = (phase >> (PHASE_FRAC_BITS - 10)) & 0x3FF; /* 10-bit frac */
    int32_t  s0   = osc_sine_table[idx0];
    int32_t  s1   = osc_sine_table[idx1];
    /* Linear interpolation: s0 + frac/1024 * (s1 - s0) */
    return s0 + (int32_t)(((s1 - s0) * (int32_t)frac) >> 10);
}

/* Render one sample from the FM voice. Returns Q15 output. */
static inline int16_t fm_voice_tick(FMVoice *v) {
    /* --- Pair 0: Op0 modulates Op1 --- */
    /* Tick modulator Op0 */
    int32_t env0   = adsr_tick(&v->op[0].env);
    int32_t raw0   = sine_from_phase(v->op[0].phase);
    v->op[0].output = (int16_t)((raw0 * env0) >> 15);
    v->op[0].phase += v->op[0].phase_inc;

    /* Tick carrier Op1 with phase modulated by Op0 */
    int32_t env1    = adsr_tick(&v->op[1].env);
    int32_t mod_ph0 = (v->op[0].output * v->mod_index[0]) >> 15;
    int32_t raw1    = sine_from_phase(v->op[1].phase + (uint32_t)(mod_ph0 << 8));
    int32_t out1    = (raw1 * env1) >> 15;
    v->op[1].phase += v->op[1].phase_inc;

    /* --- Pair 1: Op2 modulates Op3 --- */
    int32_t env2    = adsr_tick(&v->op[2].env);
    int32_t raw2    = sine_from_phase(v->op[2].phase);
    v->op[2].output = (int16_t)((raw2 * env2) >> 15);
    v->op[2].phase += v->op[2].phase_inc;

    int32_t env3    = adsr_tick(&v->op[3].env);
    int32_t mod_ph1 = (v->op[2].output * v->mod_index[1]) >> 15;
    int32_t raw3    = sine_from_phase(v->op[3].phase + (uint32_t)(mod_ph1 << 8));
    int32_t out3    = (raw3 * env3) >> 15;
    v->op[3].phase += v->op[3].phase_inc;

    /* Mix carriers: average to avoid overflow */
    int32_t mix = (out1 + out3) >> 1;
    /* Clamp to int16_t range */
    if (mix >  32767) mix =  32767;
    if (mix < -32768) mix = -32768;
    return (int16_t)mix;
}



The phase modulation shift of << 8 maps the Q15 modulator output into
the 32-bit phase accumulator space. Because the phase accumulator represents a full
[0, 2pi] cycle as [0, 2^32), a phase offset of one full cycle is 2^32 counts. The
modulator at full scale (Q15 = 32767) should produce a modulation of roughly 2*pi
radians - which is TABLE_SIZE table steps, which is TABLE_SIZE * 2^PHASE_FRAC_BITS
accumulator counts. With TABLE_SIZE=2048 and PHASE_FRAC_BITS=22, that is
2048 * 2^22 = 2^33 - one bit over 32 bits, so we scale by 2^8 (shift left 8) to map
Q15's 2^15 scale to the required 2^23 range. Adjust this scaling to taste; higher
values increase the maximum modulation index.


Voice Setup: Setting Note Frequency


/* Set the base note frequency for the voice.
   Each operator's phase_inc = note_inc * operator_ratio. */
static void fm_voice_note(FMVoice *v, float freq_hz) {
    /* base phase increment for this frequency */
    float note_inc_f = (freq_hz / SAMPLE_RATE) * TABLE_SIZE
                       * (float)(1u << PHASE_FRAC_BITS);
    uint32_t note_inc = (uint32_t)note_inc_f;
    for (int i = 0; i < 4; i++) {
        v->op[i].phase_inc = (uint32_t)(note_inc * v->op[i].ratio);
    }
}

/* Example preset: electric piano-like timbre */
static void fm_voice_init_epiano(FMVoice *v) {
    /* Operator ratios: carrier at 1x, modulator at 1x (pair 0)
       carrier at 3x, modulator at 3.5x (pair 1 - higher harmonics) */
    v->op[0].ratio = 1.0f;   /* modulator  */
    v->op[1].ratio = 1.0f;   /* carrier    */
    v->op[2].ratio = 3.0f;   /* modulator  */
    v->op[3].ratio = 3.0f;   /* carrier    */

    /* Modulation indices: pair 0 moderate, pair 1 lighter */
    v->mod_index[0] = 12000;   /* ~0.37 in Q15 */
    v->mod_index[1] =  6000;   /* ~0.18 in Q15 */

    /* ADSRs: fast attack, medium decay, sustain at 70%, medium release */
    adsr_set(&v->op[0].env,   8.0f, 200.0f, 90,  300.0f);
    adsr_set(&v->op[1].env,   8.0f, 300.0f, 70,  500.0f);
    adsr_set(&v->op[2].env,  10.0f, 150.0f, 80,  250.0f);
    adsr_set(&v->op[3].env,  10.0f, 250.0f, 60,  400.0f);

    for (int i = 0; i < 4; i++)
        v->op[i].phase = 0;
}


Benchmark: CPU% at 44.1 kHz


Measured on RP2040 at 125 MHz with SysTick (see Part 5), rendering a 256-frame buffer
with one 4-operator FM voice at 440 Hz:



Buffer size:    256 frames
Budget:         725,623 cycles  (256 * 125000000 / 44100)

Per-buffer cost:
  4x adsr_tick:         ~160 cycles total (40 each, integer only)
  4x sine_from_phase:   ~120 cycles total (30 each, table lookup + lerp)
  4x Q15 multiplies:     ~16 cycles total
  Phase increments:       ~8 cycles total
  Overhead (loop, mix):  ~20 cycles

Per-sample total:        ~324 cycles / 256 samples ≈ 1.27 cycles/sample per op
Total for 4 ops:         ~83,000 cycles per buffer
CPU%:                    11.4% of audio budget



That leaves 88.6% of the audio CPU budget for MIDI parsing (core 0), filter (Part 10),
and future polyphony. You could run four simultaneous FM voices and still have margin.


Summary


A 4-operator FM engine on the RP2040 fits comfortably in 12% of the audio CPU budget
at 44.1 kHz. The operator struct combines a phase accumulator, a frequency ratio, and
an ADSR envelope; the algorithm graph is just a fixed order of evaluation with one
operator's output added (scaled) to another's phase. ADSR in Q15 fixed-point avoids
all per-sample soft-float calls - the only floats in the engine appear during note-on
setup. The sine wavetable lives in flash XIP and is accessed sequentially, keeping
cache miss cost low.



Part 7 adds UART MIDI input so the FM engine can be played from an external
controller - handling the MIDI byte stream state machine, running status, and passing
note events from core 0 to core 1 through a lock-free ring buffer.
