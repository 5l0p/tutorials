Drums - Part 6: Per-Channel Effects with Airwindows Pd Externals

The four drum voices built in Parts 2–5 sound clean. Real drum machines and hardware synthesisers colour their output through analogue circuits — input transformers, tape heads, vinyl cutting EQ. The airwindows Pd externals wrap Chris Johnson's open-source DSP algorithms from the airwindows VST collection as Pd signal objects. This article shows how to insert them into the drum voice signal chain.

What You Will Learn

The Airwindows Externals

The airwindows project contains hundreds of high-quality DSP plugins released under the MIT licence. The tARTlite airwindows module wraps six of them as Pd external objects:

ObjectWhat it doesBest used on
mackity~Mackie 1202 input stage emulation — adds warm transformer colourAny voice for gentle analogue warmth
totape9~Tape saturation + head-bump + flutterKick (head bump at low freq), full mix
dubplate2~Vinyl cut EQ — boosts highs, rolls off lows (RIAA-style)Hi-hat, clap (adds vinyl brightness)
zacidlowpass~Aggressive resonant lowpass with meltdown parameterKick (sub shaping), snare (filtering)
chorusensemble~Multi-tap mono chorus — widens stereo imageHi-hat (shimmer), clap (spread)
xhighpass~Transparent highpass with gain and nuke controlsAny voice to remove sub content

Building the Externals

cd modules/airwindows
make

This produces .pd_linux files alongside the source. Load them into Pd by adding the directory to the search path in pd -path /path/to/airwindows/src or by adding a [declare -path ../airwindows/src] object at the top of the drum machine patch.

The Stereo Convention

Every airwindows object has two signal inlets (L, R) and two signal outlets (L, R). Drum voices produce a mono signal. For mono use, feed the same signal to both inlets: 

[outlet~ of voice]
|
[t a a]             ; split to two copies
|     |
L inlet  R inlet
|        |
[mackity~]
|        |
L out   R out
|        |
[+~]    ; sum to mono for the drum machine mix bus
|
[*~ 0.5]

Or, if the drum machine outputs stereo, keep L and R separate from this point forward.

Parameter Format

All airwindows parameters are normalised 0.0–1.0 floats. Parameters are sent as named messages to the left inlet: 

[intrim 0.6(  →  [mackity~]     ; set intrim to 60%
[output 0.8(  →  [mackity~]     ; set output level to 80%

To send from Lua, use the message receiver the Pd patch exposes: 

pd.send("kick-mackity-intrim", 0.6)
pd.send("kick-mackity-output", 0.8)

In Pd, wire a receiver to the message and forward it: 

[r $0-eff-intrim]
|
[prepend intrim]    ; adds the parameter name prefix
|
[mackity~]          ; routes to mackity~'s intrim handler

mackity~ — Input Stage Colour

The Mackie 1202 is known for a transformer-coupled input stage that adds gentle harmonic saturation and a slight low-mid emphasis at moderate gain. Use it as the first insert on any voice for analogue warmth: 

[voice outlet~]
|
[t a a]
|     |
[mackity~]
|         |
[+~][*~ 0.5]   ; sum to mono

; Parameters:
; intrim (0–1): input gain. Above 0.5 = saturation increases.
; outpad (0–1): output level. Use 0.7–0.8 to compensate for gain.

For a kick drum, intrim=0.55 and outpad=0.75 adds warmth without distortion. intrim=0.8 gives audible transformer saturation.

totape9~ — Tape Saturation

Totape9 emulates a tape machine: saturation from high input levels, a head-bump low-frequency boost, and optional wow/flutter. The headbmp and headfrq parameters control the low-frequency boost that makes tape-recorded kick drums sound particularly large: 

[voice outlet~]
|
[t a a]
|     |
[totape9~]   ; 9 parameters, all 0–1
|          |
[+~][*~ 0.5]

; Key parameters for kick drum:
; input  (0–1):  drive level. 0.5 = nominal, 0.7+ = saturation
; headbmp(0–1):  head-bump depth. 0.3–0.6 for sub weight
; headfrq(0–1):  head-bump frequency. 0.1–0.3 = low freq (50–80 Hz)
; output (0–1):  output gain

; For hi-hat: low input, minimal headbmp, tilt slightly bright
; input=0.4, tilt=0.6 (bright), headbmp=0.0, output=0.7

dubplate2~ — Vinyl Cut EQ

Dubplate emulates the RIAA equalization applied when cutting a vinyl record — a large bass cut and treble boost that gives records their characteristic sound. On a hi-hat or clap, this adds an airy brightness and a slightly retro character: 

; Parameters: input, treble, bass
; input (0–1): input gain
; treble(0–1): high-frequency boost (0.5 = neutral, 1.0 = maximum boost)
; bass  (0–1): low-frequency cut (0.5 = neutral, 0.0 = maximum cut)

; For hi-hat: treble=0.7, bass=0.3 → boosted highs, reduced lows = vinyl brightness
; For clap:   treble=0.6, bass=0.4 → moderate vinyl colour

zacidlowpass~ — Resonant Lowpass

The zacid lowpass is an aggressive filter with a meltdown parameter that pushes the resonance past self-oscillation into a saturated feedback mode. This is the 303-style acid filter that can also reshape the character of a kick or snare: 

; Parameters: cutoff, over, under, meltdown, drive, output
; cutoff   (0–1): filter cutoff frequency (log mapped)
; over     (0–1): resonance above cutoff
; under    (0–1): resonance below cutoff
; meltdown (0–1): self-oscillation/saturation (above 0.7 = unstable)
; drive    (0–1): input saturation
; output   (0–1): output level

; For kick sub shaping:
; cutoff=0.3, over=0.2, under=0.1, meltdown=0.0, drive=0.4, output=0.7
; → rolls off highs, leaves sub intact with gentle resonance

; For resonant effect (techno filter sweep):
; cutoff varies 0–0.6, over=0.5, meltdown=0.3

This external is particularly useful for automating the cutoff parameter in real time from a Lua sequence, creating filter sweep effects on the kick or snare.

chorusensemble~ — Stereo Widening

A multi-tap chorus that widens a mono signal to a convincing stereo spread. Use on hi-hat to add shimmer, or on clap to spread the snap across the stereo field: 

; Parameters: speed, range, drywet
; speed  (0–1): LFO rate. 0.2–0.4 = slow shimmer, 0.7+ = fast warble
; range  (0–1): modulation depth. 0.1–0.3 = subtle, 0.5+ = obvious chorus
; drywet (0–1): wet mix. 0.3–0.6 = blend, 1.0 = full chorus

; For hi-hat shimmer:
; speed=0.25, range=0.15, drywet=0.4

; For clap spread:
; speed=0.1, range=0.25, drywet=0.5

Bypass and Enable

Add a bypass control to each effect. In Pd, use [spigot] or a wet/dry blend: 

[voice signal]
|
[t a a]
|      |
[effect~]  dry signal
|      |
wet    |
|      |
[*~ $wet]  [*~ $dry]
|                |
[+~]   ← bypass blend
|
output

When $wet=0 and $dry=1 the effect is bypassed. A continuous blend is more interesting than a hard bypass.

Per-Channel Effects Chain in Pd

; effects_chain.pd — reusable effects chain subpatch
; Inlet: audio signal
; Outlet: processed signal
; Receivers: $0-mackity-intrim, $0-totape-input, $0-zacid-cutoff, etc.

[inlet~]
|
[t a a]
|     |
[mackity~]          ; warm transformer input
|         |
[+~][*~ 0.5]        ; collapse to mono (or keep stereo)
|
[zacidlowpass~]     ; filter shaping
|         |
[+~][*~ 0.5]
|
[totape9~]          ; tape saturation
|         |
[+~][*~ 0.5]
|
[*~ $outLevel]
|
[outlet~]

Controlling Effects from Lua

-- In module.lua:
local effects = {
    kick  = { mackity_intrim=0.55, totape_input=0.5, totape_headbmp=0.3 },
    snare = { mackity_intrim=0.5, zacid_cutoff=0.8 },
    hihat = { dubplate_treble=0.7, chorus_drywet=0.35 },
    clap  = { dubplate_treble=0.6, chorus_drywet=0.45 },
}

local function applyEffects(voice, params)
    for param, value in pairs(params) do
        pd.send(voice .. "-eff-" .. param, value)
    end
end

applyEffects("kick",  effects.kick)
applyEffects("hihat", effects.hihat)

Adding a New Airwindows Plugin

If you want to wrap an airwindows plugin not currently included, the process from the airwindows README is:

  1. Find the plugin folder: git ls-tree -d HEAD plugins/LinuxVST/src/ | grep -i keyword
  2. Add it to the sparse checkout: git sparse-checkout add plugins/LinuxVST/src/PluginName
  3. Check its parameters: grep 'kParam' vendor/airwindows/.../PluginName.h
  4. Copy an existing wrapper (e.g. src/mackity~.cpp) and replace all occurrences of "mackity" with the new plugin name, adjust the parameter list.
  5. Add a Makefile block for the new external.
  6. Run make.

Summary

The airwindows Pd externals add analogue character to the clean synthesised drum voices: mackity~ for transformer warmth on any voice, totape9~ for tape head-bump on the kick, dubplate2~ for vinyl brightness on hi-hat and clap, zacidlowpass~ for resonant filter sweeps, and chorusensemble~ for stereo spread. All parameters are normalised 0–1, sent as named messages to the object's left inlet. Wrap each effect in a dry/wet blend for seamless bypass. Control from Lua by sending to $0-eff- namespaced receivers.
