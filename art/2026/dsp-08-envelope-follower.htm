DSP - Part 8: Envelope Follower & Parameter Smoothing

An envelope follower extracts the slowly-varying amplitude contour from an audio signal. It is used as a building block in compressors, gates, auto-wah effects, and ducking systems. Related is the one-pole smoother, which eliminates the clicks (zipper noise) that occur when a parameter changes abruptly. This article covers both: the math, the C implementation, and the timing formulas that convert milliseconds into filter coefficients.

The One-Pole Lowpass as an Envelope Smoother

The simplest envelope follower is a first-order (one-pole) lowpass filter applied to the rectified input signal. The one-pole filter has a single coefficient c that determines how quickly it follows changes: 

output[n] = c * input[n] + (1 - c) * output[n-1]

This is an exponentially-weighted running average. When c is close to 1, the output tracks the input closely (fast response). When c is close to 0, the output is sluggish (slow response).

Converting a desired time constant in milliseconds to a coefficient: 

#include <math.h>

#define SAMPLE_RATE 48000

/* time_ms: time constant in milliseconds.
   Returns the one-pole coefficient for that time constant. */
float time_to_coeff(float time_ms) {
    /* One-pole response is -60 dB (exp(-1)) in 'time_ms' milliseconds. */
    float time_s = time_ms / 1000.0f;
    return expf(-1.0f / (time_s * (float)SAMPLE_RATE));
}

/* Equivalently: exp(-2*pi*fc/fs) where fc = 1/(2*pi*time_s) */

The coefficient is derived from the 1/e point of an exponential decay. At the time constant the envelope has decayed to 1/e ≈ 36.8% of its starting value - equivalent to -8.7 dB. This is the standard definition used in all analogue and digital compressor attack/release specifications.

Peak Envelope Follower

A peak follower rectifies the input (takes the absolute value) and then applies the one-pole smoother. Separate attack and release coefficients let the follower respond quickly to sudden increases (short attack) while falling back slowly (long release): 

typedef struct {
    float  attack_coeff;   /* close to 1.0 = slow, close to 0.0 = fast */
    float  release_coeff;
    float  env;            /* current envelope value */
} PeakFollower;

void peak_init(PeakFollower *p, float attack_ms, float release_ms) {
    p->attack_coeff  = time_to_coeff(attack_ms);
    p->release_coeff = time_to_coeff(release_ms);
    p->env = 0.0f;
}

float peak_process(PeakFollower *p, float x) {
    float abs_x = fabsf(x);
    float c = (abs_x > p->env) ? p->attack_coeff : p->release_coeff;
    p->env = c * p->env + (1.0f - c) * abs_x;
    return p->env;
}

void peak_render(PeakFollower *p, const float *in,
                 float *env_out, int n) {
    for (int i = 0; i < n; i++)
        env_out[i] = peak_process(p, in[i]);
}

Some implementations swap the coefficient direction: they apply the attack coefficient when the input is rising and the release coefficient when it is falling. Others use a level comparator with hysteresis. The formula above is correct for the standard analogue-style attack/release model.

RMS Envelope Follower

Peak detection is fast but susceptible to short transients. RMS (root mean square) detection averages the squared signal over a window, giving a measure that corresponds better to perceived loudness. The steps are: square, low-pass filter, square root. 

typedef struct {
    float  coeff;   /* from time_to_coeff */
    float  state;   /* running sum of x^2 (filtered) */
} RMSFollower;

void rms_init(RMSFollower *r, float window_ms) {
    r->coeff = time_to_coeff(window_ms);
    r->state = 0.0f;
}

float rms_process(RMSFollower *r, float x) {
    /* One-pole lowpass on x^2 */
    r->state = r->coeff * r->state + (1.0f - r->coeff) * x * x;
    return sqrtf(r->state);
}

Typical window for loudness measurement: 50–300 ms. Short windows (5–10 ms) give faster response but more variance. The ITU-R BS.1770 loudness standard uses a 400 ms K-weighted RMS, but for dynamics processing (compressors) 30–100 ms is common.

Converting Envelope to dB

A compressor operates on the level in dB. Converting from the linear envelope to dB: 

float lin_to_db(float x) {
    if (x <= 1e-10f) return -200.0f;
    return 20.0f * log10f(x);
}

float db_to_lin(float db) {
    return powf(10.0f, db / 20.0f);
}

One-Pole Parameter Smoother

When a parameter (filter cutoff, gain, pan) changes abruptly - for example when the user moves a slider - the jump in the audio signal causes a click or zipper noise audible as a high-frequency burst. The fix is to interpolate the parameter smoothly from its old value to its new value using a one-pole lowpass. 

typedef struct {
    float value;   /* current smoothed value */
    float coeff;   /* one-pole coefficient */
} Smoother;

void smoother_init(Smoother *s, float initial_value, float time_ms) {
    s->value = initial_value;
    s->coeff = time_to_coeff(time_ms);
}

/* Call once per sample to advance the smoother toward target. */
float smoother_tick(Smoother *s, float target) {
    s->value = s->coeff * s->value + (1.0f - s->coeff) * target;
    return s->value;
}

A smoothing time of 5–20 ms is inaudible for most parameters. For gain automation a 1–5 ms smoother is sufficient. For filter cutoff sweeps, 1 ms prevents clicks but lets the envelope drive the filter at useful rates.

Block-Level Parameter Smoothing

For efficiency, evaluate the smoother once per block and linearly interpolate within the block rather than calling smoother_tick 48000 times per second: 

void apply_gain_smoothed(float *buf, int n,
                         float gain_start, float gain_end) {
    float step = (gain_end - gain_start) / (float)n;
    float g    = gain_start;
    for (int i = 0; i < n; i++, g += step)
        buf[i] *= g;
}

/* In the callback: */
void audio_callback(AudioEngine *e, float *out, int n) {
    float gain_target = atomic_load(&e->gain_param);
    float gain_start  = smoother_tick(&e->gain_smooth, gain_target);
    float gain_end    = smoother_tick(&e->gain_smooth, gain_target);
    apply_gain_smoothed(out, n, gain_start, gain_end);
}

Gain smoother          1-5 ms      1-5 ms

Summary

An envelope follower uses a one-pole filter with separate coefficients for attack and release. The coefficient for a given time constant τ in milliseconds is exp(-1 / (sr · τ / 1000)). Peak detection follows the signal immediately on attack and decays on release. RMS detection squares the input, filters, then square-roots for a perceptually smoother result. The same one-pole structure smooths parameter changes — interpolate from current to target over a block to eliminate zipper noise.
