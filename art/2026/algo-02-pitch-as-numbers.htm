Algorithmic Composition - Part 2: Pitch as Numbers

Every algorithmic composition technique in this series outputs pitches as numbers. Before any algorithm runs, you need a clear answer to three questions: what number represents middle C, how do you convert a note number to a frequency, and given an arbitrary integer from an algorithm, how do you snap it to the nearest musically valid pitch?

MIDI solves the first two questions with a single formula. A Lua table solves the third. Once these are in place, every algorithm in Parts 3–10 can output integers freely, and the quantise function maps them to whatever scale the current piece uses — without the algorithm needing to know anything about music theory.

MIDI Note Numbers

MIDI assigns an integer from 0 to 127 to every pitch. Note 69 is A4 (concert A, 440 Hz). Note 60 is C4 (middle C, one octave below). Notes increase by one semitone per integer: note 61 is C#4, note 62 is D4, and so on. One octave is always 12 semitones, so note 72 is C5 (one octave above middle C) and note 48 is C3.

This encoding has two useful properties for algorithmic composition. First, arithmetic on note numbers produces musically meaningful results: adding 7 to any note raises it a perfect fifth; adding 12 raises it an octave. Second, the range 0–127 spans just over ten octaves, more than any acoustic instrument, so algorithms never need to worry about overflow.

Equal Temperament and Frequency

Twelve-tone equal temperament divides the octave into 12 equal semitones. "Equal" means that the frequency ratio between any two adjacent semitones is identical: the twelfth root of 2, approximately 1.05946. The formula for converting MIDI note number n to frequency in Hz is: 

f = 440 × 2 ^ ((n - 69) / 12)

Verify: n = 69 gives 440 × 2^0 = 440 Hz. n = 57 (A3, one octave below) gives 440 × 2^(-1) = 220 Hz. n = 60 (C4) gives 440 × 2^(-9/12) ≈ 261.63 Hz.

In Lua: 

local function midi_to_hz(n)
    return 440.0 * 2 ^ ((n - 69) / 12.0)
end

print(string.format("C4 = %.2f Hz", midi_to_hz(60)))   -- 261.63 Hz
print(string.format("A4 = %.2f Hz", midi_to_hz(69)))   -- 440.00 Hz
print(string.format("A3 = %.2f Hz", midi_to_hz(57)))   -- 220.00 Hz

Pure Data's [mtof] object performs the same conversion on the Pd side. When an algorithm in Lua produces a MIDI note number and sends it to Pd, the Pd patch runs it through [mtof] to get the oscillator frequency.

Scale Tables

A scale is a subset of the 12 chromatic pitches. The most compact representation is an array of semitone offsets from the root. The major scale, for example, has intervals of 2, 2, 1, 2, 2, 2, 1 semitones between consecutive degrees. Starting from 0, the accumulated offsets are: 

major = {0, 2, 4, 5, 7, 9, 11}

To play a C major scale starting at C4 (MIDI 60), add each offset to the root: 60, 62, 64, 65, 67, 69, 71. A full collection in Lua: 

local scales = {
    major      = {0, 2, 4, 5, 7, 9, 11},
    minor      = {0, 2, 3, 5, 7, 8, 10},  -- natural minor
    dorian     = {0, 2, 3, 5, 7, 9, 10},
    phrygian   = {0, 1, 3, 5, 7, 8, 10},
    lydian     = {0, 2, 4, 6, 7, 9, 11},
    mixolydian = {0, 2, 4, 5, 7, 9, 10},
    pentatonic = {0, 2, 4, 7, 9},         -- major pentatonic
    whole_tone = {0, 2, 4, 6, 8, 10},
    octatonic  = {0, 2, 3, 5, 6, 8, 9, 11},  -- diminished/octatonic
}

The root is a separate parameter, not baked into the scale table. To change key, pass a different root to the quantise function — the scale table stays the same. C major uses root 60; G major uses root 67; Eb minor uses root 63 with scales.minor.

The Quantise Function

An algorithm — a random walk, a Markov chain, an L-system — outputs integers that may not fall on any scale degree. The quantise function takes any MIDI note number (or even a float, rounded to the nearest integer) and returns the nearest pitch that is in the given scale and root.

The algorithm: reduce the input pitch to a pitch class relative to the root (take it modulo 12), then find the scale degree whose offset is nearest to that pitch class (handling the wrap-around between 11 and 0 correctly), then reconstruct an absolute MIDI note in the same octave as the input. 

-- quantise: map any MIDI note to the nearest in-scale pitch
-- pitch: any integer (MIDI note number)
-- root:  MIDI root note (e.g. 60 = C4)
-- scale: array of semitone offsets (e.g. scales.major)
local function quantise(pitch, root, scale)
    -- pitch class of input, relative to root (0-11)
    local pc = (pitch - root) % 12

    -- find nearest scale degree
    local best_offset, best_dist = 0, 13
    for _, offset in ipairs(scale) do
        -- circular distance on pitch-class ring (mod 12)
        local d = math.abs(pc - offset)
        if d > 6 then d = 12 - d end
        if d < best_dist then
            best_dist = d
            best_offset = offset
        end
    end

    -- reconstruct absolute MIDI note
    -- octave of input, measured from root
    local octave = math.floor((pitch - root) / 12)
    return root + octave * 12 + best_offset
end

Test it: 

local scales = {major = {0, 2, 4, 5, 7, 9, 11}, pentatonic = {0, 2, 4, 7, 9}}

-- C4 major, root C4 (60)
print(quantise(60, 60, scales.major))  -- 60 (C4, already in scale)
print(quantise(61, 60, scales.major))  -- 62 (D4: nearest to C#4 is D4)
print(quantise(66, 60, scales.major))  -- 67 (G4: nearest to F#4 is G4)
print(quantise(66, 60, scales.pentatonic)) -- 64 (E4: nearest to F#4 in pentatonic)

-- G major, root G4 (67)
print(quantise(61, 67, scales.major))  -- 62 (D? No: relative to G, 61 is... let's verify)
-- 61 - 67 = -6, mod 12 = 6. G major offsets: {0,2,4,5,7,9,11}.
-- Nearest to 6: 5 (F#) or 7 (G). Distance 1 each; 5 wins on tie.
-- Result: 67 + 0*12 + 5 = 72... no, octave = floor(-6/12) = -1
-- 67 + (-1)*12 + 5 = 67 - 12 + 5 = 60. So result is 60 (C4 is F# relative to G? No.)
-- C is not in G major; nearest in G major to MIDI 61 (C#4) is 62 (D4, 2 semitones up from G root = D).
-- The octave=-1 calculation places us one octave below. Let's verify manually:
-- pitch=61, root=67. pc=(61-67)%12 = (-6)%12 = 6.
-- G major: 0=G, 2=A, 4=B, 5=C, 7=D, 9=E, 11=F#. Nearest to pc=6: 5(C, d=1) or 7(D, d=1). Tie, 5 wins.
-- octave=floor((61-67)/12)=floor(-0.5)=-1.
-- result=67+(-1)*12+5=60. MIDI 60=C4. C is in G major? Yes! C is the 4th of G. Correct.

The quantise function is the bridge between every generative algorithm in this series and the scale and key of the current piece. Change root to modulate; change scale to switch modes. Neither change requires touching the algorithm.

Sending Pitches to Pure Data

In the drum machine architecture from Series 9, Lua sent parameter values to Pd via named receivers. The same pattern works here. In Pd, create a receiver: 

[receive algo-pitch]
|
[mtof]
|
[osc~]

In Lua, send a MIDI note number: 

-- send MIDI note 64 (E4) to the Pd receiver named "algo-pitch"
pd.send("algo-pitch", 64)

The Pd [mtof] object converts 64 to 329.63 Hz, which drives the oscillator. The Lua side never needs to know frequencies — only note numbers.

A Complete Pitch Utility Module

Save this as pitch.lua. Every article from Part 3 onwards requires it. 

-- pitch.lua — MIDI note utilities and scale definitions

local M = {}

M.scales = {
    major      = {0, 2, 4, 5, 7, 9, 11},
    minor      = {0, 2, 3, 5, 7, 8, 10},
    dorian     = {0, 2, 3, 5, 7, 9, 10},
    phrygian   = {0, 1, 3, 5, 7, 8, 10},
    lydian     = {0, 2, 4, 6, 7, 9, 11},
    mixolydian = {0, 2, 4, 5, 7, 9, 10},
    pentatonic = {0, 2, 4, 7, 9},
    whole_tone = {0, 2, 4, 6, 8, 10},
    octatonic  = {0, 2, 3, 5, 6, 8, 9, 11},
}

function M.midi_to_hz(n)
    return 440.0 * 2 ^ ((n - 69) / 12.0)
end

function M.quantise(pitch, root, scale)
    local pc = (pitch - root) % 12
    local best_offset, best_dist = 0, 13
    for _, offset in ipairs(scale) do
        local d = math.abs(pc - offset)
        if d > 6 then d = 12 - d end
        if d < best_dist then
            best_dist = d
            best_offset = offset
        end
    end
    local octave = math.floor((pitch - root) / 12)
    return root + octave * 12 + best_offset
end

-- Note name for a MIDI number (e.g. 60 → "C4")
local note_names = {"C","C#","D","D#","E","F","F#","G","G#","A","A#","B"}
function M.name(n)
    return note_names[(n % 12) + 1] .. tostring(math.floor(n / 12) - 1)
end

return M

Run a quick test: 

$ lua -e "
local p = require('pitch')
for _, n in ipairs({60, 62, 64, 65, 67, 69, 71, 72}) do
    io.write(p.name(n) .. '  ')
end
print()
"
C4  D4  E4  F4  G4  A4  B4  C5

Summary

MIDI note numbers provide a clean integer encoding of pitch: 69 is A4 at 440 Hz, each semitone is one unit, each octave is 12. Equal temperament converts any note number to Hz with 440 × 2^((n-69)/12); on the Pd side, [mtof] does the same. A scale is an array of semitone offsets from a root; the quantise function maps any integer to the nearest in-scale MIDI note. The pitch.lua module provides all of this for the algorithms that follow.
