libpd - Part 9: libpd in openFrameworks: ofxPd
2026-06-01

ofxPd is an openFrameworks addon that wraps libpd in an OF-idiomatic API, integrating Pure Data's DSP engine with the OF audio stream, event system, and app lifecycle. If you are already building with openFrameworks, ofxPd is the fastest path to embedded Pd audio. This article covers the addon setup, audio routing, message exchange, and subscriber pattern.

What You Will Learn


How to install ofxPd via the OF addon manager and configure the CMake build

Installing ofxPd

Clone ofxPd into your openFrameworks addons directory:


cd openframeworks/addons
git clone https://github.com/danomatika/ofxPd.git
cd ofxPd
git submodule update --init --recursive


Add ofxPd to your project. In the OF Project Generator, select it from the addons list. Alternatively, edit addons.make manually:


ofxPd


On Linux, add to your config.make:


PROJECT_LDFLAGS += -lpthread


App Skeleton

An ofxPd app needs three OF audio callbacks wired to the pd object:


/* ofApp.h */
#pragma once
#include "ofMain.h"
#include "ofxPd.h"

class ofApp : public ofBaseApp, public PdReceiver {
public:
    void setup();
    void update();
    void draw();
    void audioIn(ofSoundBuffer &input);
    void audioOut(ofSoundBuffer &output);

    /* PdReceiver callbacks */
    void receiveBang(const std::string &dest);
    void receiveFloat(const std::string &dest, float value);
    void receiveSymbol(const std::string &dest,
                       const std::string &symbol);

    ofxPd pd;
    ofSoundStream soundStream;
};


setup()


void ofApp::setup() {
    int sampleRate   = 44100;
    int bufferSize   = 512;
    int inChannels   = 0;
    int outChannels  = 2;

    /* Initialise ofxPd */
    if (!pd.init(outChannels, inChannels, sampleRate, bufferSize)) {
        ofLogError("ofApp") << "Could not initialise libpd";
        ofExit();
    }

    /* Add search paths for abstractions */
    pd.addToSearchPath(ofToDataPath("pd/abstractions"));

    /* Register this object as the message receiver */
    pd.setReceiver(this);

    /* Subscribe to named receivers */
    pd.subscribe("rms_level");
    pd.subscribe("beat_event");

    /* Open the patch (lives in bin/data/pd/) */
    Patch patch = pd.openPatch("synth.pd", ofToDataPath("pd"));
    if (!patch.isValid()) {
        ofLogError("ofApp") << "Could not open synth.pd";
    }

    /* Set up the OF sound stream */
    ofSoundStreamSettings settings;
    settings.setOutListener(this);
    settings.setInListener(this);
    settings.sampleRate = sampleRate;
    settings.bufferSize = bufferSize;
    settings.numOutputChannels = outChannels;
    settings.numInputChannels  = inChannels;
    soundStream.setup(settings);
}


Audio Routing


void ofApp::audioOut(ofSoundBuffer &output) {
    /* ofSoundBuffer provides interleaved float samples */
    pd.audioOut(output.getBuffer().data(),
                output.getNumFrames(),
                output.getNumChannels());
}

void ofApp::audioIn(ofSoundBuffer &input) {
    pd.audioIn(input.getBuffer().data(),
               input.getNumFrames(),
               input.getNumChannels());
}


ofxPd's audioOut and audioIn calls map directly to libpd_process_float under the hood, handling the tick arithmetic and buffer interleaving for you.

Sending Messages to the Patch


void ofApp::update() {
    float mouseX = ofGetMouseX() / (float)ofGetWidth();

    /* Send float */
    pd.sendFloat("cutoff", mouseX * 4000.0f);

    /* Send bang */
    pd.sendBang("trigger");

    /* Send symbol */
    pd.sendSymbol("waveform", "square");

    /* Send list */
    pd.startMessage();
    pd.addFloat(440.0f);
    pd.addFloat(0.8f);
    pd.finishList("osc_params");

    /* Typed message */
    pd.startMessage();
    pd.addFloat(1.0f);
    pd.finishMessage("dsp", "state");
}


Receiving Messages from the Patch

The PdReceiver interface (implemented by your ofApp) provides the callbacks:


void ofApp::receiveBang(const std::string &dest) {
    ofLogNotice("ofApp") << "Bang from: " << dest;
}

void ofApp::receiveFloat(const std::string &dest, float value) {
    if (dest == "rms_level") {
        levelMeter = value; /* update display */
    }
}

void ofApp::receiveSymbol(const std::string &dest,
                           const std::string &symbol) {
    ofLogNotice("ofApp") << dest << " -> " << symbol;
}


Unlike raw libpd, ofxPd safely marshals hook callbacks from the audio thread to the main thread. You can safely update OF objects and draw state from these methods.

class ofApp : public ofBaseApp,

    /* ... */
    void receiveNoteOn(const int channel,
                       const int pitch,
                       const int velocity);
};

void ofApp::receiveNoteOn(const int channel,
                           const int pitch,
                           const int velocity)
{
    ofLogNotice() << "NoteOn: " << pitch << " @ " << velocity;
}

/* In setup(): */
pd.setMidiReceiver(this);

/* Send MIDI to the patch: */
pd.sendNoteOn(0 /*channel*/, 60 /*pitch*/, 100 /*velocity*/);
pd.sendControlChange(0, 74, 64);


Multiple Patches


Patch patchA = pd.openPatch("drums.pd",  ofToDataPath("pd"));
Patch patchB = pd.openPatch("synth.pd",  ofToDataPath("pd"));

/* Both patches mix their output automatically */

/* Close one */
pd.closePatch(patchA);


All open patches share one libpd instance. Their DSP graphs are merged - signals and messages from different patches mix as if they were in the same patch. Use unique receiver names to avoid conflicts.

Live Reloading a Patch


void ofApp::keyPressed(int key) {
    if (key == 'r') {
        pd.closePatch(currentPatch);
        currentPatch = pd.openPatch("synth.pd", ofToDataPath("pd"));
    }
}


This is one of the most productive workflows: edit the patch in the Pd GUI (or a text editor), press R in your OF app, and hear the changes immediately without recompiling.

Summary


ofxPd wraps libpd in openFrameworks conventions: initialise with pd.init()
in setup(), open a patch with pd.openPatch(), and route audio
by calling pd.audioIn()/pd.audioOut() from the OF sound callbacks.
Register a subscriber class that inherits from ofxPdListener to receive
messages from the patch. For hot-reloading, close the current patch and open a fresh copy
on a keypress — the audio stream continues uninterrupted.
