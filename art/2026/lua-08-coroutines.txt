Lua - Part 8: Coroutines and C
2026-06-01


A game cutscene needs to say "move character to waypoint A, wait 2 seconds, play dialogue,
wait for player input, fade to black". Written as a flat state machine, that is a dozen
states, a dozen transition conditions, and code spread across multiple update functions.
Written as a Lua coroutine, it is eight lines that read exactly like the description.
Coroutines are Lua's answer to sequential scripting in an event-driven host: a coroutine
runs until it explicitly yields, passes a value back to the C caller, and resumes from
exactly where it stopped when the C side is ready. This article covers the complete
mechanism: creating coroutines, resuming from C with lua_resume, passing
values through yield in both directions, the status lifecycle, and a complete C game
loop driving a Lua sequence script.


What You Will Learn


The coroutine status cycle: suspended, running, dead, normal.
How coroutine.create, coroutine.resume, and coroutine.yield work from Lua.
How to resume a coroutine from C with lua_resume.
How to pass values into and out of yield in both directions.
A scripted sequence machine: C timer loop driving a Lua cutscene coroutine.
A cooperative task scheduler written in Lua and driven from C.


Coroutines from the Lua Side First


Before driving coroutines from C, understand the Lua API, because lua_resume
is a direct mapping of it. A coroutine is a thread-like object created with
coroutine.create(f) - it does not start running immediately. It is resumed
with coroutine.resume(co, ...), which starts or continues execution. When the
coroutine calls coroutine.yield(...), execution returns to the caller of
resume. The values passed to yield become the return values of
resume. The values passed to the next resume become the return
values of yield.



-- Lua demonstration of the value-passing contract
local function counter(start)
    local n = start
    while true do
        local step = coroutine.yield(n)   -- yield n, receive next step
        n = n + (step or 1)
    end
end

local co = coroutine.create(counter)

local ok, val = coroutine.resume(co, 10) -- start with argument 10
print(ok, val)  -- true   10             -- first yield

ok, val = coroutine.resume(co, 5)        -- resume, pass step=5
print(ok, val)  -- true   15             -- 10 + 5

ok, val = coroutine.resume(co, 3)        -- resume, pass step=3
print(ok, val)  -- true   18             -- 15 + 3



Status values returned by coroutine.status(co):



"suspended" - created but not yet started, or currently yielded. Can be resumed.
"running" - currently executing (you are inside it right now).
"dead" - the function returned or raised an error. Cannot be resumed.
"normal" - active but not running (it resumed another coroutine).


lua_resume: Driving Coroutines from C


lua_resume is the C equivalent of coroutine.resume. It takes
a coroutine thread (a separate lua_State *), pushes arguments, and starts
or continues execution. When the coroutine yields, lua_resume returns
LUA_YIELD with the yielded values on the coroutine's stack. When the
coroutine's function returns, lua_resume returns LUA_OK.



/* Creating and resuming a coroutine from C */

/* Step 1: load a Lua script that defines the coroutine function */
luaL_dostring(L,
    "function sequence()\n"
    "    coroutine.yield('move_to_waypoint_A')\n"
    "    coroutine.yield('play_dialogue')\n"
    "    coroutine.yield('fade_to_black')\n"
    "    return 'done'\n"
    "end\n"
);

/* Step 2: get the function and create a coroutine thread */
lua_getglobal(L, "sequence");
/* lua_newthread creates a new coroutine thread sharing the same Lua state */
lua_State *co = lua_newthread(L);    /* stack (L):  [..., co_thread] */
                                     /* stack (co): []               */
lua_xmove(L, co, 1);                 /* move 'sequence' fn to co's stack */
                                     /* stack (L):  [...]            */
                                     /* stack (co): [sequence]       */

/* Step 3: resume loop */
int nresults;
int status;

/* First resume - 0 extra arguments (besides the function itself) */
status = lua_resume(co, L, 0, &nresults);
/* stack (co): [yielded_value]  nresults=1 */

while (status == LUA_YIELD) {
    if (nresults > 0 && lua_isstring(co, -1)) {
        printf("Event: %s\n", lua_tostring(co, -1));
    }
    lua_pop(co, nresults);  /* pop yielded values before next resume */

    /* Resume again - 0 values passed back to yield's return */
    status = lua_resume(co, L, 0, &nresults);
}

if (status == LUA_OK) {
    if (nresults > 0) {
        printf("Sequence finished: %s\n", lua_tostring(co, -1));
        lua_pop(co, nresults);
    }
} else {
    /* LUA_ERRRUN: error in the coroutine */
    fprintf(stderr, "Coroutine error: %s\n", lua_tostring(co, -1));
    lua_pop(co, 1);
}



The fourth argument to lua_resume (the int *nresults pointer)
was added in Lua 5.4. It receives the number of values the coroutine yielded or returned.
In Lua 5.3 and earlier, you used lua_gettop(co) after the call instead.



lua_newthread vs a new lua_State: lua_newthread creates a new
execution thread that shares the same global state (globals, registry, loaded modules)
as the parent lua_State. It is not a new interpreter instance. This is
correct for coroutines. A separate luaL_newstate() would give complete
isolation but no shared globals.


Passing Values Through Yield


Values flow in two directions. Values pushed before lua_resume are received
as arguments (on first resume) or as the return values of coroutine.yield
(on subsequent resumes). Values pushed by the coroutine before yielding are received by
the C caller through nresults:



/* Lua function that communicates bidirectionally */
luaL_dostring(L,
    "function bidirectional(initial)\n"
    "    local x = initial\n"
    "    while true do\n"
    "        local delta = coroutine.yield(x * x)  -- yield x^2, receive delta\n"
    "        if delta == nil then break end\n"
    "        x = x + delta\n"
    "    end\n"
    "    return x\n"
    "end\n"
);

lua_getglobal(L, "bidirectional");
lua_State *co = lua_newthread(L);
lua_xmove(L, co, 1);

int nresults;
/* First resume: pass initial value 3 */
lua_pushinteger(co, 3);
int status = lua_resume(co, L, 1, &nresults);
/* co yielded 9 (3^2) */
printf("yielded: %lld\n", (long long)lua_tointeger(co, -1));  /* 9 */
lua_pop(co, nresults);

/* Second resume: pass delta = 2, coroutine will compute (3+2)^2 = 25 */
lua_pushinteger(co, 2);
status = lua_resume(co, L, 1, &nresults);
printf("yielded: %lld\n", (long long)lua_tointeger(co, -1));  /* 25 */
lua_pop(co, nresults);

/* Third resume: pass nil to break the loop */
lua_pushnil(co);
status = lua_resume(co, L, 1, &nresults);
if (status == LUA_OK)
    printf("returned: %lld\n", (long long)lua_tointeger(co, -1)); /* 5 */
lua_pop(co, nresults);


Complete Example: C Game Loop Driving a Lua Sequence


A C game loop that drives a Lua cutscene coroutine, yielding event strings that the C
engine processes:



/* sequence.c - C game loop + Lua coroutine sequence */
#include <lua.h>
#include <lualib.h>
#include <lauxlib.h>
#include <stdio.h>
#include <string.h>
#include <unistd.h>   /* for usleep (POSIX) */

/* Simulated game engine actions */
static void engine_move(const char *target)   { printf("[ENGINE] move to %s\n",  target); }
static void engine_say(const char *text)      { printf("[ENGINE] say: %s\n",     text);   }
static void engine_wait(int ms)               { printf("[ENGINE] wait %dms\n",   ms); usleep(ms * 1000); }
static void engine_fade(const char *dir)      { printf("[ENGINE] fade %s\n",     dir);    }

/* Dispatch a yield event from the coroutine */
/* Returns 1 if C should immediately re-resume (synchronous event),
 * 0 if C should wait (async, timer, input) before resuming */
static int dispatch_event(lua_State *co, int nresults) {
    if (nresults < 1 || !lua_istable(co, -nresults)) {
        lua_pop(co, nresults);
        return 1;  /* unknown/empty yield - immediately resume */
    }

    int tbl = lua_gettop(co) - nresults + 1;
    lua_getfield(co, tbl, "action");
    const char *action = lua_isstring(co, -1) ? lua_tostring(co, -1) : "";
    lua_pop(co, 1);

    if (strcmp(action, "move") == 0) {
        lua_getfield(co, tbl, "to");
        engine_move(lua_tostring(co, -1));
        lua_pop(co, 1);
    } else if (strcmp(action, "say") == 0) {
        lua_getfield(co, tbl, "text");
        engine_say(lua_tostring(co, -1));
        lua_pop(co, 1);
        engine_wait(500);  /* wait for speech duration */
    } else if (strcmp(action, "wait") == 0) {
        lua_getfield(co, tbl, "ms");
        int ms = (int)lua_tointeger(co, -1);
        lua_pop(co, 1);
        engine_wait(ms);
    } else if (strcmp(action, "fade") == 0) {
        lua_getfield(co, tbl, "direction");
        engine_fade(lua_tostring(co, -1));
        lua_pop(co, 1);
        engine_wait(800);
    }

    lua_pop(co, nresults);
    return 1;  /* resume immediately in this simple example */
}

static const char *CUTSCENE_LUA =
    "function cutscene()\n"
    "    coroutine.yield({ action='move',  to='waypoint_A' })\n"
    "    coroutine.yield({ action='wait',  ms=1000 })\n"
    "    coroutine.yield({ action='say',   text='Hello, traveller.' })\n"
    "    coroutine.yield({ action='wait',  ms=500 })\n"
    "    coroutine.yield({ action='say',   text='The bridge ahead is blocked.' })\n"
    "    coroutine.yield({ action='fade',  direction='out' })\n"
    "    return 'cutscene_complete'\n"
    "end\n";

int main(void) {
    lua_State *L = luaL_newstate();
    luaL_openlibs(L);

    if (luaL_dostring(L, CUTSCENE_LUA) != LUA_OK) {
        fprintf(stderr, "script error: %s\n", lua_tostring(L, -1));
        lua_close(L); return 1;
    }

    /* Create coroutine from the 'cutscene' function */
    lua_getglobal(L, "cutscene");
    lua_State *co = lua_newthread(L);
    lua_xmove(L, co, 1);

    printf("=== Cutscene start ===\n");

    int nresults;
    int status = lua_resume(co, L, 0, &nresults);

    while (status == LUA_YIELD) {
        dispatch_event(co, nresults);
        status = lua_resume(co, L, 0, &nresults);
    }

    if (status == LUA_OK) {
        if (nresults > 0) {
            printf("=== %s ===\n", lua_tostring(co, -1));
            lua_pop(co, nresults);
        }
    } else {
        fprintf(stderr, "coroutine error: %s\n", lua_tostring(co, -1));
    }

    lua_close(L);
    return 0;
}



gcc -o sequence sequence.c $(pkg-config --cflags --libs lua5.4)
./sequence

# [ENGINE] say: The bridge ahead is blocked.

Cooperative Task Scheduler in Lua


Coroutines also enable a cooperative scheduler written entirely in Lua, with C providing
only the clock:



-- scheduler.lua
local tasks = {}
local clock = os.clock   -- replaced by C clock in embedding

local function sleep(seconds)
    local wake = clock() + seconds
    while clock() < wake do
        coroutine.yield()   -- yield back to scheduler
    end
end

local function spawn(f)
    table.insert(tasks, coroutine.create(f))
end

local function run()
    while #tasks > 0 do
        local alive = {}
        for _, co in ipairs(tasks) do
            local ok, err = coroutine.resume(co)
            if ok and coroutine.status(co) ~= "dead" then
                table.insert(alive, co)
            elseif not ok then
                print("task error:", err)
            end
        end
        tasks = alive
    end
end

spawn(function()
    for i = 1, 3 do
        print("Task A:", i)
        sleep(0.01)
    end
end)

spawn(function()
    for i = 1, 3 do
        print("Task B:", i)
        sleep(0.007)
    end
end)

run()



In an embedded context, the C game loop calls lua_resume on the scheduler
coroutine once per frame. The scheduler runs one round of all active tasks before
yielding back to C. Each task can yield as many times as it likes within a frame budget.
The C side controls real-time pacing; Lua controls logical sequencing.


Summary


Coroutines give Lua sequential scripting inside an event-driven C host. Create a
coroutine thread with lua_newthread, load the function with
lua_xmove, and drive execution with lua_resume. Each
LUA_YIELD return carries data from the coroutine to C; the next
lua_resume carries data from C back to the coroutine. When the coroutine's
function returns, lua_resume returns LUA_OK and the thread
is dead.



The pattern composes naturally with the event table dispatch shown in this article:
yield a table describing the desired action, dispatch in C, resume when done. Scripts
read as sequential prose; the C host implements the real-time machinery.
