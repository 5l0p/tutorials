Containers - Part 19b: Mini Container Runtime in Go - Part 2
2026-06-01


Part 1 established the runtime's architecture and implemented namespace setup,
pivot_root, virtual filesystem mounting, and UTS. This part completes the
runtime by adding the four remaining isolation layers: a veth pair connecting
the container to the host network, cgroups v2 for resource limits, capability
dropping to reduce privilege, and a seccomp filter to cut off dangerous syscalls.
At the end we run a real process inside the fully assembled container.

network.go - Veth Pair Setup


Network setup happens in two phases. The parent (running in the host namespace)
creates the veth pair and assigns an address to the host end. The child (running
inside the container network namespace) brings up its end and adds the default
route. The parent-side code runs after the child has started but before the
sync pipe signals it to continue.



package main

import (
    "fmt"
    "net"
    "os"
    "runtime"

    "github.com/vishvananda/netlink"
)

const (
    hostVethName = "miniruntime0"
    contVethName = "eth0"
    hostAddr     = "10.210.0.1/24"
    contAddr     = "10.210.0.2/24"
    contGW       = "10.210.0.1"
)

// setupNetwork is called from the parent process.
// pid is the container init's host PID; its network namespace fd is at
// /proc/<pid>/ns/net.
func setupNetwork(pid int) error {
    veth := &netlink.Veth{
        LinkAttrs: netlink.LinkAttrs{Name: hostVethName},
        PeerName:  contVethName,
    }
    // Remove any leftover interface from a previous run.
    if old, err := netlink.LinkByName(hostVethName); err == nil {
        netlink.LinkDel(old)
    }

    if err := netlink.LinkAdd(veth); err != nil {
        return fmt.Errorf("LinkAdd veth: %w", err)
    }

    // Open the container's network namespace fd.
    contNSPath := fmt.Sprintf("/proc/%d/ns/net", pid)
    contNS, err := os.Open(contNSPath)
    if err != nil {
        return fmt.Errorf("open container netns: %w", err)
    }
    defer contNS.Close()

    // Move the container-side peer into the container namespace.
    peer, err := netlink.LinkByName(contVethName)
    if err != nil {
        return fmt.Errorf("LinkByName peer: %w", err)
    }
    if err := netlink.LinkSetNsFd(peer, int(contNS.Fd())); err != nil {
        return fmt.Errorf("LinkSetNsFd: %w", err)
    }

    // Configure the host end.
    host, err := netlink.LinkByName(hostVethName)
    if err != nil {
        return fmt.Errorf("LinkByName host: %w", err)
    }
    ip, ipnet, _ := net.ParseCIDR(hostAddr)
    ipnet.IP = ip
    if err := netlink.AddrAdd(host, &netlink.Addr{IPNet: ipnet}); err != nil {
        return fmt.Errorf("AddrAdd host: %w", err)
    }
    return netlink.LinkSetUp(host)
}

// setupContainerNetwork is called from the child process, already inside
// the container's network namespace.
func setupContainerNetwork() error {
    // LockOSThread keeps this goroutine on one OS thread while we
    // manipulate thread-local network namespace state.
    runtime.LockOSThread()
    defer runtime.UnlockOSThread()

    lo, err := netlink.LinkByName("lo")
    if err != nil {
        return fmt.Errorf("lo: %w", err)
    }
    netlink.LinkSetUp(lo)

    eth, err := netlink.LinkByName(contVethName)
    if err != nil {
        return fmt.Errorf("%s: %w", contVethName, err)
    }
    ip, ipnet, _ := net.ParseCIDR(contAddr)
    ipnet.IP = ip
    if err := netlink.AddrAdd(eth, &netlink.Addr{IPNet: ipnet}); err != nil {
        return fmt.Errorf("AddrAdd container: %w", err)
    }
    if err := netlink.LinkSetUp(eth); err != nil {
        return fmt.Errorf("LinkSetUp container: %w", err)
    }

    // Default route via the host veth address.
    gw := net.ParseIP(contGW)
    return netlink.RouteAdd(&netlink.Route{
        Scope: netlink.SCOPE_UNIVERSE,
        Gw:    gw,
    })
}

// cleanupNetwork removes the host veth. The container end disappears
// automatically when the container network namespace is destroyed.
func cleanupNetwork() {
    if link, err := netlink.LinkByName(hostVethName); err == nil {
        netlink.LinkDel(link)
    }
}


cgroup.go - cgroups v2 Resource Limits


The parent calls setupCgroupV2 after starting the child but before
signalling it via the sync pipe. This guarantees the child is in the cgroup
before it begins executing any workload.



package main

import (
    "fmt"
    "os"
    "path/filepath"
    "strconv"
)

const (
    cgBase    = "/sys/fs/cgroup/miniruntime"
    memLimit  = 128 * 1024 * 1024 // 128 MB
    cpuQuota  = "50000 100000"     // 50% of one core
    pidsLimit = 64
)

func writeFile(path, value string) error {
    return os.WriteFile(path, []byte(value), 0644)
}

func setupCgroupV2(pid int) error {
    // Enable required controllers in the root cgroup's subtree.
    if err := writeFile("/sys/fs/cgroup/cgroup.subtree_control",
        "+memory +pids +cpu"); err != nil {
        // Non-fatal: the controllers may already be enabled.
        _ = err
    }

    if err := os.MkdirAll(cgBase, 0755); err != nil {
        return fmt.Errorf("mkdir cgroup: %w", err)
    }

    settings := [][2]string{
        {"memory.max",      strconv.Itoa(memLimit)},
        {"memory.swap.max", "0"},
        {"cpu.max",         cpuQuota},
        {"pids.max",        strconv.Itoa(pidsLimit)},
    }
    for _, kv := range settings {
        if err := writeFile(filepath.Join(cgBase, kv[0]), kv[1]); err != nil {
            return fmt.Errorf("write %s: %w", kv[0], err)
        }
    }

    // Attach the container init process.
    return writeFile(filepath.Join(cgBase, "cgroup.procs"), strconv.Itoa(pid))
}

func cleanupCgroup() {
    // The directory can only be removed when empty (all processes have exited).
    os.Remove(cgBase)
}


caps.go - Dropping Capabilities


Capability dropping runs inside the child after pivot_root and network
configuration, but before the seccomp filter. We use the raw capset syscall
via golang.org/x/sys/unix to set only the capabilities a typical container
workload needs.



package main

import (
    "fmt"
    "unsafe"

    "golang.org/x/sys/unix"
)

// keepCaps is the capability bitmask Docker retains by default.
// Bit positions correspond to the CAP_* constants in linux/capability.h.
const keepCaps uint64 = 0 |
    (1 << 0)  | // CAP_CHOWN
    (1 << 1)  | // CAP_DAC_OVERRIDE
    (1 << 3)  | // CAP_FOWNER
    (1 << 4)  | // CAP_FSETID
    (1 << 5)  | // CAP_KILL
    (1 << 6)  | // CAP_SETGID
    (1 << 7)  | // CAP_SETUID
    (1 << 8)  | // CAP_SETPCAP
    (1 << 10) | // CAP_NET_BIND_SERVICE
    (1 << 13) | // CAP_NET_RAW (kept for ping; drop with --cap-drop NET_RAW)
    (1 << 18) | // CAP_SYS_CHROOT
    (1 << 29) | // CAP_AUDIT_WRITE
    0

func dropCapabilities() error {
    // Step 1: Drop everything not in keepCaps from the bounding set.
    // Capabilities removed from the bounding set can never return via exec.
    for cap := uint(0); cap <= 40; cap++ {
        if keepCaps&(1<<cap) != 0 {
            continue
        }
        err := unix.Prctl(unix.PR_CAPBSET_DROP, uintptr(cap), 0, 0, 0)
        if err != nil && err != unix.EINVAL {
            return fmt.Errorf("PR_CAPBSET_DROP %d: %w", cap, err)
        }
    }

    // Step 2: Set effective, permitted, and inheritable sets to keepCaps.
    type capHeader struct {
        version uint32
        pid     int32
    }
    type capData struct {
        effective   uint32
        permitted   uint32
        inheritable uint32
    }
    hdr := capHeader{version: 0x20080522, pid: 0}
    var data [2]capData
    data[0].effective   = uint32(keepCaps & 0xffffffff)
    data[0].permitted   = uint32(keepCaps & 0xffffffff)
    data[0].inheritable = 0
    data[1].effective   = uint32(keepCaps >> 32)
    data[1].permitted   = uint32(keepCaps >> 32)
    data[1].inheritable = 0

    if _, _, errno := unix.Syscall(unix.SYS_CAPSET,
        uintptr(unsafe.Pointer(&hdr)),
        uintptr(unsafe.Pointer(&data[0])), 0); errno != 0 {
        return fmt.Errorf("capset: %w", errno)
    }

    // Step 3: Set NO_NEW_PRIVS so exec'd children cannot gain privileges.
    return unix.Prctl(unix.PR_SET_NO_NEW_PRIVS, 1, 0, 0, 0)
}


seccomp.go - Syscall Filter


The seccomp filter is the last thing applied before exec. It must come
after NO_NEW_PRIVS (set inside dropCapabilities). The filter below uses a
blacklist approach matching Docker's default: all syscalls allowed except a
specific set that is known to be dangerous or unnecessary in containers.



package main

import (
    "fmt"
    "unsafe"

    "golang.org/x/sys/unix"
)

// BPF instruction encoding
const (
    bpfLD  = 0x00
    bpfW   = 0x00
    bpfABS = 0x20
    bpfJMP = 0x05
    bpfJEQ = 0x10
    bpfK   = 0x00
    bpfRET = 0x06
)

const (
    retKill  = uint32(0x80000000) // SECCOMP_RET_KILL_PROCESS
    retAllow = uint32(0x7fff0000) // SECCOMP_RET_ALLOW
    retEPERM = uint32(0x00050000) | 1 // SECCOMP_RET_ERRNO(EPERM)
)

const auditArchX86_64 = uint32(0xc000003e)

type bpfInsn struct {
    code uint16
    jt   uint8
    jf   uint8
    k    uint32
}

// blockedSyscalls lists syscall numbers (x86-64) that are blocked.
// Sources: Docker moby default seccomp profile + OCI spec recommendations.
var blockedSyscalls = []uint32{
    96,  // gettimeofday -- not blocked; below is the actual list:
}

func init() {
    // Reset to the real list (the above initialisation was a placeholder).
    blockedSyscalls = []uint32{
        101, // ptrace
        312, // kcmp
        298, // perf_event_open
        317, // add_key
        318, // request_key
        250, // keyctl
        169, // reboot
        320, // kexec_file_load
        246, // kexec_load
        165, // mount
        235, // unshare
        182, // acct
        163, // syslog (klogctl)
        186, // gettid -- NOT blocked; removed
        281, // userfaultfd
        154, // sched_setscheduler (allowed in Docker, blocked here)
        179, // quotactl
        174, // create_module
        175, // init_module
        176, // delete_module
        177, // get_kernel_syms
        178, // query_module
        180, // nfsservctl
        183, // security
        185, // lookup_dcookie
        214, // epoll_create -- NOT blocked
        236, // move_pages
        256, // migrate_pages
        237, // mbind
        273, // set_mempolicy
        274, // get_mempolicy
        275, // mq_open -- NOT blocked
    }
    // Rebuild without the ones marked NOT blocked above.
    blockedSyscalls = []uint32{
        101, // ptrace
        312, // kcmp
        298, // perf_event_open
        317, // add_key
        318, // request_key
        250, // keyctl
        169, // reboot
        320, // kexec_file_load
        246, // kexec_load
        165, // mount
        235, // unshare
        182, // acct
        163, // syslog (klogctl)
        281, // userfaultfd
        179, // quotactl
        174, // create_module
        175, // init_module
        176, // delete_module
        236, // move_pages
        256, // migrate_pages (migrate_pages)
        237, // mbind
    }
}

func buildSeccompFilter() []bpfInsn {
    var prog []bpfInsn

    // 1. Load and validate the architecture.
    prog = append(prog,
        bpfInsn{bpfLD | bpfW | bpfABS, 0, 0, 4},  // load arch (offset 4)
        bpfInsn{bpfJMP | bpfJEQ | bpfK, 1, 0, auditArchX86_64},
        bpfInsn{bpfRET | bpfK, 0, 0, retKill},     // wrong arch: kill
    )

    // 2. Load the syscall number.
    prog = append(prog,
        bpfInsn{bpfLD | bpfW | bpfABS, 0, 0, 0},  // load nr (offset 0)
    )

    // 3. For each blocked syscall: if nr == blocked, return EPERM.
    for _, nr := range blockedSyscalls {
        prog = append(prog,
            bpfInsn{bpfJMP | bpfJEQ | bpfK, 0, 1, nr},
            bpfInsn{bpfRET | bpfK, 0, 0, retEPERM},
        )
    }

    // 4. Default: allow.
    prog = append(prog,
        bpfInsn{bpfRET | bpfK, 0, 0, retAllow},
    )

    return prog
}

type sockFprog struct {
    len    uint16
    _      [6]byte
    filter *bpfInsn
}

func installSeccompFilter() error {
    prog := buildSeccompFilter()
    fp := sockFprog{
        len:    uint16(len(prog)),
        filter: &prog[0],
    }
    _, _, errno := unix.Syscall(unix.SYS_PRCTL,
        unix.PR_SET_SECCOMP,
        unix.SECCOMP_MODE_FILTER,
        uintptr(unsafe.Pointer(&fp)))
    if errno != 0 {
        return fmt.Errorf("PR_SET_SECCOMP: %w", errno)
    }
    return nil
}


Putting It All Together


With all the source files in place, the complete call sequence is:



# From parent.go:
# 1. exec.Command("/proc/self/exe", "init") with CLONE_NEW* flags
# 2. setupCgroupV2(child.Process.Pid)
# 3. setupNetwork(child.Process.Pid)
# 4. write "go" to sync pipe
# 5. child.Wait()

# From child.go (init mode):
# 1. setupRootfs(cfg.Rootfs)      -- pivot_root, mount /proc /dev /sys /tmp
# 2. syscall.Sethostname(...)     -- UTS namespace hostname
# 3. read sync pipe               -- wait for parent
# 4. setupContainerNetwork()      -- configure eth0, lo, default route
# 5. dropCapabilities()           -- bounding set + effective/permitted + NO_NEW_PRIVS
# 6. installSeccompFilter()       -- block dangerous syscalls
# 7. syscall.Exec(cmd)            -- replace init with the target process

NAT for Outbound Connectivity


The container can reach 10.210.0.1 (the host veth) but not the internet yet.
Add a MASQUERADE rule on the host after starting the runtime:



# Enable IP forwarding
sudo sysctl -w net.ipv4.ip_forward=1

# NAT for the container subnet
sudo iptables -t nat -A POSTROUTING \
    -s 10.210.0.0/24 ! -o miniruntime0 -j MASQUERADE

# Allow forwarding
sudo iptables -A FORWARD -i miniruntime0 -j ACCEPT
sudo iptables -A FORWARD -o miniruntime0 -j ACCEPT

# Inside the container:
ping -c 2 8.8.8.8
# PING 8.8.8.8: 64 data bytes
# 64 bytes from 8.8.8.8: seq=0 ttl=... time=...


What This Runtime Still Lacks


Our runtime is deliberately minimal. Production runtimes add:



OCI image support - pulling, verifying, and extracting image layers (overlayfs). Our runtime requires a pre-extracted root directory.
Container state management - a database of running containers, PIDs, cgroup paths, network interfaces, so you can list, inspect, stop, and delete containers by name.
More complete /dev - production runtimes bind-mount many devices from the host's /dev (GPU devices, FUSE, sound) rather than creating a minimal synthetic /dev.
Volume mounts - bind-mounting host paths into the container's mount namespace.
Health checks and logging - capturing stdout/stderr, PID1 exit handling.
A richer seccomp policy - our filter is hand-written BPF; production runtimes load the OCI seccomp JSON spec.
Time namespace - clock isolation (Linux 5.6+).


Comparing with runc


# What runc does under the hood for `runc run`:
# 1. Reads config.json (OCI spec)
# 2. Creates process in new namespaces (same CLONE_NEW* flags we use)
# 3. Calls newuidmap/newgidmap for UID/GID maps
# 4. Sets up rootfs (overlayfs layers + bind mounts from spec)
# 5. Calls pivot_root (same syscall sequence)
# 6. Mounts /proc /dev /sys /dev/pts from spec mounts array
# 7. Applies cgroup limits via systemd or direct cgroupfs writes
# 8. Drops capabilities per spec's process.capabilities
# 9. Loads seccomp profile from spec's linux.seccomp JSON
# 10. Writes container state to /run/runc/<id>/state.json
# 11. Calls exec() on the container entrypoint

# We do all of the same things in the same order.
# runc just does it more carefully and against the OCI spec.


Summary


The complete mini container runtime applies all six isolation layers in sequence:
namespaces (PID, mount, UTS, network, IPC, user) via CLONE_NEW* flags on
the child process; rootfs isolation via pivot_root with virtual filesystems
mounted inside; resource limits via cgroups v2 memory.max, cpu.max, and
pids.max; privilege reduction via capability dropping with capset and
the bounding set; and finally syscall filtering with a seccomp BPF program. The
re-exec pattern keeps the Go runtime clean with respect to namespace boundaries.



The runtime does in roughly 400 lines of Go what Docker, Podman, and runc do in
hundreds of thousands - but the kernel primitives are identical. The next article
covers the OCI Runtime Specification that runc implements: the config.json
format, the container lifecycle, and how your mini-runtime fits into the broader
OCI ecosystem.
