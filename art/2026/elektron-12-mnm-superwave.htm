Elektron Synthesis - Part 12: Monomachine SuperWave Engine

The Monomachine's SuperWave engine stacks multiple detuned oscillators to create an extraordinarily thick, wide sound — the classic "super saw" or "hypersawtooth" sound made famous by the Roland JP-8000. Where analogue polyphonic synthesisers achieve this thickness by running multiple physical oscillators in near-unison, the SuperWave engine implements it digitally with N oscillators pitched at calculated detune offsets. This article builds a complete SuperWave implementation in Pd.

Detune Spread Calculation

With N oscillators, one sits at the root pitch (centre) and the others are spread symmetrically above and below by multiples of the DETUNE step. For a 7-oscillator bank: 

; 7 oscillators, DETUNE in cents:
; osc 1: root − 3*detune
; osc 2: root − 2*detune
; osc 3: root − 1*detune
; osc 4: root              (centre, always present)
; osc 5: root + 1*detune
; osc 6: root + 2*detune
; osc 7: root + 3*detune

; Convert cents to Hz multiplier:
; multiplier = 2^(detune_cents / 1200)
; e.g. detune_cents=10: multiplier = 2^(10/1200) = 1.00579

freq_n = root_hz × 2^((n - centre) × detune_cents / 1200)

In Pd with [expr]: 

[r $0-freq_hz]    → $root
[r $0-det_cents]  → $det

; Oscillator N frequency:
[expr $root * pow(2, ($n * $det) / 1200)]
; where $n is -3, -2, -1, 0, 1, 2, 3

Amplitude Mix Between Centre and Edge Oscillators

The MIX parameter controls the relative amplitude between the centre oscillator and the outer oscillators. The JP-8000 super saw uses a specific empirically derived amplitude distribution — the centre oscillator is slightly louder than the edges to maintain a perceived centre in the stereo image: 

; Empirical super saw amplitudes (7 oscillators):
; Centre (n=0): 1.0
; n=±1:         0.776
; n=±2:         0.664
; n=±3:         0.537

; MIX parameter morphs between equal mix (mix=0) and this empirical distribution (mix=1):
; amp_n = 1 × (1-mix) + empirical_n × mix

; In Pd, store amplitude weights as an array and use [tabread]:
[array sw_amps 7]    ; fill with [0.537, 0.664, 0.776, 1.0, 0.776, 0.664, 0.537]
                      ; or lerp based on MIX

; Per oscillator gain:
[tabread sw_amps]  ← n + 3   ; index 0–6
|
[*~ $mix_weight]

Phase Randomisation

When all oscillators start with the same phase, the initial burst of the note sounds phasey and comb-filtered. Randomising the initial phase of each oscillator prevents this and creates a smoother, more organic onset: 

; PHASE parameter: amount of initial phase randomisation per oscillator
; phase_n = n × $phase / count  (linear spread)  OR
; phase_n = random() × $phase   (random spread)

; In Pd, set the initial phase when triggering each oscillator.
; [osc~] does not accept a phase offset directly, but you can
; add a fixed DC phase using [+~] before the oscillator lookup,
; or use [tabread4~] with a phase offset:

[phasor~ $freq_n]
|
[+~ $phase_offset_n]    ; fixed offset per oscillator
|
[wrap~]                 ; keep 0–1
|
[*~ 2][-~ 1]            ; → sawtooth wave

Waveform Variants

SuperSaw (WAVE=SAW): Each oscillator outputs a sawtooth. This is the most common SuperWave form — the stacked sawtooth sound heard in most EDM synth stabs, supersaws, and pad sounds.

SuperSquare (WAVE=SQR): Each oscillator outputs a square/pulse wave. The PW parameter adjusts all oscillators' pulse widths simultaneously. Narrower pulses create a sharper, nasal quality; 50% duty cycle produces the classic hollow square character.

SuperTriangle (WAVE=TRI): Each oscillator outputs a triangle wave. Softer than saw or square; produces a warm, slightly padded texture. Often used for softer lead sounds or gentle pads.

Complete SuperWave Pd Patch (7 oscillators, Sawtooth)

; 7 oscillators, detuned sawtooth:
[r $0-freq]     → root_hz
[r $0-det]      → detune_cents (0–100)
[r $0-mix]      → mix (0–1)

; Per oscillator (n = -3 to +3):
[expr root_hz * pow(2, $n * $det / 1200)]    ; freq_n
|
[phasor~]
|
[+ phase_n]
|
[wrap~]
|
[*~ 2][-~ 1]                                  ; sawtooth
|
[*~ amp_weight_n]                             ; per-osc amplitude
|
[*~ 1/7]                                      ; normalise (7 oscillators)

; Sum all 7:
[+~] ...

; Filter (MNM shared filter — 2-pole multimode):
[lop~ cutoff_hz] OR [vcf~ cutoff_hz res_q]
|
; Amplitude ADSR:
[*~] ← [vline~] ADSR envelope
|
[*~ level * vel]
|
[outlet~]

2-Pole Multimode Filter

The Monomachine's filter is shared across all engines — a 2-pole state variable filter (SVF) supporting LP, BP, and HP simultaneously. Implement in Pd with two integrators in feedback: 

; State variable filter (SVF), 2-pole:
; Inputs: x (audio), f (cutoff: 0–1, maps to 2*sin(π*fc/sr)), q (resonance: 0–1)

; States: LP (lowpass), BP (bandpass), HP (highpass)
; Update per sample:
; LP[n] = LP[n-1] + f * BP[n-1]
; HP[n] = x[n] - LP[n] - q*BP[n-1]
; BP[n] = f * HP[n] + BP[n-1]

; In Pd, implement with [+~], [*~], and unit delay [z-1]:
; Each unit delay is one sample (can't do in pure vanilla Pd without external)

; Approximation using [lop~] and [hip~] in feedback:
; The [vcf~] object in Pd is a 2-pole resonant filter that implements SVF-like
; behavior — use [vcf~] for a quality SVF:
[vcf~ $cutoff_hz $q]     ; two outputs: bandpass, lowpass
; highpass = input - lowpass - bandpass

Summary

The Monomachine SuperWave engine stacks 2–7 detuned oscillators (sawtooth, square, or triangle) with per-oscillator amplitude weighting, initial phase randomisation, and all sharing a 2-pole multimode filter and a common ADSR. The detune spread (in cents) controls the chorus width; the MIX parameter controls centre/edge amplitude balance. Seven oscillators at 10 cents spread with JP-8000-style amplitude distribution produces the canonical super saw sound. Narrower detune (1–3 cents) produces subtle ensemble thickness; wider detune (20+ cents) produces obvious vibrato/chorus character.
