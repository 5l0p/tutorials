Algorithmic Composition - Part 10: Euclidean Rhythms

Most rhythmic patterns in Western popular and classical music place beats on metrically strong positions: downbeats, half-beats, quarter-beats. The result is a feel of regularity and predictability. A different approach asks: if you have k beats to distribute in a bar of n steps, what is the most even distribution? The answer turns out to be the Euclidean rhythm — and the most even distributions happen to match recurring patterns in world music that Western notation would call syncopated.

Toussaint (2004) showed that patterns generated by Bjorklund's algorithm — developed for accelerator timing systems at CERN, not for music — correspond to the clave rhythms of African and Cuban music, the bossa nova timeline, Bulgarian aksak rhythms, and West African bell patterns. Distributing beats evenly, with no knowledge of music, produces rhythms with deep cultural pedigree.

The Euclidean Rhythm Formula

Given k beats and n slots, slot i (0-indexed) contains a beat if and only if: 

(i * k) mod n  <  k

This places a beat at slot i when multiplying i by the beat density k/n, modulo n, falls below k. The formula distributes k onsets as evenly as possible across n slots.

In Lua: 

-- Euclidean rhythm: k beats in n slots.
-- Returns a 1-indexed array of 0s and 1s; 1 = beat, 0 = rest.
local function euclidean(k, n)
    local pattern = {}
    for i = 0, n - 1 do
        pattern[i + 1] = ((i * k) % n < k) and 1 or 0
    end
    return pattern
end

-- Print a pattern as dots and Xs
local function print_pattern(pat, name)
    io.write(name .. ": ")
    for _, v in ipairs(pat) do io.write(v == 1 and "X" or ".") end
    print()
end

print_pattern(euclidean(3, 8),  "E(3,8)  ")  -- tresillo
print_pattern(euclidean(5, 8),  "E(5,8)  ")  -- bossa nova
print_pattern(euclidean(3, 16), "E(3,16) ")
print_pattern(euclidean(7, 16), "E(7,16) ")  -- Brazilian necklace
print_pattern(euclidean(5, 12), "E(5,12) ")  -- Afro-Cuban
print_pattern(euclidean(2, 3),  "E(2,3)  ")  -- basic clave cell

Output: 

E(3,8)  : X..X..X.
E(5,8)  : X.XX.XX.  (one rotation of the bossa nova clave)
E(3,16) : X.....X.....X...
E(7,16) : X.XX.X.XX.X.XX..
E(5,12) : X..X.X..X.X.
E(2,3)  : X.X

Rotation

The formula starts the pattern at slot 0. Many musical uses of these rhythms start at a different point in the cycle (a rotation). Rotation is a simple array operation: 

-- Rotate a pattern by r positions (positive = shift right = delay onset)
local function rotate(pattern, r)
    local n      = #pattern
    local result = {}
    for i = 1, n do
        result[i] = pattern[((i - 1 - r) % n) + 1]
    end
    return result
end

-- E(3,8) rotated by 2: shifts the first beat two steps right
-- [X..X..X.] → [..X..X..X] → [...X..X.X] (approx, wraps)
print_pattern(rotate(euclidean(3, 8), 2), "E(3,8) r=2")

Output: 

E(3,8) r=2: ..X..X.X

Polyrhythm from Coprime Periods

Two patterns with coprime (mutually prime) lengths produce a polyrhythm that repeats only after lcm(n1, n2) steps. For n1=8 and n2=12 (gcd=4, not coprime), the pattern repeats after 24 steps. For n1=8 and n2=9 (gcd=1, coprime), it repeats after 72 steps.

Stacking E(3,8) on a kick and E(5,9) on a hi-hat creates a rhythm that the ear cannot easily predict — a useful property for generative music that should avoid feeling repetitive: 

local function lcm(a, b)
    local function gcd(x, y) return y == 0 and x or gcd(y, x % y) end
    return a * b // gcd(a, b)
end

-- Extend a pattern to length L by repetition
local function extend_to(pattern, L)
    local n      = #pattern
    local result = {}
    for i = 1, L do
        result[i] = pattern[((i - 1) % n) + 1]
    end
    return result
end

local p1  = euclidean(3, 8)   -- kick: 3 beats in 8 steps
local p2  = euclidean(5, 9)   -- hat:  5 beats in 9 steps
local L   = lcm(8, 9)         -- 72 steps before exact repetition

local kick_cycle = extend_to(p1, L)
local hat_cycle  = extend_to(p2, L)

-- Print a combined 16-step preview
io.write("Kick: ")
for i = 1, 16 do io.write(kick_cycle[i] == 1 and "K" or ".") end
print()
io.write("Hat:  ")
for i = 1, 16 do io.write(hat_cycle[i] == 1 and "H" or ".") end
print()

Output: 

Kick: K..K..K.K..K..K.
Hat:  H..HH..H.H..HH..

Integrating with the Drum Machine

The drums-07 sequencer accepts per-step velocity arrays (0 = silent, 1–127 = velocity). Replace the stored 16-step pattern with a Euclidean pattern generated on demand: 

-- Replace drums-07 fixed pattern with Euclidean rhythm
local kick_pat = euclidean(4, 16)   -- standard four-on-the-floor
local snare_pat = rotate(euclidean(3, 16), 4)  -- backbeat feel
local hat_pat   = euclidean(7, 16)  -- 7-in-16 hi-hat

-- Convert to velocity arrays (hit = 100, rest = 0)
local function to_velocities(pattern, vel)
    vel = vel or 100
    local v = {}
    for i, b in ipairs(pattern) do v[i] = b == 1 and vel or 0 end
    return v
end

pd.send("kick-vel",  table.unpack(to_velocities(kick_pat)))
pd.send("snare-vel", table.unpack(to_velocities(snare_pat)))
pd.send("hat-vel",   table.unpack(to_velocities(hat_pat)))

To evolve the rhythm over time, change k or r per bar: 

local bar = 0
local function on_bar()
    bar = bar + 1
    -- Every 4 bars, increase the hat density by 1 until k=9, then reset to 3
    local k_hat = 3 + ((bar // 4) % 7)
    local hat   = to_velocities(euclidean(k_hat, 16))
    pd.send("hat-vel", table.unpack(hat))
end

Summary

Euclidean rhythms distribute k beats across n steps as evenly as possible using the formula (i * k) mod n < k. The resulting patterns coincide with rhythmic timelines found in African, Cuban, Brazilian, Turkish, and Balkan music — a striking structural fact with no obvious explanation. Rotating a pattern shifts its starting point without changing its internal structure. Two patterns with coprime period lengths create polyrhythms that repeat only after lcm(n1, n2) steps. Euclidean pattern arrays slot directly into the drums-07 sequencer's per-step velocity format, replacing static patterns with rule-generated ones.
