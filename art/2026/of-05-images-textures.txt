openFrameworks - Part 5: Images & Textures
2026-09-26


Images are central to creative coding - as backgrounds, brushes, data sources, and render
targets. openFrameworks provides three levels of abstraction: ofPixels (CPU
memory, raw pixel data), ofTexture (GPU memory, a single OpenGL texture object),
and ofImage (a paired ofPixels + ofTexture that handles
loading and synchronisation between the two). Understanding which layer you need for a given
task prevents a lot of unnecessary copying between CPU and GPU.


What You Will Learn


The three-layer model: ofPixels (CPU), ofTexture (GPU), ofImage (both together)
Loading images from disk with ofImage::load() and drawing with draw()
Manipulating pixels on the CPU with ofPixels: get, set, and iterate over pixels
Uploading pixel data to the GPU with ofTexture::loadData()
Render-to-texture with ofFbo: drawing into an offscreen buffer then using it as a texture
Image format types: OF_IMAGE_COLOR, OF_IMAGE_COLOR_ALPHA, and PNG transparency



Loading and Drawing with ofImage


ofImage is the workhorse. Call load() in setup(); it
reads the file from disk, decodes it (supporting PNG, JPEG, BMP, GIF, TIFF via FreeImage),
and uploads the result to a GPU texture.



// ofApp.h
#include <ofMain.h>

class ofApp : public ofBaseApp {
public:
    void setup();
    void draw();
private:
    ofImage background;
    ofImage sprite;
};



// ofApp.cpp
void ofApp::setup() {
    background.load("images/bg.jpg");
    sprite.load("images/ship.png");    // PNG supports transparency
}

void ofApp::draw() {
    // Draw at original size, top-left corner
    background.draw(0, 0);

    // Draw at a specified position and size
    sprite.draw(100, 100, 64, 64);

    // Draw centred
    float cx = ofGetWidth()  / 2.0f - sprite.getWidth()  / 2.0f;
    float cy = ofGetHeight() / 2.0f - sprite.getHeight() / 2.0f;
    sprite.draw(cx, cy);
}



Asset files go in the bin/data/ folder of your project. oF automatically sets
the working directory to bin/data/ on startup, so you can use relative paths
like "images/bg.jpg".


Checking Load Success


if (!background.load("images/bg.jpg")) {
    ofLogError("ofApp") << "Failed to load background image.";
}



Always check the return value during development. A missing asset silently produces an
uninitialised 0x0 texture that causes subtle rendering artefacts rather than an obvious crash.


Image Properties


int w = background.getWidth();
int h = background.getHeight();
ofImageType t = background.getImageType();  // OF_IMAGE_COLOR, OF_IMAGE_COLOR_ALPHA, ...

// Resize on the CPU (resamples the pixel buffer)
background.resize(1920, 1080);

// Mirror
background.mirror(false, true);   // (flipVertical, flipHorizontal)


ofPixels: CPU-Side Pixel Access


ofPixels is a container for raw pixel data in CPU memory. It knows its width,
height, and channel layout. Access it from an ofImage with
getPixels():



ofPixels &pix = background.getPixels();

// Read a pixel
ofColor c = pix.getColor(x, y);
int r = c.r;
int g = c.g;
int b = c.b;

// Write a pixel
pix.setColor(x, y, ofColor(255, 0, 0));

// After modifying pixels, upload the new data to the GPU texture
background.update();



A common pattern: iterate over all pixels and apply a per-pixel effect.



void ofApp::setup() {
    img.load("images/photo.jpg");

    ofPixels &pix = img.getPixels();
    int w = img.getWidth();
    int h = img.getHeight();

    for (int y = 0; y < h; y++) {
        for (int x = 0; x < w; x++) {
            ofColor c = pix.getColor(x, y);
            // Convert to greyscale
            unsigned char grey = (unsigned char)(0.299f * c.r + 0.587f * c.g + 0.114f * c.b);
            pix.setColor(x, y, ofColor(grey, grey, grey));
        }
    }
    img.update();   // push modified pixels to GPU
}



For large images this loop is slow. If you need real-time per-pixel processing, do it in a
fragment shader on the GPU instead (see the Shaders article).


Creating an Image Programmatically


ofImage noiseImg;
noiseImg.allocate(512, 512, OF_IMAGE_COLOR);

ofPixels &pix = noiseImg.getPixels();
for (int y = 0; y < 512; y++) {
    for (int x = 0; x < 512; x++) {
        float n = ofNoise(x * 0.02f, y * 0.02f);
        unsigned char v = (unsigned char)(n * 255);
        pix.setColor(x, y, ofColor(v, v / 2, 255 - v));
    }
}
noiseImg.update();


ofTexture: The GPU Layer


ofTexture wraps a single OpenGL texture object. You rarely need to use it
directly when working with ofImage, but it is essential when you want to upload
custom data to the GPU - for example, audio samples for a shader visualiser.



ofTexture tex;
tex.allocate(512, 1, GL_R32F);      // 512-wide, 1-pixel tall, single float channel

// Upload a vector of floats (e.g., FFT magnitudes)
std::vector<float> fftData(512, 0.0f);
// ... fill fftData from audio analysis ...
tex.loadData(fftData.data(), 512, 1, GL_RED);

// Use in a shader
shader.setUniformTexture("fftTex", tex, 1);



Texture coordinates in oF are in pixel space by default (not normalised 0–1). This is
controlled by ofEnableNormalizedTexCoords() / ofDisableNormalizedTexCoords().


ofFbo: Render to Texture


A framebuffer object (FBO) lets you render a scene to an off-screen texture instead of
directly to the screen. You can then use that texture as input to another render pass, apply
post-processing, or save the result to disk.



// ofApp.h
class ofApp : public ofBaseApp {
public:
    void setup();
    void update();
    void draw();
private:
    ofFbo canvas;
    ofImage brush;
};



void ofApp::setup() {
    canvas.allocate(ofGetWidth(), ofGetHeight(), GL_RGBA);
    canvas.begin();
        ofClear(0, 0, 0, 255);   // clear the FBO to opaque black
    canvas.end();

    brush.load("images/brush.png");
}

void ofApp::update() {
    if (ofGetMousePressed()) {
        canvas.begin();
            ofSetColor(255, 255, 255, 80);
            brush.draw(ofGetMouseX() - 16, ofGetMouseY() - 16, 32, 32);
        canvas.end();
    }
}

void ofApp::draw() {
    ofSetColor(255);
    canvas.draw(0, 0);           // draw the accumulated canvas to screen
}



The FBO persists between frames - anything you render into it stays until you explicitly
clear it. This is the standard technique for paint applications, feedback loops, and
long-exposure simulations.


Saving Images to Disk


// Save the current screen to a PNG
ofSaveScreen("screenshot.png");

// Save an ofFbo contents
ofImage saved;
canvas.readToPixels(saved.getPixels());
saved.update();
saved.save("output.png");

// Save a specific ofImage
img.save("processed.jpg");



Files are saved relative to bin/data/. JPEG quality can be set:
img.save("out.jpg", OF_IMAGE_QUALITY_MEDIUM).


Pixel Formats and Channels


ofImage supports three channel layouts:



  OF_IMAGE_GRAYSCALE - 1 channel, 8 bits per pixel.
  OF_IMAGE_COLOR - 3 channels (RGB), 24 bits per pixel.
  OF_IMAGE_COLOR_ALPHA - 4 channels (RGBA), 32 bits per pixel.



PNG files with transparency are automatically loaded as OF_IMAGE_COLOR_ALPHA.
JPEGs are always OF_IMAGE_COLOR. Transparency from a PNG sprite is correctly
composited when alpha blending is enabled.


Summary


ofImage combines CPU pixel storage (ofPixels) and GPU texture
(ofTexture); call image.load("file.png") and
image.draw(x, y). Modify pixels on the CPU with
image.getPixels().setColor(x, y, col) then
image.update() to push changes to the GPU. Use ofFbo as an
offscreen render target: fbo.begin(), draw normally,
fbo.end(), then fbo.draw() or bind its texture for shader input.
