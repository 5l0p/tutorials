DSP - Part 7: Filter Cookbook: Bandpass, Notch & Shelves
2026-08-31


The previous article covered lowpass and highpass filters - the most common starting
points. This article completes the Audio EQ Cookbook family: bandpass filters, notch
filters, peaking EQ, and shelving filters. Together these cover every band shape you
need for a parametric equaliser or a synthesis signal chain.

Shared Intermediate Values


Every formula below uses the same pre-computation from the previous article. We factor
it into a helper to avoid repeating it:



#include <math.h>

#define M_PI 3.14159265358979323846f

typedef struct {
    float b0, b1, b2, a1, a2;
} BiquadCoeffs;

typedef struct {
    float w0, cos_w, sin_w, alpha;
} FilterIntermed;

static FilterIntermed filter_intermed(float fc, float q, float fs) {
    FilterIntermed v;
    v.w0    = 2.0f * M_PI * fc / fs;
    v.cos_w = cosf(v.w0);
    v.sin_w = sinf(v.w0);
    v.alpha = v.sin_w / (2.0f * q);
    return v;
}


Bandpass Filter (BPF)


A bandpass filter passes a band of frequencies centred on Fc and attenuates everything
above and below. The Audio EQ Cookbook provides two variants:



Constant skirt gain: peak gain is Q - louder at higher Q.
Constant 0 dB peak gain: peak is always 0 dBFS regardless of Q.



The constant 0 dB version is generally more useful in EQ contexts:



/*
  Bandpass, constant 0 dB peak gain.
  b0 =   sin_w / 2  =  sin_w / (2*Q) * Q  = alpha * Q
  b1 =   0
  b2 =  -sin_w / 2
  a0 =   1 + alpha
  a1 =  -2 * cos_w
  a2 =   1 - alpha
*/
void biquad_bpf(BiquadCoeffs *c, float fc, float q, float fs) {
    FilterIntermed v = filter_intermed(fc, q, fs);
    float a0 = 1.0f + v.alpha;

    c->b0 =  (v.sin_w / 2.0f)  / a0;
    c->b1 =   0.0f;
    c->b2 = -(v.sin_w / 2.0f)  / a0;
    c->a1 = -2.0f * v.cos_w    / a0;
    c->a2 = (1.0f - v.alpha)   / a0;
}



The bandwidth of the bandpass filter in octaves is related to Q by:



BW_octaves ≈ 1 / Q   (at moderate Q values)

Q = 0.7071  →  ~1.4 octaves bandwidth
Q = 1.0     →  ~1 octave
Q = 2.0     →  ~0.5 octave
Q = 10.0    →  ~0.1 octave (very narrow)


Notch Filter (Band-Reject)


A notch filter is the complement of a bandpass: it removes a band of frequencies
and passes everything else. Q controls the width of the notch in the same way as the
bandpass. The Cookbook coefficients:



void biquad_notch(BiquadCoeffs *c, float fc, float q, float fs) {
    FilterIntermed v = filter_intermed(fc, q, fs);
    float a0 = 1.0f + v.alpha;

    c->b0 =  1.0f           / a0;
    c->b1 = -2.0f * v.cos_w / a0;
    c->b2 =  1.0f           / a0;
    c->a1 = -2.0f * v.cos_w / a0;
    c->a2 = (1.0f - v.alpha)/ a0;
}



Note that b0 = b2 = 1/a0 and b1 = a1. This makes sense: a notch passes all frequencies
except the centre frequency, where it produces complete cancellation.


Peaking EQ Filter


A peaking EQ (or peak/dip filter) boosts or cuts a band of frequencies by dBgain dB at
the centre frequency Fc, with Q controlling the width. It is the workhorse of a
parametric EQ. The Cookbook uses an auxiliary amplitude factor A:



A = 10^(dBgain / 40)     /* sqrt of the linear amplitude */



Then:



void biquad_peak(BiquadCoeffs *c, float fc, float q,
                 float db_gain, float fs) {
    FilterIntermed v = filter_intermed(fc, q, fs);
    float A  = powf(10.0f, db_gain / 40.0f);
    float a0 = 1.0f + v.alpha / A;

    c->b0 = (1.0f + v.alpha * A)  / a0;
    c->b1 = -2.0f * v.cos_w       / a0;
    c->b2 = (1.0f - v.alpha * A)  / a0;
    c->a1 = -2.0f * v.cos_w       / a0;
    c->a2 = (1.0f - v.alpha / A)  / a0;
}



Setting db_gain = 0 gives a flat response (an all-pass with gain = 1). Positive
values boost; negative values cut. The same Q is used for both boost and cut.


Low Shelf Filter


A low shelf boosts or cuts all frequencies below Fc by dBgain dB, with a smooth
transition centred at Fc. It is used for bass enhancement or bass reduction in mastering
and mixing.



void biquad_low_shelf(BiquadCoeffs *c, float fc, float q,
                      float db_gain, float fs) {
    float A  = powf(10.0f, db_gain / 40.0f);
    float w0 = 2.0f * M_PI * fc / fs;
    float cw = cosf(w0), sw = sinf(w0);
    float al = sw / 2.0f * sqrtf((A + 1.0f/A) * (1.0f/q - 1.0f) + 2.0f);
    float sq = 2.0f * sqrtf(A) * al;

    float a0 = (A+1.0f) + (A-1.0f)*cw + sq;

    c->b0 =  A * ((A+1.0f) - (A-1.0f)*cw + sq)   / a0;
    c->b1 =  2.0f*A * ((A-1.0f) - (A+1.0f)*cw)   / a0;
    c->b2 =  A * ((A+1.0f) - (A-1.0f)*cw - sq)   / a0;
    c->a1 = -2.0f * ((A-1.0f) + (A+1.0f)*cw)     / a0;
    c->a2 =  ((A+1.0f) + (A-1.0f)*cw - sq)        / a0;
}


High Shelf Filter


A high shelf boosts or cuts all frequencies above Fc. Its formula mirrors the low shelf
with sign changes on the cos_w terms:



void biquad_high_shelf(BiquadCoeffs *c, float fc, float q,
                       float db_gain, float fs) {
    float A  = powf(10.0f, db_gain / 40.0f);
    float w0 = 2.0f * M_PI * fc / fs;
    float cw = cosf(w0), sw = sinf(w0);
    float al = sw / 2.0f * sqrtf((A + 1.0f/A) * (1.0f/q - 1.0f) + 2.0f);
    float sq = 2.0f * sqrtf(A) * al;

    float a0 = (A+1.0f) - (A-1.0f)*cw + sq;

    c->b0 =  A * ((A+1.0f) + (A-1.0f)*cw + sq)   / a0;
    c->b1 = -2.0f*A * ((A-1.0f) + (A+1.0f)*cw)   / a0;
    c->b2 =  A * ((A+1.0f) + (A-1.0f)*cw - sq)   / a0;
    c->a1 =  2.0f * ((A-1.0f) - (A+1.0f)*cw)     / a0;
    c->a2 =  ((A+1.0f) - (A-1.0f)*cw - sq)        / a0;
}


A Parametric EQ in C


With these formulas in hand, building a multi-band parametric EQ is straightforward.
Here is a 5-band EQ (low shelf, three peaking bands, high shelf):



typedef struct {
    float b0, b1, b2, a1, a2;
    float s1, s2;
} BiquadTDF2;

void biquad_set(BiquadTDF2 *f, const BiquadCoeffs *c) {
    f->b0=c->b0; f->b1=c->b1; f->b2=c->b2;
    f->a1=c->a1; f->a2=c->a2;
}

float biquad_process(BiquadTDF2 *f, float x) {
    float y = f->b0*x + f->s1;
    f->s1   = f->b1*x - f->a1*y + f->s2;
    f->s2   = f->b2*x - f->a2*y;
    return y;
}

#define SAMPLE_RATE 48000

typedef struct {
    BiquadTDF2 lo_shelf;
    BiquadTDF2 peak[3];
    BiquadTDF2 hi_shelf;
} ParamEQ5;

void param_eq5_init(ParamEQ5 *eq) {
    BiquadCoeffs c;

    biquad_low_shelf (&c, 100.0f, 0.7071f, 0.0f, SAMPLE_RATE);
    biquad_set(&eq->lo_shelf, &c);

    biquad_peak(&c,  400.0f, 1.5f,  0.0f, SAMPLE_RATE);
    biquad_set(&eq->peak[0], &c);
    biquad_peak(&c, 1000.0f, 1.5f,  0.0f, SAMPLE_RATE);
    biquad_set(&eq->peak[1], &c);
    biquad_peak(&c, 5000.0f, 1.5f,  0.0f, SAMPLE_RATE);
    biquad_set(&eq->peak[2], &c);

    biquad_high_shelf(&c, 8000.0f, 0.7071f, 0.0f, SAMPLE_RATE);
    biquad_set(&eq->hi_shelf, &c);
}

float param_eq5_process(ParamEQ5 *eq, float x) {
    x = biquad_process(&eq->lo_shelf, x);
    for (int i = 0; i < 3; i++)
        x = biquad_process(&eq->peak[i], x);
    x = biquad_process(&eq->hi_shelf, x);
    return x;
}


Testing: Frequency Response Measurement


To verify a filter's frequency response offline, feed it an impulse (1.0 at sample 0,
0.0 thereafter) and examine the output. The magnitude of the DFT of the impulse response
is the filter's magnitude response.



#include <stdio.h>

void measure_response(BiquadTDF2 *f, float fs) {
    /* Impulse response */
    float ir[4096] = {0.0f};
    ir[0] = 1.0f;
    for (int i = 1; i < 4096; i++)
        ir[i] = biquad_process(f, ir[i]);
    ir[0] = biquad_process(f, ir[0]);

    /* Print magnitude at a few spot frequencies using DFT */
    float test_hz[] = {100, 500, 1000, 5000, 10000};
    for (int fi = 0; fi < 5; fi++) {
        float w = 2.0f * M_PI * test_hz[fi] / fs;
        float re = 0.0f, im = 0.0f;
        for (int i = 0; i < 4096; i++) {
            re += ir[i] * cosf(w * i);
            im += ir[i] * sinf(w * i);
        }
        float mag = sqrtf(re*re + im*im);
        printf("%.0f Hz: %.2f dB\n",
               test_hz[fi], 20.0f * log10f(mag + 1e-30f));
    }
}


Summary


Bandpass filters pass a frequency band and reject the rest. Notch filters do the
opposite. Peaking EQ boosts or cuts around a centre frequency while leaving the
rest flat. Low and high shelf filters apply gain below or above a transition
frequency. All share the same w0/alpha/A intermediates derived in the previous
article. Cascade them in series — each biquad independently processes the
output of the previous — to build a full parametric equaliser.
