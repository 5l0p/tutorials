Containers - Part 3: The clone() and unshare() Syscalls
2026-08-01


Two syscalls are the foundation of every container runtime: clone()
creates a new process and optionally places it in new namespaces at birth, and
unshare() moves the calling process itself into new namespaces without
forking. Understanding them at the syscall level removes the mystery from
tools like unshare(1), docker run, and runc.

clone() - Creating a Process in New Namespaces


clone() is the Linux-specific generalisation of fork(). Its
signature (in glibc wrapper form) is:



#include <sched.h>

int clone(int (*fn)(void *), void *stack, int flags, void *arg, ...);



The key difference from fork() is the flags argument. You can OR in
any of the CLONE_NEW* constants to create new namespaces for the child:



CLONE_NEWPID - new PID namespace
CLONE_NEWNS - new mount namespace
CLONE_NEWNET - new network namespace
CLONE_NEWUTS - new UTS namespace
CLONE_NEWIPC - new IPC namespace
CLONE_NEWUSER - new user namespace
CLONE_NEWCGROUP - new cgroup namespace



Unlike fork(), clone() takes a function pointer that the child
executes, and you must supply a stack (the child runs on a separate stack you
allocate).


Full C Example: clone() with Multiple Namespaces


#define _GNU_SOURCE
#include <sched.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <sys/types.h>
#include <sys/wait.h>
#include <unistd.h>

#define STACK_SIZE (1024 * 1024)

static int child_main(void *arg) {
    (void)arg;

    /* Inside new UTS namespace - set our own hostname */
    if (sethostname("container", 9) == -1) {
        perror("sethostname");
        return 1;
    }

    char host[64];
    gethostname(host, sizeof(host));
    printf("[child] hostname : %s\n", host);
    printf("[child] PID      : %d\n", getpid());
    printf("[child] UID      : %d\n", getuid());

    /* Exec a shell so we can poke around */
    char *argv[] = { "/bin/sh", NULL };
    execv("/bin/sh", argv);
    perror("execv");
    return 1;
}

int main(void) {
    char *stack = malloc(STACK_SIZE);
    if (!stack) { perror("malloc"); return 1; }

    int flags = CLONE_NEWUTS | CLONE_NEWPID | CLONE_NEWIPC |
                CLONE_NEWNS  | SIGCHLD;

    pid_t pid = clone(child_main,
                      stack + STACK_SIZE,
                      flags,
                      NULL);
    if (pid == -1) { perror("clone"); free(stack); return 1; }

    printf("[parent] child host PID: %d\n", pid);

    int status;
    waitpid(pid, &status, 0);
    free(stack);
    return WEXITSTATUS(status);
}



Compile with gcc -o container container.c and run with
sudo ./container. Inside the shell you will see PID 2 (the shell; the
child_main function that called exec became PID 1), and the hostname
"container" does not affect the host.


unshare() - Detaching from Namespaces in Place


unshare() moves the calling process into new namespaces without
forking. It takes the same CLONE_NEW* flags:



#include <sched.h>

int unshare(int flags);



A minimal C program that gives the current shell a new UTS namespace:



#define _GNU_SOURCE
#include <sched.h>
#include <stdio.h>
#include <unistd.h>

int main(void) {
    if (unshare(CLONE_NEWUTS) == -1) {
        perror("unshare");
        return 1;
    }
    sethostname("unshared-host", 13);

    char host[64];
    gethostname(host, sizeof(host));
    printf("new hostname: %s\n", host);

    /* Stay alive so you can inspect /proc/self/ns/uts */
    pause();
    return 0;
}



The shell-level wrapper unshare(1) calls this syscall internally:
sudo unshare --uts --pid --fork bash.



Note the --fork flag: for PID namespaces, unshare() detaches the
current process but any child processes will be in the new namespace with PID
2+. The process that called unshare() itself cannot become PID 1 - it
must fork a child that becomes PID 1.


Go: SysProcAttr.Cloneflags


Go's os/exec and syscall packages expose namespace creation via
syscall.SysProcAttr. Setting Cloneflags passes those flags directly
to the underlying clone() syscall:



package main

import (
    "fmt"
    "os"
    "os/exec"
    "syscall"
)

func main() {
    cmd := exec.Command("/bin/sh")
    cmd.Stdin  = os.Stdin
    cmd.Stdout = os.Stdout
    cmd.Stderr = os.Stderr

    cmd.SysProcAttr = &syscall.SysProcAttr{
        Cloneflags: syscall.CLONE_NEWUTS |
                    syscall.CLONE_NEWPID |
                    syscall.CLONE_NEWIPC |
                    syscall.CLONE_NEWNS,
    }

    if err := cmd.Run(); err != nil {
        fmt.Fprintln(os.Stderr, err)
        os.Exit(1)
    }
}



Run with sudo go run main.go. The shell it launches runs inside the four
new namespaces. This is almost exactly what container runtimes do before
configuring the rootfs and cgroups.


Go: unshare via syscall.Unshare


The Go equivalent of unshare() is syscall.Unshare():



package main

import (
    "fmt"
    "os"
    "syscall"
)

func main() {
    err := syscall.Unshare(syscall.CLONE_NEWUTS)
    if err != nil {
        fmt.Fprintln(os.Stderr, "unshare:", err)
        os.Exit(1)
    }

    err = syscall.Sethostname([]byte("go-container"))
    if err != nil {
        fmt.Fprintln(os.Stderr, "sethostname:", err)
        os.Exit(1)
    }

    hostname, _ := os.Hostname()
    fmt.Println("hostname:", hostname)
}


Side-by-Side Comparison


The table below compares the two approaches:



clone() / Cloneflags - child is born in the new namespace. The parent retains its original namespaces. Correct for container runtimes.
unshare() / syscall.Unshare - the calling process moves to the new namespace. Any further forks inherit it. Correct for tools that want to reconfigure themselves (e.g., the shell unshare(1) command).



For PID namespaces specifically, clone() with CLONE_NEWPID is the
only way to get PID 1 in the new namespace. unshare(CLONE_NEWPID) only
affects future children of the calling process; the calling process itself
is not renumbered.


Capability Requirements


Creating most namespace types requires CAP_SYS_ADMIN. The exception is
CLONE_NEWUSER: user namespaces can be created by unprivileged processes
on kernels where /proc/sys/user/max_user_namespaces is non-zero (and
/proc/sys/kernel/unprivileged_userns_clone is 1 on some distributions).
Rootless container runtimes like Podman use this extensively.



# Check if unprivileged user namespaces are allowed
cat /proc/sys/kernel/unprivileged_userns_clone
# 1 = allowed, 0 = root-only


The /proc/sys/user Limits


The kernel enforces per-user limits on namespace creation to prevent
resource-exhaustion attacks. The limits live in /proc/sys/user/:



ls /proc/sys/user/
max_cgroup_namespaces
max_ipc_namespaces
max_mnt_namespaces
max_net_namespaces
max_pid_namespaces
max_time_namespaces
max_user_namespaces
max_uts_namespaces



Each file contains an integer limit per user. The default is typically 7680 on
modern kernels. Setting any of them to 0 disables creation of that namespace
type by non-root users entirely.


Summary


clone() creates a child process born directly inside new namespaces — the
right tool for container runtimes. unshare() moves the calling process
into new namespaces — the right tool for shell utilities. In Go,
SysProcAttr.Cloneflags maps directly to clone() flags. For PID
namespaces, clone() is the only way to achieve PID 1 in the new namespace;
unshare(CLONE_NEWPID) only affects future children. Most namespace creation
requires CAP_SYS_ADMIN except user namespaces, which unprivileged processes
can create when permitted by kernel configuration.
